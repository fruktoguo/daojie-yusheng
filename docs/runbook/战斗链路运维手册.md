# 战斗链路运维手册

解决战斗不结算、技能不生效、怪物行为异常等战斗系统问题的诊断入口。

## 可观测事件

### 结构化拒绝日志

运行时输出 `combat_action_rejected` 结构化日志，关键字段：

- `reason`：机器可读拒绝原因（`line_of_sight_blocked`、`target_instance_mismatch`、`no_runtime_targets_in_warning_cells`）
- `phase`：动作阶段（`instant`、`chant_start`、`chant_resolve`）
- `actor`：发起者（`player:<id>` 或 `monster:<runtimeId>`）
- `action`：动作/技能 id
- `instance`：地图实例 id

### 内部分层事件

`buildCombatEvents()` 产出四层事件：AOI 表现、玩家通知、内部审计、内部诊断。审计和诊断不进入客户端 S2C payload。

## 诊断流程

### 怪物吟唱预警格命中但没有攻击信息

1. 查 `combat_action_rejected` 日志
2. 看 `phase=chant_resolve`
3. `no_runtime_targets_in_warning_cells` → 完成时预警格没有权威玩家目标
4. `line_of_sight_blocked` → 有目标但视线校验拒绝
5. `target_instance_mismatch` → 玩家位置真源和实例占位不一致
6. 没有拒绝日志 → 跑 `smoke:world-runtime-combat-action-service` 验证诊断链

### 怪物一直释放同一技能

1. 跑 `pnpm --filter @mud/server smoke:monster-skill`
2. 确认唤灵真人配置顺序和 fallback smoke 是否通过
3. 看技能冷却、血量阶段、目标状态 buff 或地形条件是否让高优先级技能不可用
4. 运行时选择和配置 smoke 是权威判断，不要只看客户端表现

### 玩家或怪物攻击完全没有通知

1. 判断是"未命中/闪避"还是"没有进入结算"
2. 闪避应有 combat notice
3. 没有 notice → 查 diagnostic reason
4. 合法落空应有 `no_targets` 或 `no_runtime_targets_in_warning_cells`
5. 异常落空 → 需补结构化拒绝原因或 smoke

## 验证命令

```bash
pnpm --filter @mud/server smoke:world-runtime-combat-action-service
pnpm --filter @mud/server smoke:combat
pnpm --filter @mud/server smoke:monster-combat
pnpm --filter @mud/server smoke:monster-skill
pnpm --filter @mud/server bench:combat
pnpm audit:protocol
pnpm audit:boundaries
pnpm --filter @mud/server proof:production-boundaries
```

## 当前验证不覆盖的范围

- 战斗审计持久化
- pending cast 断线重连/服务重启恢复
- 100 玩家 + 50 怪物并发场景
