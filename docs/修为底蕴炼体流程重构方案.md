# 修为、底蕴与炼体流程重构方案

## 1. 目标

本文用于重构当前版本的境界成长主流程，将原本偏“战斗直接推境界”的节奏，改为：

- `战斗 -> 获得少量直接修为 + 大量底蕴`
- `打坐 -> 消化底蕴并转化为修为`
- `炼体 -> 将底蕴先转入炼体经验池，再按时间持续消化`

本次方案的核心不是简单调倍率，而是改变资源流向与消化顺序，让：

- 战斗负责“积累资源”
- 打坐负责“消化修为”
- 炼体负责“消化底蕴”
- 底蕴从“修为满后自动吃掉的溢出”改为“明确需要处理的成长资源”

## 2. 当前版本问题

当前版本的境界成长存在以下结构性问题：

- 战斗与修炼都能直接推进境界修为，玩家容易形成“刷怪即升级”的线性体验。
- 底蕴当前主要是修为满后才出现的副产物，不是一个需要主动规划的资源。
- 底蕴会自动参与境界修为加速，玩家几乎不需要感知“消化”过程。
- 炼体当前是把底蕴直接瞬时转成经验，缺少沉淀和节奏层。
- 整体更像“战斗直接涨经验”，而不是“战斗得资粮、闭关化资粮”。

本次重构的目标，就是把这套体验拉回“战斗得资粮，打坐化资粮”的修仙节奏。

## 3. 设计总原则

- 战斗不再是境界升级的直接主渠道，而是底蕴积累主渠道。
- 打坐成为境界修为的主消化渠道。
- 底蕴不再自动注入境界修为，只能通过明确途径消耗。
- 炼体不再瞬时吃满，而是进入经验池后逐息消化。
- 资源流尽量单向清晰，避免“任何地方都能自动补一点”的隐式规则。
- 首版先重构境界修为、底蕴、炼体三条线，不主动扩散到突破条件、开天门结构、功法经验结构。

## 4. 新资源流总览

本次调整后，角色成长主资源分为三类：

### 4.1 境界修为

- 直接决定当前小境界进度。
- 主要来源改为：
  - 战斗产出的少量直接修为
  - 打坐产出的修为
- 不再自动吃底蕴加速，只有打坐时才允许底蕴参与转化。

### 4.2 底蕴

- 作为战斗后的主要沉淀资源。
- 主要来源改为：
  - 战斗产出的绝大部分成长值
  - 战斗直接修为部分在写入失败时的折返
- 只能用于：
  - 打坐时加速转化为境界修为
  - 转入炼体经验池

### 4.3 炼体经验池

- 不是直接的炼体经验条，而是待消化池。
- 来源是底蕴转入。
- 进入经验池后不会立刻升级，而是按固定速率逐息消化到炼体经验条。

## 5. 战斗产出重构

## 5.1 战斗成长基值

保留现有战斗经验基值的计算思路。  
也就是当前战斗击杀结算里，原本用于推境界修为的那套“怪物等级、怪物层次、衰减、贡献占比、倍率修正”仍可继续作为基础产值来源。

本文把这部分值统一称为：

- `战斗成长基值`

举例：

- 若某次战斗当前按旧口径应获得 `100` 点境界修为，则本方案下视为本次获得了 `100` 点战斗成长基值。

## 5.2 战斗拆分规则

战斗成长基值不再整体直接进入境界修为，而是默认拆成两部分：

- `10%` 进入境界修为
- `90%` 进入底蕴

首版默认口径：

- `直接修为 = round(战斗成长基值 * 0.1)`
- `底蕴 = 战斗成长基值 - 直接修为`

举例：

- 原本 `100` 点战斗经验
- 现在改为：
  - `10` 点境界修为
  - `90` 点底蕴

## 5.3 直接修为不受底蕴影响

战斗获得的直接修为必须与底蕴系统解耦：

- 不读取当前底蕴数量
- 不自动消耗底蕴补差
- 不触发“有底蕴则额外加速”的逻辑

也就是：

- 战斗给多少直接修为，就只按战斗拆分结果写多少
- 底蕴只单独入账，不参与这次战斗的直接修为放大

## 5.4 写入失败时的折返规则

战斗拆分出的直接修为在以下场景可能无法完整写入：

- 当前小境界已满
- 本次直接修为超过当前剩余修为槽

首版建议统一规则：

- 未能写入境界修为的部分，一律折返为底蕴

这样可以保证：

- 战斗成长值不会平白损失
- 玩家在满修为或接近满修为时，战斗收益自然更偏向底蕴

## 6. 打坐流程重构

## 6.1 打坐定位

当前“修炼”在本方案中，建议明确为“打坐/闭关”语义。

调整后的定位：

- 打坐是境界修为的主要消化渠道
- 打坐是底蕴转修为的唯一通道
- 打坐不再只是被动挂机涨一点，而是成长主循环中的必要步骤

即整体节奏改为：

- `战斗拿资粮`
- `打坐化资粮`
- `突破升境`

## 6.2 打坐基础收益

打坐时，每息直接获得全量修为收益。

这里的“全量修为”指：

- 打坐本身的基础修为产出不再被拆分
- 打坐产出的基础值全部直接进入境界修为

也就是说：

- 战斗产出是“拆分后入账”
- 打坐产出是“完整修为入账”

## 6.3 底蕴加速规则

若玩家当前有可用底蕴，则打坐可触发三倍加速。

首版建议定义为：

- 基础打坐产出为 `X`
- 若底蕴足够，则本息实际获得 `3X` 境界修为
- 同时消耗 `2X` 底蕴

这样三倍的构成是：

- `1X` 来自正常打坐
- `2X` 来自底蕴燃烧加速

这套口径的优点是：

- 规则直观
- 与当前“底蕴自动参与三倍推进”的体验接近
- 但把触发点严格收敛到打坐，不再污染战斗结算

## 6.4 底蕴不足时的处理

若当前底蕴不足以支撑完整三倍加速，则建议按分段处理：

- 能支撑三倍时：获得 `3X` 修为，消耗 `2X` 底蕴
- 不足以支撑三倍时：只获得基础 `1X` 修为，不做部分倍率插值

首版不建议做 `1.4x`、`2.1x` 这类半档倍率，原因是：

- 玩家感知弱
- 规则不直观
- UI 解释成本高

如后续需要更细腻的消化感，再追加“按可用底蕴线性补足”的版本。

## 6.5 打坐满修为时的规则

当前小境界已满时，打坐不应再继续把修为转成底蕴，否则会形成自循环。

因此首版建议：

- 若当前境界修为已满且尚未突破，打坐的修为推进停止
- 此时不再额外消耗底蕴加速境界
- 打坐仍可保留其他不冲突收益
  - 例如功法经验
  - 这部分是否继续保留，见本文“待确认项”

## 7. 底蕴的唯一消费口

本方案下，底蕴只能进入两条消费路径：

1. 打坐时转化为境界修为
2. 转入炼体经验池

除此之外，底蕴不再自动参与：

- 战斗修为加速
- 任意场景的隐式补差
- 满修为时的自动溢出反哺

也就是说，底蕴从“隐藏辅助资源”改为“显式待消化资源”。

## 8. 炼体系统重构

## 8.1 调整目标

当前炼体是：

- 消耗底蕴
- 立即转成炼体经验
- 立即触发升级

本方案改为：

- 消耗底蕴
- 先进入炼体经验池
- 再按每息固定速率持续消化

也就是让炼体和境界一样，都从“即时结算”改成“需要时间消化”。

## 8.2 新增炼体经验池

新增概念：

- `炼体经验池`

用途：

- 存放由底蕴转入、但尚未真正消化的炼体经验

资源流变为：

- `底蕴 -> 炼体经验池 -> 炼体当前经验 -> 炼体升级`

## 8.3 底蕴转炼体经验池

保留“玩家主动把底蕴灌入炼体”的交互，但效果改为：

- 不再直接提升炼体经验
- 只把对应值写入炼体经验池

转化倍率本次不强制改动。  
若沿用现有口径，可暂时继续保持：

- `1 点底蕴 = 5 点炼体经验池`

后续若要重平衡，再单独调整。

## 8.4 炼体经验池消化速率

炼体经验池每息自动消化固定量经验，写入当前炼体经验条。

用户提出的目标口径为：

- 每息消化“当前升级上限所需经验”的千分之一

首版建议公式：

- `每息炼体消化量 = 当前炼体 expToNext / 1000`

若实现层只走整数，建议：

- `每息炼体消化量 = max(1, ceil(expToNext / 1000))`

这样炼体升级时间和当前层级需求自动绑定：

- 下一层需求越大，单息消化量越高
- 但总体仍保持“约一千息消化完一整层”的量纲

## 8.5 炼体经验池的行为建议

首版建议炼体经验池按以下规则处理：

- 只要经验池里有值，就每息自动消化
- 不需要额外再开一个“炼体打坐”开关
- 当前层升阶后，下一息按新 `expToNext` 重新计算消化量
- 经验池允许长期积压，等待后续慢慢消化

这样能把体验稳定在：

- 战斗获得底蕴
- 一部分去打坐化境界
- 一部分灌入炼体池慢慢养身体

## 9. 新流程示意

## 9.1 境界成长

```text
战斗
  -> 产出战斗成长基值 B
  -> 拆分为 10% 直接修为 + 90% 底蕴

打坐
  -> 基础获得修为 X
  -> 若底蕴足够，则额外燃烧 2X 底蕴
  -> 本息总共获得 3X 修为

突破
  -> 当境界修为满后，按原有突破条件进入下一境界
```

## 9.2 炼体成长

```text
战斗
  -> 获得底蕴

主动灌体
  -> 消耗底蕴
  -> 写入炼体经验池

每息自动消化
  -> 从炼体经验池扣除固定量
  -> 写入炼体经验条
  -> 满后升级
```

## 10. 与当前版本的关键差异

| 项目 | 当前版本 | 重构后 |
| --- | --- | --- |
| 战斗对境界 | 直接给大量修为 | 只给少量直接修为，大头改为底蕴 |
| 底蕴来源 | 主要来自满修为后的溢出 | 主要来自战斗拆分 |
| 底蕴用途 | 自动补境界、灌注炼体 | 只能打坐转修为或转炼体经验池 |
| 修炼/打坐 | 本身涨修为，也隐式吃底蕴 | 成为唯一的底蕴转修为入口 |
| 炼体 | 瞬时吃底蕴、瞬时升级 | 先入经验池，再逐息消化 |
| 成长节奏 | 战斗直接推等级 | 战斗拿资源，打坐和炼体负责消化 |

## 11. 范围边界

本方案首版默认只改以下内容：

- 战斗产出拆分
- 打坐与底蕴加速关系
- 底蕴消费口径
- 炼体经验池与炼体消化速率

本方案首版默认不主动改以下内容：

- 境界表和每级所需修为
- 突破条件表
- 开天门、逆天改命、碎灵丹等现有消耗结构
- 功法层数经验表
- 功法经验获取公式

## 12. 实现建议拆分

后续真正落地时，建议按以下顺序拆开发：

### 12.1 第一阶段：资源口径改造

- 移除战斗和打坐中“底蕴自动补境界”的旧逻辑
- 把战斗境界收益改成“直接修为 + 底蕴”拆分
- 保留旧 UI，先把服务端资源口径跑通

### 12.2 第二阶段：打坐重构

- 将打坐改成唯一底蕴转修为入口
- 加入三倍加速与底蕴消耗规则
- 调整打坐提示文案与面板说明

### 12.3 第三阶段：炼体经验池

- 新增炼体经验池存档字段
- 底蕴灌体改为写入经验池
- 每 tick 自动消化炼体经验池

### 12.4 第四阶段：UI 与提示补齐

- 面板显示：
  - 战斗获得的直接修为
  - 战斗获得的底蕴
  - 打坐是否触发底蕴加速
  - 炼体经验池剩余量
  - 炼体每息消化量

## 13. 待确认项

以下项目用户尚未明确，本方案先列出，后续落地前应再确认一次：

### 13.1 功法经验是否跟随打坐保留

当前未明确“打坐”是否继续提供功法经验。  
本文默认：

- 本次只重构境界修为、底蕴、炼体
- 功法经验流先不动

若后续要改，可再单独设计“战斗悟性/打坐参悟/功法熟练度”的新流程。

### 13.2 战斗拆分比例是否固定

当前提案明确示例为 `10 / 90`。  
首版建议把它做成配置常量，默认值为：

- `战斗直接修为比例 = 10%`
- `战斗底蕴比例 = 90%`

后续可按节奏再调。

### 13.3 打坐基础修为是否沿用当前修炼值

当前提案明确了“打坐给全量修为”和“底蕴时三倍加速”，但没有明确基础打坐值是否沿用现有每息修为值。  
首版建议：

- 先沿用当前修炼基础值与灵气乘区
- 只改资源流，不先改底层产量

### 13.4 炼体经验池是否自动消化

本文默认自动消化。  
如果后续想把炼体也做成主动闭关行为，可以再拆成：

- 自动缓慢消化
- 主动炼体时额外加速消化

## 14. 一句话总结

本次方案的本质是把当前“战斗直接升级”的经验制，改成“战斗积累底蕴，打坐消化修为，炼体慢慢化池”的修仙资源流。

战斗不再直接把人一路推着升境，而是把玩家推入“闭关消化”的下一阶段。
