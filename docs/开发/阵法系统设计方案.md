# 阵法系统设计方案

本文是阵法系统的开发规格草案，面向 next 主线落地评审。当前阶段只设计并准备实现 V1：通用阵盘、配置驱动阵法、布阵预览、服务端权威运行时、聚灵阵、固脉阵。

本文不描述已经上线的事实。落地目录以 `packages/*` 为准，`legacy/*` 只作为审计和旧规则参考。

## 结论

阵法是地图上的临时运行时对象，由玩家消耗阵盘、灵石和灵力布置。阵盘不绑定阵法，只提供灵气预算增幅。阵法模板由配置文件定义，配置决定资源折算、占比限制、范围曲线、目标选择和预置效果。

V1 只实现两个阵法：

- 聚灵阵：提升覆盖地块的 `aura.refined.neutral` 灵气，效果类似地图自带稳定灵气源，不给玩家加灵力回复 buff。
- 固脉阵：暂停覆盖范围内可攻击地块复生，并暂停玩家技能生成的临时地块自然消失；临时地块被摧毁时仍然消失。

阵法布下后，效果、范围、占比和阵盘倍率固化。后续只允许开启、关闭、补充灵力预算。关闭状态消耗速度为开启状态的 `1/10`。

## 设计目标

- 阵法配置化：新增阵法优先改内容配置，不改运行时流程。
- 阵盘通用化：阵盘只提供倍率，不绑定具体阵法。
- 服务端权威：资源扣除、实例创建、范围命中、地块改写、地块锁全部在服务端 tick 受控流程内完成。
- 可预览：布阵前可预览资源消耗、阵法总灵气预算、效果值、范围、开启/关闭消耗和预计持续时间。
- 可验证：每个玩法规则都有可写 smoke 的验收点。
- 可扩展：后续可增加护身阵、杀伐阵、长期阵法、阵眼破坏，但不污染 V1 主链。

## 非目标

- V1 不做跨地图阵法。
- V1 不做永久阵法和宗门大阵。
- V1 不做多人共同维持。
- V1 不做阵眼、破阵、驱散。
- V1 不做配置脚本或任意表达式执行。
- V1 不把阵法实例作为长期数据库真源；临时实例重启丢失可以接受。

## 核心规则

### 资源模型

布阵消耗三类资源：

- 阵盘：背包消耗物品，布阵成功后消耗 1 个。
- 灵石：决定阵法基础灵气预算。
- 玩家灵力：作为启动/布阵成本，由阵法配置公式决定。

阵盘品阶倍率：

| 阵盘 | 倍率 |
| --- | ---: |
| 凡品阵盘 | 1 |
| 黄阶阵盘 | 2 |
| 玄阶阵盘 | 4 |
| 地阶阵盘 | 8 |

基础计算：

```text
基础灵气预算 = 投入灵石数量 * auraPerSpiritStone
阵法总灵气预算 = 基础灵气预算 * 阵盘倍率
```

聚灵阵 V1 采用：

```text
auraPerSpiritStone = 100
```

示例：

```text
投入灵石 = 100
阵盘 = 玄阶阵盘，倍率 4
基础灵气预算 = 100 * 100 = 10000
阵法总灵气预算 = 10000 * 4 = 40000
```

### 三项占比

布阵时玩家分配三项占比：

- 效果占比
- 范围占比
- 持续占比

约束：

```text
effectPercent + rangePercent + durationPercent = 100
每项最低 10
每项最高 80
默认值：33 / 33 / 34
```

计算：

```text
效果灵气 = 阵法总灵气预算 * effectPercent / 100
范围灵气 = 阵法总灵气预算 * rangePercent / 100
持续只看 durationPercent，不直接把持续占比换成一份灵气数值
```

示例：

```text
阵法总灵气预算 = 40000
效果占比 = 80
范围占比 = 80

效果灵气 = 32000
范围灵气 = 32000
```

上例只用于说明单项上限下的效果或范围推导。实际布阵时三项总和仍必须等于 100，不能同时让效果和范围都为 80。

### 持续消耗

持续占比决定预算消耗速度，不直接决定阵法初始效果。

```text
基准持续占比 = 33
开启状态每日消耗 = 阵法总灵气预算 / (durationPercent / 33)
关闭状态每日消耗 = 开启状态每日消耗 * 0.1
```

示例：阵法总灵气预算 `40000`。

| 持续占比 | 开启每日消耗 | 预计开启持续 |
| ---: | ---: | ---: |
| 33 | 约 40000 | 约 1 天 |
| 66 | 约 20000 | 约 2 天 |
| 80 | 约 16500 | 约 2.42 天 |

关闭状态只降低消耗并停止效果，不重算阵法强度和范围。

### 固化规则

布阵成功时固化：

- 阵法模板 ID
- 阵盘物品 ID 和倍率
- 灵石投入
- 灵力消耗
- 三项占比
- 阵法总灵气预算
- 效果灵气
- 范围灵气
- 最终范围
- 目标效果值
- 开启/关闭消耗速度

布阵后不可修改：

- 阵法类型
- 阵盘
- 三项占比
- 效果强度
- 范围

布阵后允许：

- 开启
- 关闭
- 补充灵力预算

补充只增加 `remainingAuraBudget`，不增强阵法。

## 配置文件

建议新增：

```text
packages/server/data/content/formations.json
packages/shared/src/formation-types.ts
```

`formations.json` 是服务端内容真源。`formation-types.ts` 定义共享类型、预览输入输出、客户端可展示的公开摘要。

客户端可以用公开摘要做本地预览，但确认布阵时服务端必须重新计算，不能接受客户端传回的派生结果。

### 公式

阵法公式复用技能公式结构，避免新增一套 DSL。

```ts
type FormationFormula = SkillFormula | { var: FormationFormulaVar; scale?: number };

type FormationFormulaVar =
  | 'formation.qiCost'
  | 'formation.spiritStoneCost'
  | 'formation.diskAmplifier'
  | 'formation.baseAuraBudget'
  | 'formation.totalAuraBudget'
  | 'formation.effectAura'
  | 'formation.rangeAura'
  | 'formation.durationPercent'
  | 'formation.remainingAuraBudget'
  | 'formation.activeDailyAuraCost'
  | 'formation.inactiveDailyAuraCost'
  | 'caster.realmLevel'
  | 'caster.attr.spirit';
```

规则：

- 运算符沿用 `add/sub/mul/div/min/max/clamp`。
- 不支持配置写 JS。
- 变量解析表由阵法系统独立提供。
- 公式结果必须经过服务端归一化，例如取整、限制最小值、限制最大值。

### 范围配置

阵法范围复用 `TargetingGeometrySpec`，最终解析成现有 targeting 几何。

可复用形状：

| shape | 含义 |
| --- | --- |
| `single` | 单格 |
| `line` | 直线 |
| `area` | 圆形 |
| `box` | 方形/矩形 |
| `orientedBox` | 朝向矩形 |
| `ring` | 环形 |
| `checkerboard` | 棋盘 |

阵法额外支持范围曲线：

```ts
interface FormationAreaDef {
  targeting: TargetingGeometrySpec;
  rangeCurve?: FormationRangeCurveDef;
}

type FormationRangeCurveDef =
  | {
      type: 'doublingSteps';
      source: 'formation.rangeAura';
      baseAura: number;
      minRadius: number;
      maxRadius: number;
    }
  | {
      type: 'anchoredGeometricSquare';
      source: 'formation.rangeAura';
      anchorAura: number;
      anchorRadius: number;
      baseAura: number;
      growthRate: number;
      minRadius: number;
      maxRadius: number;
    }
  | {
      type: 'thresholdTable';
      source: 'formation.rangeAura';
      thresholds: Array<{ radius: number; aura: number }>;
    };
```

曲线只负责把 `rangeAura` 解析成半径或额外范围。解析后仍交给 `computeAffectedCellsFromAnchor()` 计算覆盖格。

### 目标选择

## 聚灵阵

### 玩法语义

聚灵阵提升覆盖地块的灵气。它不是玩家 buff，也不是 `qiRegenRate` 修改器。

实现目标：

- 覆盖地块的 `aura.refined.neutral` 向目标灵气值收敛。
- 阵法大约需要一天把覆盖地块推到目标灵气表现。
- 达到目标值后，阵法补充速率与目标灵气对应的衰减速率平衡。
- 阵法关闭、预算耗尽或移除后，源项停止，地块回到原有灵气衰减/恢复逻辑。

### 数值规则

```text
targetAura = effectAura * conversionRatio
conversionRatio = 1
```

示例：

```text
阵法总灵气预算 = 40000
效果占比 = 80
effectAura = 32000
targetAura = 32000
```

范围：

```text
shape = area
baseAura = 1000
每翻倍增加 1 格半径
minRadius = 1
```

示例：

```text
rangeAura = 32000
rangeAura / baseAura = 32
log2(32) = 5
radius = 5
```

## 固脉阵

### 玩法语义

固脉阵作用于地块运行时：

- 覆盖范围内可攻击地块不会复生。
- 覆盖范围内玩家技能生成的临时地块不会自然消失。
- 临时地块被攻击摧毁时仍然消失。
- 阵法关闭、预算耗尽或移除后，地块恢复原有规则。

### 数值规则

效果灵气在 V1 只作为门槛：

```text
effectAura >= 1 时生效
```

范围：

```text
shape = box
baseAura = 100
growthRate = 1.5
anchorAura = 32000
anchorRadius = 8
minRadius = 1
```

商业实现必须用测试锁定关键样例。若后续策划继续调曲线，可以把 `anchoredGeometricSquare` 替换成 `thresholdTable`，避免浮点边界导致线上表现不稳定。

```text
rangeAura = 32000
radius = 8
覆盖范围 = 15 x 15
```

## 服务端运行时

建议新增：

```text
packages/server/src/runtime/world/world-runtime-formation.service.ts
```

职责：

- 查询阵法模板。
- 校验布阵请求。
- 解析阵盘倍率。
- 扣除玩家灵力、灵石和阵盘。
- 解析固化值。
- 创建和索引阵法实例。
- 处理开启、关闭、补充。
- 在 tick 中扣减剩余预算。
- 在 tick 中执行阵法效果。
- 生成 AOI 投影。

请求入口只入队玩家意图。实际状态变化必须在 tick 执行链中发生。

### 布阵执行顺序

1. 收到 `createFormation` 请求，只记录玩家意图。
2. tick 消费意图。
3. 校验玩家当前地图、坐标和状态。
4. 校验阵法模板、阵盘、灵石、灵力和占比。
5. 计算 `baseAuraBudget`、`totalAuraBudget`、`effectAura`、`rangeAura`。
6. 解析范围和效果值。
7. 扣除灵力。
8. 扣除灵石。
9. 消耗阵盘。
10. 创建阵法实例。
11. 单播操作结果和资源变化。
12. AOI 广播阵法出现。

资源扣除和实例创建必须保持同一 tick 内的原子语义。任何一步失败都不能留下半扣资源或半创建实例。

### Tick 结算

每张地图 tick 内处理：

1. 扫描本地图阵法实例。
2. 根据 `active` 选择消耗速度。
3. 扣减 `remainingAuraBudget`。
4. 预算耗尽则移除实例并广播。
5. 未开启则跳过效果执行。
6. 到达 `nextEffectTick` 时解析覆盖格。
7. 执行效果。
8. 标记地块资源或地块规则 dirty。
9. 生成必要的 AOI delta。

热路径要求：

- 覆盖格在实例创建时预计算，除非地图结构变化需要重算。
- 不在 tick 中解析完整 JSON 配置。
- 不在 tick 中重复构造长字符串 key。
- 不在高频包发送完整阵法模板。

### 聚灵阵运行时

聚灵阵接入当前地块资源：

```text
resourceKey = aura.refined.neutral
```

结算口径：

- 对覆盖格读取当前灵气。
- 目标灵气为 `targetAura`。
- 如果当前灵气低于目标灵气，按半衰期收敛公式补充。
- 如果当前灵气达到或高于目标灵气，不额外抬升。
- 到目标值后补充速率与目标衰减速率平衡。

该效果会实际改写地块灵气，因此改写结果继续走现有地图 `tile_resource` 持久化主链。

### 固脉阵运行时

固脉阵不改写地图静态配置，只提供运行时锁。

建议在地图实例上提供查询：

```ts
isTerrainRespawnLocked(x: number, y: number): boolean
isTemporaryTerrainDecayLocked(x: number, y: number): boolean
```

调用点：

- 可攻击地块复生流程检查 `isTerrainRespawnLocked()`。
- 临时地块自然消失流程检查 `isTemporaryTerrainDecayLocked()`。
- 临时地块被摧毁流程不检查该锁。

### 请求

```ts
interface C2S_CreateFormationRequest {

  instanceId: string;
  spiritStoneCost: number;
  qiCost?: number;
}
```

### 低频静态

阵法模板公开摘要放低频静态层：

```ts
interface FormationTemplateSummary {
  id: string;
  name: string;
  desc: string;
  category?: string;
  cost: PublicFormationCostSummary;
  allocation: FormationAllocationDef;
  presentation?: FormationPresentationDef;
}
```

不要在世界 tick 包里发送模板说明、完整公式和长文本。

### 地图实例投影

```ts
interface FormationWorldProjection {
  instanceId: string;
  formationId: string;
  x: number;
  y: number;
  active: boolean;
  remainingAuraBudget: number;
  estimatedRemainingTicks: number;
  area: TargetingGeometrySpec;
}
```

事件建议：

- `formationAdded`
- `formationRemoved`
- `formationUpdated`

`formationUpdated` 只用于开启、关闭、补充、剩余预算低频变化。聚灵阵导致的地块灵气变化应复用已有地块资源/地图 delta，不通过阵法事件重复发送。

## 持久化边界

V1 临时阵法实例不落库。

需要持久化：

- 玩家消耗的阵盘。
- 玩家消耗的灵石。
- 玩家消耗的灵力后形成的玩家状态。
- 聚灵阵实际写入的地块灵气，走 `tile_resource` / `aura.refined.neutral`。

不持久化：

- 临时阵法实例。
- 临时阵法开启/关闭状态。
- 临时阵法剩余预算。
- 临时阵法覆盖格缓存。

如果后续做永久阵法、宗门大阵、地图机关阵，必须重新设计数据库真源，不能复用 V1 的内存实例假装长期状态。

## 客户端设计

### 布阵面板

必须展示：

- 阵法选择。
- 阵盘选择。
- 投入灵石。
- 玩家灵力消耗。
- 效果、范围、持续三项占比。
- 占比总和校验。
- 阵法总灵气预算。
- 效果预览。
- 范围预览。
- 开启/关闭状态下预计消耗和持续时间。

交互约束：

- 占比默认 `33/33/34`。
- 单项滑条最低 10，最高 80。
- 总和必须等于 100。
- 输入变化时局部更新预览，不重建整个面板。
- 地图预览只表现覆盖格，不承担合法性裁定。

### 地图表现

- 聚灵阵显示圆形覆盖区域。
- 固脉阵显示方形覆盖区域。
- 阵法显示开启/关闭状态。
- 阵法显示预计剩余时间。
- 感气视角中，聚灵阵影响下的地块灵气按现有灵气表现刷新。
- 固脉阵范围内可攻击地块和临时地块可以显示锁定提示。

## 内容校验

加载 `formations.json` 时必须校验：

- `id` 唯一。
- `allocation.default` 三项总和为 100。
- 每项默认值在 min/max 内。
- 每项 min/max 合法，且存在可行组合。
- `auraPerSpiritStone > 0`。
- `minSpiritStoneCost <= maxSpiritStoneCost`。
- `rangeCurve` 参数为正数。
- `effects` 非空。
- 效果类型必须是服务端已注册预置效果。
- `targets.kind` 与效果目标兼容。
- 聚灵阵必须使用 `aura.refined.neutral`。
- 固脉阵必须只作用地块目标。

配置错误应在启动期失败，不允许运行中静默忽略。

## 验收用例

基础校验：

- 无阵盘不能布阵。
- 灵石不足不能布阵。
- 灵力不足不能布阵。
- 占比总和不是 100 时拒绝。
- 任一占比低于 10 或高于 80 时拒绝。
- 非法阵法 ID 拒绝。
- 非法阵盘物品拒绝。

数值校验：

- 玄阶阵盘倍率为 4。
- 100 灵石在 `auraPerSpiritStone=100` 时基础灵气预算为 10000。
- 玄阶阵盘下总灵气预算为 40000。
- 效果占比 80 时效果灵气为 32000。
- 聚灵阵 `rangeAura=32000` 时半径为 5。
- 固脉阵关键样例 `rangeAura=32000` 时覆盖 `15x15`。
- 持续占比 33 时开启预计约 1 天。
- 持续占比 66 时开启预计约 2 天。
- 关闭后消耗速度为开启的 1/10。

聚灵阵：

- 不给玩家加 `qiRegenRate`。
- 只影响覆盖地块灵气。
- 地块灵气向目标值收敛。
- 目标值处补充和衰减达到平衡。
- 阵法关闭后不再补充地块灵气。
- 阵法预算耗尽后不再补充地块灵气。
- 聚灵阵写入后的地块灵气可通过现有地块详情读到。

固脉阵：

- 覆盖范围内可攻击地块不会复生。
- 阵法关闭后可攻击地块恢复复生流程。
- 覆盖范围内临时地块不会自然消失。
- 临时地块被摧毁时仍然消失。
- 范围外地块不受影响。

协议与 AOI：

- AOI 内玩家收到 `formationAdded`。
- AOI 外玩家不收到阵法实例。
- 开启、关闭、补充只发低频 `formationUpdated`。
- 高频 tick 不发送完整阵法模板。
- 聚灵阵地块灵气变化复用地块资源同步。

持久化：

- 阵盘扣除进入玩家持久化真源。
- 灵石扣除进入玩家持久化真源。
- 聚灵阵实际写入地块灵气进入地图地块资源持久化主链。
- 临时阵法实例重启后不恢复，且文案和 runbook 明确这是 V1 行为。

测试夹具：

- smoke 如果创建玩家、阵盘、灵石、地图实例或地块资源，必须带成功、失败、中断后的清理路径。

## 风险与约束

- 聚灵阵会实际改写地块灵气，必须确认写入频率不会造成地图资源持久化写放大。
- 固脉阵会改变地块生命周期，必须避免把运行时锁写回地图静态配置。
- 范围曲线必须服务端归一化，避免客户端预览和服务端实算不一致。
- 补充预算不得重算范围和效果，否则会绕过“布阵后配比不可改”的核心规则。
- 高频同步只能发送实例状态和地块资源变化，不能发送完整模板。
- 未来永久阵法必须单独设计数据库真源，不能从 V1 临时实例平滑偷渡。
