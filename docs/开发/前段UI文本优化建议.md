## 总结

### 核心问题分类

1. **工程术语入侵**（最高优先级修复）
   - "同步" → 应改为 `查探`、`调阅`、`观望`
   - "会话" → 应改为 `气机`、`印记`
   - "重连" → 应改为 `重归`、`重续`
   - "请求" → 应改为 `呈报`、`上达`
   - "读取/加载" → 应改为 `查探`、`观阅`
   - "地块" → 应改为 `地`、`方`、`所在`
   - "角色" → 应改为 `身`、`此身`
   - "服务端" → 应改为 `天道`（玩家不需要知道server）
   - "NPC" → 直接使用 `修士`、`人` 或具体NPC名
   - "PVP" → `争锋`、`斗法`
   - "开发者" → `天道执掌` 或 `执掌者`

2. **过度解释**（中等优先级）
   - 大量tooltip/hint文本过长，像产品需求文档
   - 应当在提示中精简为一句话，详细说明放到帮助系统

3. **风格不一致**
   - 同一概念在不同面板用了不同措辞
   - 有的很有仙侠味（`飞书台`、`气机细察`），有的很现代（`确认弹窗`、`帧率浮层`）

4. **alert() 弹窗**（建议移除）
   - `suggestion-panel.ts` 中多处使用原生 `alert()` 做表单验证
   - 应统一替换为游戏内toast或内联错误提示

5. **GM面板**（不需要仙侠化）
   - GM面板的所有文本（`gm.ts`、`gm-world-viewer.ts`、`gm-map-editor.ts`）应保持工程化
   - 这些是给GM/开发者看的工具界面，强行仙侠化反而影响效率

### 建议优先级

1. **P0（必须改）**: 第一、二、三章 — toast提示和空状态，出现频率最高
2. **P1（高优先级）**: 第十七、十八、十九章 — 世界迁移、观察/详情弹窗
3. **P2（中等）**: 第四-十六章 — 各面板细节调整
4. **P3（低优先级）**: 代码注释清理、changelog微调
