# 功法加成设计

本文档以当前主线配置为基础，数据源为 `packages/server/data/content/techniques/**/*.json`；内功数值按本设计文档中的重分配口径调整，属于目标设计值。

统计口径：

- “效果”列的属性为该功法满层后的目标累计值。
- 术法、妖兽、秘术、神通的主要价值通常在 `skills`；若无 `skills` 且无属性，本文按“机制预留/占位”处理，不假定已有隐藏收益。
- 当前六维顺序为：体魄、神识、身法、根骨、力道、经脉。
- 角色最终属性还会经过同品阶功法池软衰减；本文说明的是单本功法设计值，不直接等同最终面板增量。

## 总体设计口径

- 内功是长期成长与基础面板来源，允许满层后提供较高的累计六维。
- 术法以主动技能、冷却、耗灵倍率、范围和状态效果为主，不提供直接六维加成，避免同时抢占内功定位。
- 妖兽类用于承载怪物、异类或 Boss 专用技能组，同样不提供直接六维加成；其强度通过技能数量、冷却、范围、状态和 AI 模板控制。
- 秘术用于后续规则改写、探索、掉落、运势或特殊战斗机制；在机制未落地前不提供直接六维，避免形成必学面板税。
- 神通是高稀缺技能槽，数值重点应在高冷却强技能和释放条件上，直接六维只保留少量方向性补强。
- 内功重分配口径：`混元载脉图` 降到约三分之一，削减量乘 2 后回流给此前内功；凡阶不参与该次回流，黄阶和玄阶按 6:4 承接。
- 内功六维口径：默认保持各品阶满层总属性池不变；若某品阶主动下修，则在该品阶内部维持境界等级递增和功法特色。单本功法按名称和定位集中到 2 到 3 个主项，不再追求单本或品阶内六维齐平。

## 后续调整原则

- 若新增同品阶内功，优先按境界等级落入对应满层总六维区间；同等级再按功法定位微调，不让低等级总量压过高等级。
- 术法和妖兽类默认不补直接六维；若未来确需例外，必须先说明为什么不能通过技能、冷却、范围、状态或 AI 模板解决。
- 若秘术开始落地机制，应先定义机制价值，再决定是否需要极少量属性；不要先补六维。
- 若继续调整力道、经脉投放，需要先确认它们在数值公式中的边际收益，并保持“品阶总池均衡、单本功法按主题偏科”的口径。
- 若高阶功法数量增加，必须同步复查同品阶软衰减上限，避免“全学一遍”成为唯一最优路线。
