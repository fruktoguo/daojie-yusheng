# 持久化设计

本文统计当前 `packages/server` 主线实际建表代码中的 PostgreSQL 持久化结构，并说明现在的拆表方式。

统计来源：

- `packages/server/src/persistence/**`
- 少量仍写 `persistent_documents` 的运行时服务：宗门、GM 配置等
- 当前执行环境没有 `SERVER_DATABASE_URL/DATABASE_URL`，因此本文件按主线代码中的 `CREATE TABLE IF NOT EXISTS`、`ALTER TABLE ADD COLUMN IF NOT EXISTS` 和实际写入路径整理，不是 live DB 反查结果。

## 拆表方式

当前拆表不是按“一个大对象一张表”，而是按状态拥有者和持久化域拆：

| 层级 | 拆法 | 说明 |
| --- | --- | --- |
| 通用文档 | `persistent_documents(scope, key, payload)` | 低频配置、迁移输入、GM 元数据、宗门等仍在 scope JSON 中 |
| 玩家身份 | `server_player_identity / player_identity` | 登录身份和玩家 ID 映射 |
| 玩家快照 | `server_player_snapshot` | 玩家全量兜底快照，分域未覆盖或迁移时使用 |
| 玩家分域 | `player_*` | 位置、血量、境界、背包、装备、任务、战斗偏好、日志等按域拆表 |
| 邮件 | `player_mail*` | 邮件主体、附件、归档、计数分表 |
| 强一致操作 | `durable_operation_log / outbox_event / asset_audit_log*` | 资产操作日志、事务外投递、审计 |
| 集群路由 | `node_registry / player_session_route` | 节点和玩家会话路由 |
| 刷盘队列 | `player_flush_ledger / instance_flush_ledger` | 玩家和地图实例的 domain worker 认领队列 |
| 实例目录 | `instance_catalog` | 地图实例元信息、租约、归属、销毁时间 |
| 实例分域 | `instance_*` | 地图资源、地块损坏、地面物品、容器、怪物运行态、事件、overlay、checkpoint 等 |

当前实例持久化拆表状态：

| 域 | 当前表 | 当前状态 |
| --- | --- | --- |
| 地块资源 | `instance_tile_resource_state` | 已拆，按 `instance_id + resource_key + tile_index` 存 |
| 地块损坏 | `instance_tile_damage_state` | 已拆，按 `instance_id + tile_index` 存 |
| 地面物品 | `instance_ground_item` | 已拆，但当前写入是按实例 replace |
| 容器 | `instance_container_state / instance_container_entry / instance_container_timer` | 已拆主状态、条目、计时器，运行时回读时重建容器 payload |
| 高价值怪物 | `instance_monster_runtime_state` | 已拆，普通低价值怪不落库 |
| 动态事件 | `instance_event_state` | 已有表和读写 |
| 动态 overlay | `instance_overlay_chunk` | 已拆 chunk，当前 runtime portal overlay 已接入 |
| 阵法 | `instance_formation_state` | 已拆，一阵一行；灵力余量、开关、阵眼位置和占比配置均为结构化列 |
| 冷字段兜底 | `instance_checkpoint` | 仍作为 checkpoint / fallback |
| 恢复水位 | `instance_recovery_watermark` | 实例级恢复元信息 |

还没有落到实例分域专表的 scope JSON：

- `server_sects_v1`：宗门当前仍存在 `persistent_documents`

## 通用文档表

### `persistent_documents`

| 字段 | 类型 | 内容 |
| --- | --- | --- |
| `scope` | `varchar(64)` | 文档域名，例如市场、GM、宗门、旧地图快照 |
| `key` | `varchar(100)` | 域内主键 |
| `payload` | `jsonb` | 域内完整 JSON 文档 |
| `updatedAt` | `timestamptz` | 更新时间 |

主键：`(scope, key)`。

| `server_map_aura_v1` | 旧地图快照 / 兼容恢复 / 迁移输入 |

| `server_redeem_codes_v1` | 兑换码文档 |
| `server_suggestions_v1` | 建议反馈文档 |
| `server_market_orders_v1` | 市场订单 |
| `server_market_trade_history_v1` | 市场交易历史 |
| `server_market_storage_v1` | 旧市场仓储 |
| `server_db_backups_v1` | GM 数据库备份元数据 |
| `server_db_jobs_v1` | GM 数据库任务状态 |
| `server_afdian_config_v1` | 爱发电配置 |

## 当前风险与未完成拆表

| 项 | 当前状态 |
