# 持久化表结构现状

本文统计当前 `packages/server` 主线实际建表代码中的 PostgreSQL 持久化结构。调度、dirty domain、worker、失败重试和恢复策略见 [持久化设计.md](./持久化设计.md)。

统计来源：

- `packages/server/src/persistence/**`
- `packages/server/src/runtime/world/**` 中仍直接建表或写 `persistent_documents` 的服务
- `packages/server/src/http/native/**` 中 GM 管理态持久化

当前执行环境没有 `SERVER_DATABASE_URL/DATABASE_URL`，因此本文按代码中的 `CREATE TABLE IF NOT EXISTS`、`ALTER TABLE ADD COLUMN IF NOT EXISTS`、实际写入路径整理，不是 live DB 反查。

## 总览

| 层级 | 表/结构 | 说明 |
| --- | --- | --- |
| 通用文档 | `persistent_documents(scope, key, payload)` | 低频配置、迁移输入、GM 元数据、宗门等 |
| 身份 | `server_player_identity / player_identity` | 登录身份和玩家 ID 映射 |
| 玩家兜底快照 | `server_player_snapshot` | 玩家全量兜底快照 |
| 玩家分域 | `player_*` | 位置、血量、境界、背包、装备、任务、偏好、日志等 |
| 邮件 | `player_mail*` | 邮件主体、附件、归档、计数 |
| 强一致操作 | `durable_operation_log / outbox_event / asset_audit_log*` | 资产操作、事务外投递、审计 |
| 集群路由 | `node_registry / player_session_route` | 节点和玩家会话路由 |
| 刷盘账本 | `player_flush_ledger / instance_flush_ledger` | 玩家和地图实例 domain worker 认领队列 |
| 实例目录 | `instance_catalog` | 地图实例元信息、租约、归属、销毁时间 |
| 实例分域 | `instance_*` | 地块、容器、怪物运行态、事件、overlay、checkpoint 等 |
| 阵法 | `instance_formation_state` | 阵法实例结构化真源 |

## 当前结构缺口

| 项 | 当前状态 |
| --- | --- |
| 宗门 | 仍是 `persistent_documents(scope=server_sects_v1)` |
| GM 地图配置 | 已持久化到 `persistent_documents(scope=server_gm_map_config_v1)`，不是专表 |
| 地块复生 wall-clock | `instance_tile_damage_state` 只有 `respawn_left_ticks`，没有 `respawn_at_ms` |

| 地面物品 | 表已拆，写入仍偏 replace-by-instance |
