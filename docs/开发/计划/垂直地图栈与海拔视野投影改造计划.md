# 垂直地图栈与海拔视野投影改造计划

> 本文面向 当前生产主线收敛“多层地图、海拔排序、跨层视野投影、地下遮蔽、GM 世界管理分组”的设计与实施计划。
>
> 主工作区固定为 `packages/*`。本文只定义计划与验收口径，不改变当前运行时代码。

---

## 目标结论

多层地图不应继续依赖 `floorLevel` 做运行时判断。正式模型应以“地图栈 + 海拔排序 + 覆盖范围 + 垂直遮蔽规则”为主：

- `floorLevel` 和 `floorName` 只作为显示、编辑与内容作者辅助字段。
- 运行时视野投影使用独立的海拔字段，例如 `elevation` 或 `baseElevation`。
- 同一主地图及其次级地图在逻辑层预先归入一个地图栈。
- 地图栈内的层按海拔排序，视野投影时不再临时查父图。
- 地下层默认封闭，不应因为存在 `parentMapId` 就自动透光。
- GM 世界管理应按“地图栈 + 分线”分组展示，主地图在前，次级地图缩进排列。

示例规则：

```text
玩家位于海拔 100 的第 10 层
-> 某坐标被海拔 90 的第 9 层覆盖：先看到 9 层，该坐标后续低层被 9 层遮挡
-> 另一个坐标没有被 9/8/7/6 层覆盖：继续向下扫描，直到命中海拔 50 的第 5 层
-> 地下层默认 sealed：除非有明确洞口、天井、井口、透明地面等规则，否则不参与透光
```

## 当前代码基线

### 已确认事实

- [x] 地图文档已有 `parentMapId / parentOriginX / parentOriginY / floorLevel / floorName / spaceVisionMode`。
- [x] 当前 `spaceVisionMode = parent_overlay` 只要求存在父地图和父图对齐坐标。
- [x] 当前没有正式的 `elevation`、`baseElevation`、`verticalVisionMode` 或地图栈索引。
- [x] 当前 `height` 表示地图行高，不能复用为海拔。
- [x] 当前二楼、门楼二层、部分地下地图都可能使用 `parent_overlay`。
- [x] 当前地下层没有统一的“默认不透光”运行时语义。
- [x] 当前 GM 世界管理实例列表按 `templateId + templateName` 分组。
- [x] 当前 GM 实例摘要没有 `rootMapId / parentMapId / elevation / stackGroup` 这类分组字段。

### 当前风险

- [ ] `parent_overlay` 语义过粗：既表达“属于父图”，又表达“可以看见父图”，无法区分室内二楼、地下室、洞窟、湖底、天井等情况。
- [ ] `floorLevel` 只能表达粗略楼层，不能表达海拔差、半层、坡地、塔楼、山崖平台等非建筑楼层。
- [ ] 视野链路如果继续按当前图临时投影父图，后续十层、半覆盖、多层遮挡会变成大量特判。
- [ ] GM 端如果继续自己按模板分组，会把同一主地图的次级地图拆成多个独立组，分线展示也会混乱。

## 领域模型

### 地图模板字段

建议在地图文档和共享类型中补充以下字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `rootMapId` | `string` | 地图栈根地图 ID；主地图可等于自身 |
| `parentMapId` | `string?` | 直接父地图 ID，保留现有语义 |
| `parentOriginX/Y` | `number?` | 当前地图左上角在父图或根图上的对齐坐标 |
| `elevation` | `number` | 运行时海拔，视野排序主字段 |
| `floorLevel` | `number?` | 展示和编辑辅助，不参与运行时排序 |
| `floorName` | `string?` | 展示名称 |
| `verticalVisionMode` | `open / sealed / shaft / transparent / one_way` | 垂直视野语义 |
| `verticalOcclusionMode` | `blocks / partial / none` | 命中本层后是否遮挡后续层 |

第一版可以保守收敛为：

- 主地图默认 `rootMapId = id`，`elevation = 0`。
- 地上二层默认 `verticalVisionMode = open`，`verticalOcclusionMode = blocks`。
- 地下层默认 `verticalVisionMode = sealed`，`verticalOcclusionMode = blocks`。
- 未显式填写的新字段通过加载期 normalize 生成，不要求一次性改完所有旧地图 JSON。

### 地图栈索引

启动期建立只读索引：

```ts
interface MapStackLayer {
  mapId: string;
  rootMapId: string;
  parentMapId?: string;
  parentOriginX: number;
  parentOriginY: number;
  rootOriginX: number;
  rootOriginY: number;
  width: number;
  height: number;
  elevation: number;
  floorLevel?: number;
  floorName?: string;
  verticalVisionMode: string;
  verticalOcclusionMode: string;
}

interface MapStack {
  rootMapId: string;
  layersByElevationAsc: MapStackLayer[];
  layersByElevationDesc: MapStackLayer[];
}
```

运行时实例再绑定分线：

```ts
interface RuntimeMapStack {
  rootMapId: string;
  linePreset: string;
  lineIndex: number;
  layersByElevationDesc: Array<MapStackLayer & { instanceId: string }>;
}
```

## 视野投影规则

### 坐标口径

- [ ] 所有跨层判断统一转成 root 坐标。
- [ ] 当前地图本地坐标只用于读取当前层地块。
- [ ] 父图坐标只作为兼容字段，不作为长期投影中心。
- [ ] `parentOriginX/Y` 在加载期被折算为 `rootOriginX/Y`。
- [ ] 地图层 footprint 使用 `rootOriginX/Y + width/height` 表示。

### 层级扫描

- [ ] 玩家所在层由 `currentMapId + instanceId` 唯一确定。
- [ ] 玩家视野内每个本地坐标先投影到 root 坐标。
- [ ] 从当前层海拔开始，按目标方向扫描同一 `RuntimeMapStack`。
- [ ] 每层先判断 root 坐标是否落入该层 footprint。
- [ ] 未覆盖该坐标的层不参与遮挡，继续扫描下一层。
- [ ] 命中覆盖层后，根据该层 `verticalVisionMode` 和 `verticalOcclusionMode` 决定是否显示、遮挡或继续穿透。

### 遮蔽与地下

- [ ] 地下层默认 `verticalVisionMode = sealed`。
- [ ] 地下层不因为 `parentMapId` 或 `parentOriginX/Y` 自动透光。
- [ ] 可透光地下入口必须显式配置为 `shaft`、`transparent` 或等价模式。
- [ ] 普通地板、屋顶、岩层默认遮挡低层。
- [ ] 窗户只影响同层水平视线，不自动代表垂直透视。
- [ ] 天井、井口、破洞、透明地板等需要后续独立地块语义或遮罩支持。

### 视觉强度

- [ ] 投影结果附带 `elevationDelta`。
- [ ] 投影结果附带 `visibilityStrength` 或等价枚举。
- [ ] 小海拔差可显示完整地块与实体。
- [ ] 中海拔差可弱化实体和地块细节。
- [ ] 大海拔差默认只显示地形轮廓或完全隐藏实体。
- [ ] 具体表现由客户端渲染层消费，不把视觉效果写死在运行时规则里。

## GM 世界管理分组

### 服务端合同

`GmWorldInstanceSummary` 建议补充：

| 字段 | 说明 |
|---|---|
| `rootMapId` | 所属地图栈根地图 |
| `rootMapName` | 根地图展示名 |
| `parentMapId` | 直接父地图 |
| `elevation` | 海拔排序 |
| `floorLevel` | 展示楼层 |
| `floorName` | 展示楼层名 |
| `stackGroupKey` | `rootMapId + linePreset + lineIndex` |
| `stackOrder` | 同组内排序 |
| `isRootLayer` | 是否主地图层 |

### 前端展示

- [ ] 左侧列表先按 `stackGroupKey` 分组。
- [ ] 每组标题显示主地图名、分线类型、线路序号。
- [ ] 主地图实例顶格显示在组内第一项。
- [ ] 次级地图按海拔顺序缩进显示。
- [ ] 次级地图标题比主地图短一截，形成文档目录式层级。
- [ ] 多条线时，每条线显示完整地图栈。
- [ ] 不再让每个次级地图单独显示为一条独立分线。
- [ ] 选中次级地图时仍以 `instanceId` 查询运行态，不改变 GM 详情接口主键。

## 分阶段计划

## 阶段 0：冻结语义

- [ ] 明确 `floorLevel` 只做显示，不参与运行时排序。
- [ ] 明确 `elevation` 是跨层视野排序的唯一主字段。
- [ ] 明确 `height` 保持地图尺寸语义，禁止复用为海拔。
- [ ] 明确 `parent_overlay` 进入兼容期，后续由地图栈和垂直视野字段承接。
- [ ] 明确地下默认不透光，显式洞口或透明结构才允许透光。
- [ ] 明确 GM 分组复用服务端地图栈结果，不在客户端二次猜父子结构。

## 阶段 1：共享类型与地图文档 schema

- [ ] 在 `packages/shared` 增加地图垂直字段类型。
- [ ] 补充 normalize 默认值。
- [ ] 补充校验：有 `parentMapId` 的地图必须能归入某个 `rootMapId`。
- [ ] 补充校验：`elevation` 必须是有限数字。
- [ ] 补充校验：地下或负楼层地图默认不能自动启用开放式垂直视野。
- [ ] 保留旧字段兼容读取，避免一次性破坏现有地图数据。

## 阶段 2：启动期地图栈索引

- [ ] 在服务端地图模板仓储或相邻只读服务中建立 `MapStackIndex`。
- [ ] 按 `rootMapId` 聚合地图模板。
- [ ] 把每层 `parentOriginX/Y` 折算为 root 坐标。
- [ ] 生成升序、降序两套海拔排序数组。
- [ ] 检测循环父子关系。
- [ ] 检测同一 root 下重复或缺失海拔的风险，并给出诊断。
- [ ] 为运行时提供按 `mapId` 查询所属 stack 和 layer 的 O(1) 接口。

## 阶段 3：运行时实例栈绑定

- [ ] 按 `rootMapId + linePreset + lineIndex` 建立 `RuntimeMapStack`。
- [ ] 主地图实例和次级地图实例归入同一分线地图栈。
- [ ] 创建新分线时同时准备该 root 下必要的次级地图实例。
- [ ] 玩家跨层传送时保持同一分线，不意外掉回默认公共线。
- [ ] GM 手动创建分线时返回完整 stack 信息。

## 阶段 4：跨层视野投影

- [ ] 把当前 `parent_overlay` 父图投影收敛为地图栈投影查询。
- [ ] 视野计算保留当前同层 LOS 规则。
- [ ] 对可跨层坐标执行 root 坐标扫描。
- [ ] 命中层后返回目标 `instanceId / mapId / x / y / elevationDelta / visibilityStrength`。
- [ ] 地块、玩家、怪物、NPC、传送点、地标、容器、地面物品统一走同一投影结果。
- [ ] AOI 仍只返回玩家可见范围内的实体，不扩大到全图或全栈广播。

## 阶段 5：客户端渲染表现

- [ ] 客户端消费投影地块时区分当前层和跨层来源。
- [ ] 不同海拔差使用不同透明度、颜色或轮廓表现。
- [ ] 跨层实体可按强度显示弱化状态。
- [ ] 地下 sealed 层不显示跨层地块和实体。
- [ ] 保持浅色、深色、手机模式下可读。
- [ ] 保持当前主地图渲染和普通移动交互不被跨层表现打断。

## 阶段 6：GM 世界管理展示

- [ ] 服务端 GM 实例列表返回 stack group 字段。
- [ ] GM 左侧列表按 stack group 渲染。
- [ ] 主地图顶格，次级地图缩进。
- [ ] 同一分线内只显示一个组标题。
- [ ] 多分线显示为多组完整地图栈。
- [ ] 保持点击实例后仍按 `instanceId` 查看运行态。
- [ ] 保持列表刷新时选中项、滚动位置和展开状态连续。

## 阶段 7：内容迁移

- [ ] 为 `yunlai_town` 及其二楼、门楼、地窖、暗室、矿窖、湖底洞天补齐垂直字段。
- [ ] 二楼和门楼按开放式地上层处理。
- [ ] 地窖、暗室、矿窖默认 sealed。
- [ ] 湖底洞天保持 isolated 或明确设为 sealed，除非后续设计要求能从湖面透视。
- [ ] 检查所有 `parent_overlay` 地图，逐个决定是开放层、封闭层还是独立层。
- [ ] 更新地图内容一致性检查，禁止新地下图默认开放透光。

## 阶段 8：验证与回归

- [ ] 增加地图栈索引单测或 smoke。
- [ ] 增加二楼窗口可见父图的回归验证。
- [ ] 增加地下层不可透光的回归验证。
- [ ] 增加十层示例：上层半覆盖时只遮挡覆盖区域。
- [ ] 增加 GM 实例列表按地图栈和分线分组的验证。
- [ ] 运行 `pnpm --filter @mud/server compile`。
- [ ] 涉及客户端后运行对应客户端构建或最小 UI 验证。
- [ ] 涉及协议字段后确认 shared/server/client 三端类型一致。

## 非目标

- [ ] 本计划不直接设计完整 3D 地图。
- [ ] 本计划不把玩家移动从二维网格改为三维坐标。
- [ ] 本计划不把所有地图一次性改造成多层建筑。
- [ ] 本计划不要求客户端一次性实现复杂光照、透视或高度差动画。
- [ ] 本计划不改变服务端玩家每秒一次操作、tick 权威、AOI 最小范围广播等主线约束。

---

## 完成定义

本计划完成时应满足：

- [ ] 地图加载期能稳定生成地图栈索引。
- [ ] 运行时跨层视野不再以 `floorLevel` 判断。
- [ ] 地下地图默认不会透光。
- [ ] 上层半覆盖遮挡规则可证明。
- [ ] 不同海拔差能向客户端输出可表现的强度信息。
- [ ] GM 世界管理按地图栈和分线分组展示。
- [ ] 主地图和次级地图在 GM 中是一组，不再各自伪装成独立分线。
- [ ] 现有二楼窗口看一楼的能力不回退。
- [ ] 验证链覆盖 server compile、跨层视野 smoke、GM 分组最小验证。
