# 前端技术性文本排查报告

> 目标：排查 `packages/client` 中所有面向玩家可见的技术性文本、开发术语、内部实现暴露，收集问题并给出修改建议。
> 范围：普通玩家可见的所有 UI 文本（不含 GM 面板）。
> 原则：不直接修改，仅记录和建议。

## 一、高优先级（明显不适合玩家看到）

### T-001 FPS tooltip 暴露"客户端"和技术术语

- **文件**：`src/main-runtime-monitor-source.ts:311-312`
- **原文**：
  - `'客户端当前渲染帧率未采样'`
  - `` `客户端当前渲染帧率约 ${...} FPS，LOW ${...}，1% LOW ${...}` ``

## 五、修改原则建议

1. **去技术化**：将所有"客户端"、"服务端"、"帧率"、"采样"、"浮层"等术语替换为玩家能理解的日常用语
2. **去实现化**：将"本地"、"会话"、"同步"、"持久化"等暴露存储/网络实现的词汇移除或替换
3. **去策划化**：将"乘区"、"底座"、"软衰减"、"移动预算"、"基时"等数值策划内部术语改为通俗表述
4. **世界观一致**：将"出生点"、"Debug"等与修仙世界观不搭的词汇替换为符合游戏设定的表述
5. **安全 fallback**：所有 fallback 文本不应暴露内部标识符（英文 trigger 名、ID 等），应使用通用描述兜底
6. **i18n 统一**：所有面向玩家的硬编码中文应迁移至 i18n 体系，便于后续维护和多语言扩展
