# 地图地块多层拆分改造计划

> 解决问题：将 `TileType` 从单一枚举拆分为多层 cell state（terrain/surface/structure/interactable），使地形、铺装、结构、交互物可共存，消除规则写死和投影混乱。

## 目标模型

```
cell → terrain（草地/泥地/石地/沼泽/水/虚空）
     → surface（道路/地板/地毯，可选）
     → structure（墙/门/窗/树/矿，可选）
     → groundItems（掉落物堆，沿用现有对象层）
     → interactables（阵法/箱子/传送点，沿用对象层）
     → 服务端合成 walkable/blocksSight/durability/renderTile
```

## 合成规则

```ts
walkable = terrain.walkable && (surface?.allows ?? true) && !structure?.blocksMove && !blockingInteractable
blocksSight = terrain.blocksSight || structure?.blocksSight || sightBlockingInteractable
traversalCost = surface ? surface.traversalCost : terrain.traversalCost
durability = structure ? structureDurability(builder, multiplier) : terrainDurability(realmLv, multiplier)
```

## 性能设计

- composite flags 在 layer 变化时增量重算（`Uint32Array`）
- 寻路/视线直接读 bitfield，消除字符串比较
- 网络 patch 只发变化 cell，不全量重发
- cell 查询不分配短命对象
