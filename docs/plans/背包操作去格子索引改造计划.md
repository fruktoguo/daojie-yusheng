# 背包操作去格子索引改造计划

> 解决问题：任何玩家资产操作都不能依赖背包数组下标或 UI 格子位置。客户端只能提交稳定物品身份，服务端在权威执行时按当前背包真源解析目标，避免强化、丢弃、拿取、市场、装备等操作在背包重排后串物品。

## 状态

已完成。

## 目标口径

- 玩家操作 payload 不再携带 `slotIndex` 作为目标身份。
- 服务端资产操作统一按 `itemInstanceId` 定位背包物品。
- 堆叠物品拆分、消耗、丢弃、挂单、强化、装备、阵法创建都以同一套身份解析规则执行。
- `slot_index` 可以继续作为数据库展示顺序、面板排序和 patch 位置，但不能作为玩家操作目标。
- 装备槽位 `weapon/head/body/legs/accessory` 是语义槽，不属于本计划要移除的背包格子索引。

## 当前问题链路

### 使用 / 丢弃 / 摧毁

数据流：客户端背包按钮 dataset `slotIndex` -> `C2S.UseItem/DropItem/DestroyItem` -> gateway -> command queue -> tick 执行 -> `PlayerRuntimeService` 按 `player.inventory.items[slotIndex]` 取物品。

风险：命令入队到执行之间，背包可能因拾取、丢弃、强化完成、市场结算、整理、堆叠合并而重排，旧下标可能指向另一件物品。

### 装备

数据流：客户端点击背包装备 -> payload `slotIndex + expectedItemInstanceId` -> command queue -> `dispatchEquipItem` -> `equipItem(playerId, slotIndex, expectedItemInstanceId)`。

风险：`expectedItemInstanceId` 目前是乐观校验，迁移软模式下 mismatch 可能只 warn。核心定位仍是 `slotIndex`。

### 强化

数据流：强化候选 `ref.source='inventory', slotIndex` -> `startEnhancement` -> `resolveEnhancementTarget` 按下标取目标 -> `extractInventoryItemAt` 移入 `lockedItems` -> 结束后按 `itemInstanceId` 解锁回背包。

风险：锁定空间后的链路正确，但启动前目标解析仍依赖下标。强化期间的队列 payload 也可能保存旧下标。

### 市场 / 拍卖

数据流：客户端寄售选择 `slotIndex` -> `CreateMarketSellOrderView` / `SellMarketItemView` -> `MarketRuntimeService.createSellOrder/sellNow` -> 按下标拆背包物品。

风险：寄售确认前后背包重排会卖错目标；堆叠拆分仍可能落到错误堆叠。

### 阵法创建

数据流：客户端阵盘弹窗保存 `slotIndex` -> `FormationCreatePayload.slotIndex` -> `WorldRuntimeFormationService.dispatchCreateFormation` -> 按下标读取并消耗阵盘。

风险：阵盘选择后背包变化会创建/消耗错误物品。

## 边界决策

| 范围 | 决策 |
|------|------|
| 玩家操作目标 | 必须使用 `itemInstanceId` |
| 操作数量 | 使用独立 `count` 字段，服务端从目标实例当前 count 中拆分 |
| 旧客户端兼容 | 不再接受旧 `slotIndex` 操作目标；缺少 `itemInstanceId` 时 fail closed |
| 数据库 `slot_index` | 保留为排序/展示/兼容字段，不作为操作身份 |
| PanelDelta `slotIndex` | 暂时保留为客户端列表 patch 坐标；不是操作目标 |
| 装备槽位 | 保留语义槽位，不改成 instanceId |
| 堆叠合并 | 继续按 `itemId#enhanceLevel` 合并；合并后保留胜出堆叠的 `itemInstanceId` |

## 实施步骤

### Phase 1：共享协议与目标类型

- [x] 新增共享类型 `InventoryItemRef`，字段至少包含 `itemInstanceId: string`。
- [x] 将 `UseItemView` 改为 `itemRef + count? + use options`。
- [x] 将 `DropItemView` / `DestroyItemView` 改为 `itemRef + count`。
- [x] 将 `EquipView` 改为 `itemRef`，移除 `expectedItemInstanceId` 的双轨语义。
- [x] 将 `CreateMarketSellOrderView` / `SellMarketItemView` 改为 `itemRef + quantity + price`。
- [x] 将 `EnhancementTargetRef` 的 inventory 目标改为 `itemInstanceId`，装备目标继续使用语义 `slot`。
- [x] 将 `FormationCreatePayload` 改为 `itemRef`。
- [x] 移除所有旧 `slotIndex` C2S payload 入口，禁止新增使用点。

### Phase 2：服务端背包身份解析层

- [x] 在 `PlayerRuntimeService` 增加按 `itemInstanceId` 查找当前背包项的窄口。
- [x] 增加 `splitInventoryItemByInstanceId(playerId, itemInstanceId, count)`。
- [x] 增加 `consumeInventoryItemByInstanceId(playerId, itemInstanceId, count)`。
- [x] 增加 `destroyInventoryItemByInstanceId(playerId, itemInstanceId, count)`。
- [x] 增加 `useItemByInstanceId(playerId, itemInstanceId, options)`。
- [x] 增加 `equipItemByInstanceId(playerId, itemInstanceId)`。
- [x] 所有新窄口在找不到实例、实例在锁定空间、count 不足、类型不符时 fail closed。

### Phase 3：服务端业务入口替换

- [x] `WorldRuntimeUseItemService` 改为按 `itemInstanceId` 执行使用、建宗、地图解锁、复活点绑定、地块资源物品。
- [x] `WorldRuntimeItemGroundService` 丢弃改为按 `itemInstanceId` 拆出物品并落地。
- [x] `WorldRuntimeEquipmentService` 装备改为按 `itemInstanceId` 取目标。
- [x] `WorldRuntimeFormationService` 创建阵法改为按 `itemInstanceId` 校验并消耗阵盘。
- [x] `CraftPanelRuntimeService` 强化目标和保护物解析改为按 `itemInstanceId`，队列 payload 不再保存下标。
- [x] `MarketRuntimeService` 挂单、拍卖、快速出售改为按 `itemInstanceId` 拆分。
- [x] command queue 的 `useItem/dropItem/equip/startEnhancement/createFormation` 不再保存背包下标。

### Phase 4：客户端选择状态与发包替换

- [x] 背包面板按钮和弹窗保存 `itemInstanceId`，不保存 `slotIndex`。
- [x] 使用、丢弃、摧毁、装备发包改为 `itemRef`。
- [x] 强化候选 key 从 `inventory:${slotIndex}` 改为 `inventory:${itemInstanceId}`。
- [x] 强化保护物选择值改为 `itemInstanceId`。
- [x] 市场普通寄售、拍卖寄售、快速出售改为保存 `itemInstanceId`。
- [x] 阵法创建弹窗保存阵盘 `itemInstanceId`。
- [x] UI 渲染仍可使用数组顺序生成 DOM，但事件目标必须从 item 自身读取身份。

### Phase 5：兼容与清理

- [x] gateway 不再接受旧 `slotIndex` payload，缺少稳定实例 ID 直接拒绝。
- [x] 移除旧 `slotIndex` C2S 字段和兼容转换。
- [x] 检查 `ITEM_INSTANCE_ID_HARD_CHECK` 迁移语义：背包资产操作已按实例 ID fail closed；剩余 `expectedItemInstanceId` 仅用于装备语义槽乐观校验，不再承担背包定位。
- [x] 更新相关 mechanics 文档中“背包槽位”作为操作目标的表述。

## 不能做的事

- [x] 不能把 `itemId` 当操作身份；同类物品存在不同强化等级、不同实例和不同锁定状态。
- [x] 不能继续在 tick 执行阶段用客户端提交的数组下标二次查物品。
- [x] 不能让锁定空间物品被普通背包操作命中。
- [x] 不能为了兼容旧客户端让 runtime 层继续接受 `slotIndex`。
- [x] 不能把装备语义槽位和背包格子索引混为一谈。
