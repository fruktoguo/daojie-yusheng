# 刷盘优化计划

> 目标：降低玩家刷盘与 flush ledger 的体积膨胀，明确各分域是否全量落盘，逐步把最重的域拆成更薄的写入路径。

## 完成记录

- 已将玩家 projectable flush task 的 ledger payload 从“同一玩家多 domain 复用同一份组合 snapshot”改为“每个 domain 只保存该 domain-gated snapshot”。消费侧逐 task 按单 domain 写入 `PlayerDomainPersistenceService.savePlayerSnapshotProjectionDomains()`，避免一个重 payload 被同一玩家多个 ledger row 重复保存。
- 新增 `player-flush-payload-report`，with-db 下按 `player_flush_ledger` / `instance_flush_ledger` domain 统计 `payload_jsonb` 平均、最大、p95、payload row 数，并统计玩家分域 JSONB 细节列体积分布与 ledger relation size。
- 新增 `flush-ledger-retention-smoke`，with-db 下 seed completed ledger rows，验证旧 `payload_jsonb` 被置空、超过 row retention 的 completed 空 payload row 被删除。
- 本次 with-db 报告显示玩家 ledger 大头为 `inventory`（p95 11542B，max 11542B）、`attr`（p95 5498B，max 11911B）、`body_training/profession/progression`（p95 4-5KB 量级）；玩家分域真源 JSONB 列整体明显小于 ledger 中转 payload，例如 `player_inventory_item.raw_payload` p95 5B、max 65B。
- retention 验证前后 `player_flush_ledger` relation size 从约 183099392B 降到 63488000B，证明 completed payload 清理后可观察到物理空间下降；后续持续堆积可用同一 report 继续观测增长是否停止。
- 已将 `inventory` 真源写入从“全行无差别 upsert + 整玩家 stale cleanup”收敛为行级增量 apply：同 `item_instance_id` 数量变化只更新 `count/raw_payload`，格子变化通过临时槽 staging 后只移动 `slot_index`，未变行不刷新 `updated_at`。

## 开发任务

- [x] 识别 `inventory`、`buff`、`quest`、`progression` 的主要膨胀来源
  - `inventory`：ledger domain snapshot 中的 inventory item 集合，是当前最大玩家 domain payload。
  - `buff` / `quest`：ledger 中为 domain snapshot，最终真源为行级表 + `raw_payload` 细节列，report 可继续按列追踪。
  - `progression`：ledger payload 来自 progression bundle；最终真源热字段已列式化，低频细节在 `attr` / job / record 相关表承载。
- [x] 评估并落地 `inventory` 从整组重写继续拆成更细粒度写计划
  - 结论：已先落地 DB apply 侧 item-level diff；ledger payload 仍是 inventory domain snapshot，后续若继续变薄，可再把 ledger 输入拆成 item-level upsert/delete diff。
  - 最小实现：运行时背包先规范化成稳定行，再按 `item_instance_id` 和库内当前行做 key 级 diff；新增行 insert，只改数量/实例 payload 的行 update，格子变化行通过临时槽 staging 后 update `slot_index`，缺失行按 stale `item_instance_id` delete。

## 风险

- [x] 过度拆分会增加写路径复杂度
  - inventory 已先落地 DB apply 侧行级 diff，并通过 smoke 约束空覆盖、未变行不刷新与 flush 消费路径；ledger 输入是否继续拆成 item-level diff 仍需单独评估。
- [x] inventory 的 item_instance_id 修复必须保留
  - 已由 `player-domain-persistence-smoke` / `flush-presence-payload-smoke` 覆盖关键路径。
- [x] 不能把 dirty / retry / claim 中的账本误删
  - retention 仅作用于 completed、未 claim、未 retry-delay 的 rows；smoke 已验证 completed payload/row 清理路径。
- [x] 不能把 ledger 当成最终真源
  - 已通过 report 和代码路径区分 ledger 中转 payload 与玩家分域表真源。
