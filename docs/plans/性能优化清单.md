# 性能优化清单

## 待处理
- [ ] `packages/server/src/network/world-sync-threat.service.ts`
  - 问题：`diffThreatArrows()` 会为 `previous`、`current`、`adds`、`removes` 反复生成相同的 key；`buildThreatArrows()` 还会在极小数组时照常排序。
  - 影响：威胁箭头是同步包的一部分，重复 key 生成和无条件排序会放大 diff 路径的同步开销。
  - 处理建议：按 key 一次性建索引后做差集，且只在确有多项时排序。
  - 判断：可在不改变设计和表现的前提下修复。

## 已完成
- [x] `packages/server/src/network/world-sync-map-snapshot.service.ts`

  - 结果：减少视野同步热路径中的重复查询和对象访问开销，不改变行为。

  - 结果：减少 worker 池高频调度入口上的同步开销，不改变行为。
