# 通用行动意图队列计划

> 解决问题：将挖矿、采集和标准技艺统一到行动意图系统中，支持被动打断后自动恢复，替代现有 `TechniqueActivityQueueService`。

## 设计决策

| 决策 | 选择 |
|------|------|
| 挖矿启动入口 | 沿用锁定目标（engageBattle） |
| 主动移动 | remove（不再 sleep） |
| 持久化 | 不持久化（断线清空） |
| 与 pipeline 关系 | 意图层在 pipeline 之上 |
| 队列 vs 单条 | 队列，按 slot 分组 |

## 架构

```
ActionIntentService（统一意图调度器）
  push / sleep / wake / remove / tick / getActive / list
    │ 委托
    ▼
ActionIntentStrategy 接口（每种 kind 实现一个）
  execute / queryProgress / isCompleted / canResume / onSleep / onTerminate / conflictsWith
```

### Slot 分组

| Slot | 包含 kind | 互斥规则 |
|------|-----------|----------|
| `field` | mining, gather, building | 同时只能一条 active |
| `craft` | alchemy, forging, enhancement | 同时只能一条 active |

两个 slot 之间不互斥。

### 状态机

```
push → pending → (tick/条件满足) → active → (被动打断) → sleeping → (wake) → active
                                          → (主动终止/完成) → [removed]
```

## 事件→动作对照

**被动打断（sleep）**：被攻击、自动战斗临时锁敌
**恢复（wake）**：反击结束无敌人、回到采集点 1 格内、craft job 完成
**明确终止（remove）**：矿脉挖碎、容器耗尽、主动移动、关闭自动战斗、死亡、换图、断线

## 边界约束

- 不持久化（断线清空）
- 不在 tick 热路径做 DB IO
- 不改变现有 pipeline 内部逻辑
- 不改变现有协议事件名
- 队列最大长度 20
