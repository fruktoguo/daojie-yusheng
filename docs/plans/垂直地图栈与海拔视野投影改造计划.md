# 垂直地图栈与海拔视野投影改造计划

> 解决问题：多层地图不应继续依赖 `floorLevel` 做运行时判断，需要以"地图栈 + 海拔排序 + 覆盖范围 + 垂直遮蔽规则"为主，地下层默认不透光。

## 目标

- `elevation` 是跨层视野排序的唯一主字段（`floorLevel` 只做显示）
- 同一主地图及其次级地图归入一个地图栈，按海拔排序
- 地下层默认 sealed，不因 `parentMapId` 自动透光
- GM 世界管理按"地图栈 + 分线"分组展示

## 约束

- `height` 保持地图尺寸语义，禁止复用为海拔
- `parent_overlay` 进入兼容期，后续由地图栈承接
- 不直接设计 3D 地图，不改玩家移动为三维坐标
- 不改变 tick 权威、AOI 最小范围广播等主线约束

## 领域模型

### 地图模板新增字段

| 字段 | 说明 |
|------|------|
| `rootMapId` | 地图栈根地图 ID |
| `elevation` | 运行时海拔，视野排序主字段 |
| `verticalVisionMode` | `open / sealed / shaft / transparent / one_way` |
| `verticalOcclusionMode` | `blocks / partial / none` |

### 地图栈索引（启动期只读）

```ts
interface MapStack {
  rootMapId: string;
  layersByElevationDesc: Array<MapStackLayer & { instanceId: string }>;
}
```

## 视野投影规则

1. 所有跨层判断统一转成 root 坐标
2. 从当前层海拔开始，按目标方向扫描同一 RuntimeMapStack
3. 每层判断 root 坐标是否落入该层 footprint
4. 命中后根据 `verticalVisionMode` + `verticalOcclusionMode` 决定显示/遮挡/穿透
5. 投影结果附带 `elevationDelta` + `visibilityStrength`
