# 运行时事件总线设计计划

> 解决问题：统一散落在各处的运行时临时事件（notices/combatEffects/gmStatePush），提供语义化方法集 + 合并/限流/丢弃规则，tick 内收集、组包时 drain。

## 定位

- 不落库的运行时临时事件门面
- 语义化方法（无泛型 publish/subscribe）
- tick 内收集 → WorldSyncService 组包时 drain → flushTick 清理
- 不是 outbox 前置层，不承载资产正确性

## 公开 API

```typescript
@Injectable()
export class RuntimeEventBusService {
  queuePlayerNotice(playerId, notice): void;
  queuePlayerPanelPatch(playerId, panelKind, patch): void;
  queueActiveJobProgress(playerId, progress): void;
  queueTechniquePanelRefresh(playerId, kind): void;
  queueCombatPresentation(instanceId, event): void;
  queueGmStatePush(playerId): void;
  queueAoiPresentation(instanceId, event): void;
  queuePlayerStateDelta(playerId, delta): void;
  queuePlayerFeedback(playerId, feedback): void;
  flushPlayer(playerId): FlushResult;
  flushInstance(instanceId): FlushResult;
  flushTick(): FlushResult;
  discardPlayer(playerId): void;
  discardInstance(instanceId): void;
}
```

## 合并/限流规则

| 方法 | 模式 | 上限 |
|------|------|------|
| playerNotice | 追加 | 32/player/tick |
| panelPatch | 合并（同 panelKind 深合并） | 1/kind |
| jobProgress | 覆盖（同 jobId 只保留最后） | 1/job |
| techniquePanelRefresh | 去重 | 1/kind |
| combatPresentation | 追加 | 64/instance/tick |
| aoiPresentation | 追加+同实体合并 | 128/instance/tick |
| stateDelta | 合并 | 1/player |
| feedback | 追加 | 8/player/tick |

## 数据流

```
Runtime 各域 → queue*() → tick 末尾 WorldSyncService drain → worldDelta.eventBus → socket.emit
                                                           → flushTick() 记录指标并清空
```

## 与 Outbox 的关系

EventBus 永远不承载资产正确性。Outbox 是白名单能力：
- 允许：交易/拾取丢下/邮件/兑换码/GM
- 禁止：采集/挖矿/战斗表现/制作进度/面板刷新

白名单操作必须先完成领域事务，再 queue EventBus 表现事件。

## 实施步骤

| Phase | 内容 | 前置 |
|-------|------|------|
| 1 | 骨架 + queuePlayerNotice + flushTick | 无 |
| 2 | 迁移 PlayerRuntimeService.notices → EventBus | P1 |
| 3 | 迁移 pendingCombatEffects → EventBus | P1 |
| 4 | 迁移 GM statePush → EventBus | P1 |
| 5 | 新增 panelPatch/jobProgress/stateDelta/feedback | P2 |
| 6 | 客户端消费侧 + 协议集成 | P5 |
| 7 | 指标 + smoke + bench + 清理旧代码 | P2-4 |

## 性能约束

- queue 方法 O(1)，不分配新对象
- flush 总耗时 < 10ms（5000 玩家）
- 内存峰值 < 15MB，每 tick flush 后清零
- PlayerEventQueue 懒创建，下线时 discard
