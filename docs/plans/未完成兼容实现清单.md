## 当前条目

### C001 战斗链路统一分层迁移

- 状态：迁移中，兼容实现仍在生产主线。
- 主域：`packages/server/src/runtime/world/combat`。
- 关联文档：`docs/architecture/ADR-战斗链路统一分层与过渡迁移.md`。
- 当前目标：把玩家普攻、玩家技能、怪物普攻、怪物技能、地块、阵法、容器等战斗入口统一到 `CombatAction / CombatTarget / CombatOutcome` 主链路。

#### 当前兼容点

- `WorldRuntimeCombatCommandService` 仍通过 `WorldRuntimeCombatActionService` 包装旧执行服务，并在回调里调用旧的 `WorldRuntimeBasicAttackService`、`WorldRuntimePlayerSkillDispatchService` 和 `WorldRuntimeBattleEngageService`。
- `WorldRuntimePlayerSkillDispatchService` 仍保留玩家技能 legacy target 形态，并在 action plan 与旧技能结算之间通过 `toLegacyPlayerSkillTargets()` 做互转。
- `WorldRuntimeCombatActionService` 仍接受 legacy target 输入，并用 `source: "legacy_targets"` 标记旧目标来源。
- 部分技能/普攻结算、AOI 表现、玩家通知、审计和 pending cast 恢复仍未完全由统一 action/outcome 主链路独立承接。

#### 不能立即移除的原因

- 玩家技能结算和表现仍依赖旧 `PlayerCombatService` 结果结构。
- 现有生产入口需要继续覆盖怪物、阵法、地块、容器等多类目标，直接删除旧分支会改变战斗行为。
- pending cast 仍以运行态内存为主，Redis/DB 恢复链和 outbox 审计持久化还没有完全接管。

#### 退出条件

- 普攻、玩家技能、怪物技能等生产入口不再把旧服务作为最终结算回调。
- 技能目标在统一 `CombatTarget` 形态内完成解析、合法性判断、结算和结果应用，不再需要 legacy target 往返转换。
- AOI 表现、玩家通知、内部诊断和审计事件都由统一 outcome/event 构建器覆盖。
- pending cast 恢复和审计持久化进入明确真源或 outbox 链路。
- 相关 smoke、bench、protocol/audit 门禁能证明移除兼容实现没有回退战斗行为。
