# 服务端与客户端多线程并行化改造计划

> 解决问题：将 worker pool 从 opt-in 产品功能改为默认执行模型，消除无效预序列化，让 worker 输出真正成为写库输入。

## 约束（不变量）

- 服务端权威不变：worker 不持有真源、不持连接
- 协议字段不变
- 1Hz tick 不变，fan-out 必须在 tick interval 内 join 完成
- 5000 玩家 / 10000 实例 / 8C16G 单服口径不变

## 三类 Worker 形态

| 类型 | 职责 | 并发度 |
|------|------|--------|
| CPU pool (Encoding) | protobuf encode / A* / FOV / fengshui | min(N_cpu-2, 6) |
| Instance pool | monster AI intent proposals | 按 instanceId 哈希分片 |
| Persistence pool | write plan 构造 | 2 |
