## 已处理问题

### F-004 [P1] QQ 加群入口会导航当前游戏页

- 文件：`packages/client/src/main-startup-bindings.ts:372-383,481-494`
- 现象：点击 QQ 加群入口后通过 `window.location.href = ...` 在当前页面跳转外部协议/链接。
- 影响：可能让当前游戏客户端离页，打断 socket、Canvas、输入与弹层上下文；属于外链场景的全量页面打断风险。
- 处理：已改为在用户点击激活窗口内使用 `window.open(..., '_blank', 'noopener,noreferrer')` 唤起 QQ，同时保留群号复制和被拦截提示，不再改写当前页 location。
- 验证：`pnpm verify:client` 通过（客户端构建与 production boundary check 均通过；仅保留 Vite/protobuf eval 与大 chunk 既有警告）。

### F-003 [P1] React 原型脚手架地图字段只显示 `mapId`

- 文件：`packages/client/src/react-ui/app/ReactUiScaffold.tsx:27-45`、`packages/client/src/ui/panel-system/types.ts:207-232`、`packages/client/src/main-runtime-state-source.ts:386-400,647-652`
- 现象：React UI 脚手架的「地图」字段读取 `shellState.runtime.mapId`，面向玩家/测试者时可能显示内部地图 ID。
- 影响：命中「只显示 id 或元数据而没有显示正确中文名字/真实数据」要求，且 React 渐进式 UI 会把旧 DOM 已处理过的中文地图名能力倒退成 ID 展示。
- 处理：已给面板运行态补 `mapName`，首包与地图静态同步从 `mapMeta.name`/进入包名称桥接中文名；React 展示优先使用 `mapName`，`mapId` 仅保留在 title 中辅助排障。
- 验证：`pnpm verify:client` 通过（客户端构建与 production boundary check 均通过；仅保留 Vite/protobuf eval 与大 chunk 既有警告）。


### F-001 [P1] React 原型关闭按钮触发全页刷新打断

- 文件：`packages/client/src/react-ui/app/ReactUiScaffold.tsx:92-100`
- 现象：点击「关闭 React 脚手架」先写入 `__toggleMudReactUi__(false)`，随后执行 `window.location.reload()`。
- 影响：会断开 socket、重建 Canvas/DOM、丢失当前焦点/滚动/弹层/输入上下文，属于明确的前端全量刷新打断。
- 处理：已改为只切换 React UI feature flag，由 `shellStore` 驱动 `ReactUiRoot` 返回 `null`，不再刷新页面。
- 验证：`pnpm verify:client` 通过（客户端构建与 production boundary check 均通过；仅保留 Vite/protobuf eval 与大 chunk 既有警告）。

## 待决定问题

### F-002 [P0] 生产构建版本变化时仍会强制 `location.replace` 刷新

- 文件：`packages/client/src/version-reload.ts:168-172`、`packages/client/src/main-app-runtime-context.ts:60-62`、`packages/client/src/gm.ts:636-640`
- 现象：检测到新 `buildId` 后，客户端提示后延迟执行 `window.location.replace(...)`。
- 影响：部署时会打断在线玩家正在进行的移动、交易、阅读、输入、弹层选择等操作；GM 页面也会被强制刷新。
- 待决定：新版本策略应选择哪种口径：A. 仅提示并由用户手动刷新；B. 玩家空闲/不在关键操作时自动刷新；C. 保留当前强制刷新但加更明确倒计时和可取消。
- 推荐默认：A，避免任何自动全量刷新打断，后续可补服务端兼容窗口和版本过期提示。
