# 兼容层设计规范

## 背景

后续数据结构、配置 schema、内容模板和 AI 生成草稿会持续演进。为了避免生产运行时长期背负旧字段、旧格式和旧语义分支，兼容逻辑不再作为业务代码中的常驻 adapter 或 lazy 升级路径实现。

兼容转换应作为 GM 运维能力：由 GM 快捷指令显式触发，一键把旧数据转换为新真源格式，并提供审计、预览、执行、回读和回滚证据。

## 核心规则

- 数据相关兼容只放在 GM 快捷指令的一键转换能力里，不放在 runtime、shared、content loader、客户端显示或普通业务服务里。
- 新格式上线后，生产代码只接受新真源格式；旧格式进入系统前必须先通过 GM 一键转换。
- 所有转换器统一放在一个兼容转换目录中注册、管理和调用，不能分散成零散脚本或临时函数。
- 每个转换器必须具备幂等设计、dry-run 预览、影响范围统计、执行日志、结果回读和失败恢复说明。
- 转换器只做格式迁移和等价语义映射，不顺手改玩法平衡、生成规则、玩家资产或运营策略。
- 涉及玩家资产、功法、装备、地图、市场、邮件等真源数据时，必须记录 GM 操作人、转换版本、目标范围、转换前后摘要和验证结果。

## 目录边界

计划中的统一目录建议为：

```text
packages/server/src/gm/compat-conversions/
```

职责划分：

- `registry`：登记所有一键转换项，提供 id、名称、说明、风险等级、输入参数 schema 和执行入口。
- `conversions/`：按领域放置具体转换器，例如 `technique/`、`content/`、`map/`、`player/`、`market/`。
- `runner`：统一执行 dry-run、apply、回读验证、错误收集和审计写入。
- `types`：定义转换输入、预览摘要、执行结果、回读报告和审计结构。
- `gm endpoint`：只暴露注册表、预览和执行接口，不承载领域转换细节。
- `gm ui`：在 GM 快捷指令中展示一键转换按钮、影响范围、预览差异和执行结果。

## 转换器标准接口

每个转换器至少提供：

- `id`：稳定唯一标识，例如 `technique_arts_strength_v1_to_v2`。
- `title`：GM 展示名。
- `description`：说明转换目标和影响范围。
- `riskLevel`：`low | medium | high`。
- `scopeSchema`：目标范围参数，例如全局、指定玩家、指定配置表、指定模板 id。
- `dryRun(scope)`：只读扫描并返回影响数量、样例差异和阻断原因。
- `apply(scope)`：执行转换，必须可重复运行且不会二次污染已转换数据。
- `verify(scope)`：回读新格式并证明可通过当前 schema 和加载链路。

## GM 快捷指令流程

1. GM 打开兼容转换中心或对应快捷按钮。
2. 选择转换项和作用范围。
3. 先执行 dry-run，展示影响数量、样例、风险和预计写入对象。
4. GM 确认后执行 apply。
5. 服务端写入审计，返回成功、跳过、失败和需要人工处理的明细。
6. 自动执行 verify，验证新格式能被当前 schema、内容加载或玩家回读链路接受。
7. GM 可导出本次转换报告，用于发布记录和回滚依据。

## 禁止事项

- 禁止在运行时加载器里长期保留 `oldField ?? newField` 这类旧字段兼容。
- 禁止在 tick、战斗、属性结算、地图实例、市场成交等热路径做数据格式升级。
- 禁止客户端为旧格式补语义；客户端只展示服务端确认后的新格式数据。
- 禁止把一次性 SQL、临时 tsx、散落 helper 当作长期兼容层。
- 禁止转换器绕过 schema 校验直接写入生产真源。

## 首批落地点

- AI 术法强度草稿字段升级：例如把 `target.range/radius/width/height` 迁移为更明确的权重字段。
- 内容模板旧数值字段清理：把 `valueStats/attrs/mainCombatStatsPercent` 等迁移到当前量化字段。
- 地图旧 `TileType` 数据清理：把旧字符矩阵迁移为多层 rows 真源。
- 玩家存量功法/装备旧字段清理：只通过 GM 转换执行，不在正常加载链路新增兼容分支。

## 验证口径

- dry-run 必须能在不写库、不改文件的情况下完整统计影响面。
- apply 必须有幂等测试：重复执行不改变已转换结果。
- verify 必须走当前正式 schema 或内容加载入口，而不是转换器内部自证。
- 高风险转换必须提供小范围样本执行记录，再允许全量执行。
