# 对象克隆全面扫描计划

> 目标：找出所有"本应共享引用、却按玩家/实体/tick 复制"的对象克隆点，统一改为引用公有数据。
>
> 证据来源：正式服 heap snapshot（2440 万节点、1.53 GB 活对象、1000 万 Object）。
>
> 原则：**模板数据只存一份，运行时只存差异；能引用就引用，能公有就公有。**

### 改造优先级（按 heap 收益估算）

1. `cloneTechniqueEntry` + `toTechniqueUpdateEntry` + `hydrateTechniqueState`（功法 skills / layers 引用化）— 估计削减 15–30 万 skill / layer 对象，直接对应 heap snapshot 的"功法每玩家独立副本"问题。
2. `cloneTemporaryBuff`（4 份）+ `applyTemporaryBuff` upsert / `applyTemporaryBuffToMonster` push — 估计削减 5–10 万 buff 对象 + 大幅降低 tick 级 heap churn，恢复 `cloneVisibleBuffProjection` WeakMap 命中。
3. quest 运行态瘦身（`cloneQuestState` 全链 + `player.quests.quests` 结构）— 估计削减 5–10 万 quest 字段 / 字符串副本，对应 heap 中 quest id 出现 1–5 万次。
4. `runtimeBonuses` 派生源不入栈 + `cloneRealmState` 三份合并 — 估计削减 2–5 万 bonus / realm 副本，正好对应 heap 中 `realm_state` / `technique_aggregate` 标签 3–5 万次。
5. 妖兽 spawn 不再 `cloneSkill` / `cloneInitialBuff`，模板共享 — 估计削减 15–25 万 skill / buff 副本。
6. NPC 实例不再 `cloneShopItems / cloneQuests` — 估计削减 10–20 万 shop / quest item 副本。

### 本轮总结

- **任务系统**：`cloneQuestState` 把整条任务（含 30+ 模板字段）作为运行态副本反复拷贝；持久化结构进一步把这些模板字段沉到 DB。runtime 应只剩 `{ id, status, progress, requiredItemId?, ... }` 等可变 / 索引字段。
- **buff 系统**：4 份重复 `cloneTemporaryBuff`，每条 buff 携带 `name / desc / baseDesc / category / sourceSkillName / ...` 等模板字段；`applyTemporaryBuff` 每次 upsert 都重建 attrs / stats / qiProjection 子对象，是 tick 级 heap churn 主犯之一，并破坏投影 WeakMap 命中。
- **技能 / 功法**：`buildTechniqueRuntimeStateFromTemplate` 已正确按引用共享 skills / layers，但 `toTechniqueUpdateEntry` / `cloneTechniqueEntry` / `hydrateTechniqueState` 在三个下游又把它们 deep clone 回去，**冲掉了仓库层的设计意图**；妖兽 spawn 同病同源。
- **realm_state / technique_aggregate**：3–5 万出现量来自 `runtimeBonuses` 中的 derived 条目对象——它们在结算时被 `isDerivedRuntimeBonusSource` 显式过滤掉，本就不该入栈。
- **NPC 模板**：实例化时 `shopItems` / `quests` 全量浅拷贝，纯静态数据，应直接共享模板引用。
- **共同根因**：项目里几乎所有"状态"容器都装着"模板字段 + 运行字段"的混合结构，缺少"瘦运行态 + 模板按 id 引用"的统一约定。建议在 `packages/shared/src/` 新增一组 `*RuntimeSlim` 协议，与现有 `*View` 协议平行，把运行态压成最小可变集合，由仓库层负责模板侧补全；`QuestRuntimeStateView`（quest-types.ts:227 `{ id, status, progress }`）已经是这个方向的示例，应成为运行态、不只是同步态。
