# 小号风险检测系统设计

## 1. 文档目标

这份文档不是泛泛讨论“怎么反多开”，而是结合《道劫余生》当前真实玩法、现有数据和 GM 链路，先定义一套可落地的小号风险检测系统。

目标：

- 明确本游戏里“小号”主要靠什么方式获利
- 明确当前代码里已经能直接利用哪些信号
- 明确哪些关键信号还没有采集，后续要补什么表和日志
- 先形成“风险识别 + 限权 + GM 复核”的闭环，再决定哪些场景要自动封禁

非目标：

- 不依赖单一 IP 或单一弱信号直接封号
- 不追求一次性封死所有多号
- 不把客户端本地判断当最终结论

相关背景文档可参考 [docs/反多开方案.md](./反多开方案.md)。

## 2. 先看本游戏里，小号是怎么获利的

当前项目不是传统 MMO 全功能形态，先把边界说清楚。

### 2.1 当前已存在的重要系统

- 一个账号对应一个角色，`players.userId` 唯一，天然是一号一角
- 已有同账号顶号，不能同账号双开
- 已有 GM 快捷封号 / 解封
- 已有坊市，且有成交历史，能看到 `buyerId` / `sellerId`
- 已有系统邮件和 GM 发邮件
- 已有兑换码系统，能看到 `usedByPlayerId` / `usedByRoleName`
- 已有在线、离线挂机、离线三态
- GM 已能识别角色当前行为：战斗 / 修炼 / 炼丹 / 强化 / 采集

### 2.2 当前还不存在或不构成主风险的系统

- 目前没有玩家互发邮件
- 目前没有点对点交易
- 目前没有公会仓库、师徒奖励、邀请奖励这类典型输血通道

所以在本游戏里，小号的主要获利路径不是“面对面交易”，而是下面几类：

1. 批量开号后刷欢迎资源、兑换码、通用奖励
2. 小号集中挂机修炼、炼丹、采集、强化，再通过坊市把收益输给主号
3. 小号充当陪跑、探路、占位、代练辅助号
4. 被封后快速重开号继续接回原来的行为链

- 新号是否集中领取欢迎资源后迅速进入输血链

## 4. 当前最缺的信号

下面这些信号对识别同人多号很关键，但当前项目基本还没有正式采集。

### 4.1 登录来源审计

当前 `auth.controller.ts` / `auth.service.ts` 的注册、登录、刷新链路还没有正式记录：

- 注册 IP
- 登录 IP
- 登录 UA
- 设备标识 `deviceId`
- 登录失败次数
- 登录时间序列

这是当前最大的缺口。没有这层，只能做“行为和经济风控”，很难做“同主体识别”。

### 4.2 设备主体标识

当前没有服务端签发并持久化的 `deviceId`。

没有它就会出现两个问题：

- 同一人清缓存、换浏览器后完全断线
- GM 很难从“账号关系”快速还原出“设备关系”

### 4.3 风险事件真源

当前没有正式的风险事件表，也没有风险画像表。

所以现在即使发现“这个号很像小号”，结论也难以沉淀，重启后、换班后、二次复核时都不方便追踪。

## 5. 本游戏里，怎么判断一个号像“小号”

不建议把“小号”理解成“同 IP 的号”。在本游戏里，更合理的定义是：

> 一个账号如果主要作用不是独立游玩，而是为另一个账号或一个账号簇提供额外收益、额外效率或额外信息优势，就应进入小号风险识别范围。

具体判断拆成六个维度。

### 5.1 维度 A：主体关联

这是判断“是不是同一操作者”的维度。

强度从高到低：

1. 同 `deviceId`
2. 同注册 IP + 同登录 IP 段 + 时间高度聚集
3. 同 UA + 同时段上线 + 长期稳定共现
4. 曾被同一已封主体关联过

当前能立即做的只有：

- 账号创建时间聚集
- 上线/下线时间聚集
- 封禁关系链回溯

后续补齐登录审计后，这一维才能真正成型。

### 5.2 维度 B：成长轨迹异常

这是判断“像不像一个正常独立玩家”的维度。

本游戏里，正常独立玩家通常会出现：

- 地图推进
- 任务推进
- 战斗与修炼交替
- 背包、装备、功法有自然成长
- 行为有波动，不会永远只干一件事

高风险小号常见轨迹：

- 刚注册不久就长时间离线挂机
- 角色主要只有修炼 / 炼丹 / 采集 / 强化，几乎没有正常探索和战斗
- 在线总时长很低，但经济产出异常集中
- 很快把资源变现后自身不继续成长

这类规则特别适合本游戏，因为当前就有：

- 在线态
- 行为标签
- 炼丹任务
- 强化任务
- 采集状态
- 击杀和成长数据

### 5.3 维度 C：协同行为异常

这是判断“多个号是不是在互相配合”的维度。

本游戏里重点看：

- 多个账号长期同时上线或同时离线挂机
- 多个账号长期在同一地图活动
- 新号总是围绕同一个主号的地图和时间段出现
- 一组号行为类型高度固定，例如都只挂机修炼和炼丹

后续如果补位置采样，还可以看：

- 是否长期同图同路径
- 是否长期固定跟随
- 是否长期在同一资源点群活动

### 5.4 维度 D：经济输血异常

这是本游戏最关键的一维。

由于当前没有直接交易，小号输血大概率走坊市，所以重点规则要围绕 `market_trade_history` 建。

高风险特征：

- 新号反复卖给同一个老号
- 交易方向长期单向
- 单位时间内只有固定一两个交易对手
- 成交价格明显偏离该物品近期中位价
- 小号几乎不自用成长，产出后马上经由坊市转出

在本游戏里，以下组合尤其可疑：

1. 账号年龄很小
2. 行为长期是采集 / 炼丹 / 强化
3. 成交对象高度固定
4. 净资产持续从小号流向主号

满足 1 + 2 + 3 + 4 时，基本就是强风险样本。

### 5.5 维度 E：奖励套取异常

本游戏当前已有欢迎邮件和兑换码系统，所以要重点防“批量领一次性收益”。

高风险特征：

- 一批新号在很短时间内集中使用同一兑换码组
- 同一主体关联簇内大量账号都只完成最短领奖路径
- 领取欢迎资源后很快进入坊市成交链
- 账号后续行为极少，但奖励领取完整

这类规则适合做“限收益”而不是立即封禁。

### 5.6 维度 F：处置后复发

如果一个主体簇已经有封禁记录，后续新号又迅速复现原来模式，应提高基准风险。

重点看：

- 同一主体链路曾被封
- 解封后短期再次出现同样的经济链和行为链
- 被封主号周围迅速长出一批新号

## 6. 风险评分模型建议

建议不要做“一条规则命中就封”。更稳妥的是做累计评分。

### 6.1 分层

- 弱信号：只能加少量分，不可单独触发封禁
- 中信号：可触发观察、限权、人工复核
- 强信号：可触发强制审核，和其他信号叠加后可进入封禁流程

### 6.2 建议分值

#### A. 主体关联，0 到 40 分

- 同设备标识：40
- 同注册 IP 且 24 小时内批量开号：20
- 同登录 IP 段且长期共现：10
- 与已封主体存在强关联边：20

#### B. 成长轨迹异常，0 到 20 分

- 新号 48 小时内主要行为长期只有修炼 / 炼丹 / 采集：8
- 在线总时长低但持续离线挂机：6
- 成长停滞且资源持续外流：10

#### C. 协同行为异常，0 到 20 分

- 多账号时段高度同步：8
- 多账号长期同地图共现：6
- 多账号行为模板高度一致：6

#### D. 经济输血异常，0 到 35 分

- 新号与固定对象重复成交：10
- 净价值长期单向流出：10
- 价格明显偏离参考价：8
- 成交后小号无自然成长：7

#### E. 奖励套取异常，0 到 25 分

- 同簇短时间集中使用同组兑换码：12
- 只完成领奖最短路径：6
- 奖励到账后很快进入成交链：7

#### F. 历史复发，0 到 15 分

- 关联主体已有封禁史：10
- 关联主体已有临时限制史：5

### 6.3 风险状态建议

- `0-24`：正常
- `25-44`：观察
- `45-64`：高风险，限制高收益行为
- `65-84`：进入 GM 必复核队列
- `85+`：强风险，可临时冻结高收益行为并等待最终处置

默认规则：

- 自动系统优先做限制，不直接永久封号
- 永久封号至少要有 1 个强信号或多个中信号链式证据

## 7. 本游戏第一版最值得先落的规则

如果只做第一版，不要贪多。优先做下面四类。

### 7.1 新号坊市输血规则

规则目标：

- 抓“小号生产资源，再通过坊市卖给主号”

判定建议：

- 账号年龄小于 3 天
- 最近 24 小时主要行为是炼丹 / 采集 / 强化 / 修炼
- 最近 24 小时坊市成交对象高度集中
- 卖出次数、卖出总额显著高于买入
- 成交后该号自身成长弱

动作建议：

- 先加高风险分
- 达阈值后限制坊市挂单 / 吃单
- 推给 GM 复核

### 7.2 批量兑换码规则

规则目标：

- 抓“批量开号领兑换码”

判定建议：

- 同一时间窗口内，一批低龄账号集中使用同一兑换码组
- 这些账号上线时段高度重叠
- 这些账号随后又和同一批目标账号发生坊市成交

动作建议：

- 先冻结该簇兑换码收益或后续兑换权限
- 推给 GM 查看关联图

### 7.3 离线挂机生产号规则

规则目标：

- 抓“人不怎么玩，只负责挂收益”的生产型小号

判定建议：

- 长时间为离线挂机或低交互在线
- 行为长期单一，集中在修炼 / 炼丹 / 采集 / 强化
- 击杀、地图推进、任务推进明显不足
- 产出经坊市流向固定少数账号

动作建议：

- 标记为生产型高风险号
- 提高经济维度权重

### 7.4 封禁后复开号规则

规则目标：

- 抓“被封主体换皮回流”

判定建议：

- 某主体簇刚被封禁
- 随后短时间内出现一批新号
- 新号快速复现原本的挂机、收益和成交对象结构

动作建议：

- 直接进入高优先级复核
- 必要时临时禁止坊市和兑换码

## 8. 当前系统下，哪些判断能马上做，哪些还不能

### 8.1 马上能做

- 基于账号年龄的风险分
- 基于在线 / 离线挂机 / 离线 的风险分
- 基于战斗 / 修炼 / 炼丹 / 强化 / 采集 的行为聚类
- 基于坊市成交历史的输血检测
- 基于兑换码使用记录的批量套取检测
- 基于已封账号关系链的复发检测

### 8.2 还不能稳做

- 同设备识别
- 同人多号强归因
- 注册入口批量脚本识别
- 同网络主体高置信聚类
- 精细的同步路径跟随检测

原因不是“不会做”，而是当前缺少登录审计、设备标识和行为采样真源。

## 10. 系统输出不要直接等于封禁

建议把系统输出拆成 4 类：

1. 风险分
2. 风险标签
3. 自动限制动作
4. GM 复核建议

### 10.1 自动限制动作建议

优先限制高收益行为：

- 禁止或限额坊市
- 暂停兑换码使用
- 暂停领取特定活动奖励
- 暂停高价值物品成交

不建议第一版直接自动永久封号。

### 10.2 GM 复核时应看到什么

GM 面板后续至少要能看到：

- 当前总分
- 命中的规则列表
- 账号关联图
- 最近 7 天坊市净流向
- 兑换码使用记录
- 在线时段热力
- 行为分布：战斗 / 修炼 / 炼丹 / 强化 / 采集
- 该账号为什么被判异常，而不是只给一个总分

## 11. 建议实施顺序

### P0：先用现有数据做可落地版本

只做：

- 风险分框架
- 坊市输血检测
- 兑换码批量套取检测
- 行为型生产号检测
- GM 风险标签展示

这一步已经能抓到当前最现实的一批小号。

### P1：补登录审计和设备主体

补：

- `deviceId`
- 登录 / 注册 / 刷新 / WS 建连审计
- 同主体聚类

这一步完成后，才能更稳定识别“同人多号”。

### P2：补关系图和自动限权

补：

- 账号关系图
- 风险事件流水
- 自动限坊市 / 限兑换码 / 限奖励

### P3：补复核面板和历史追踪

补：

- GM 风控详情面板
- 风险变化时间线
- 证据快照
- 申诉与解封记录

## 12. 结论

在《道劫余生》当前版本里，小号识别的核心不是“找出谁同 IP”，而是识别三件事：

1. 这个号是不是像一个独立玩家在正常成长
2. 这个号是不是在为另一个号提供收益或效率
3. 这一串账号之间是否形成了可解释的主体关联和资源流向

就当前项目而言，最先该做的不是“登录一刀切”，而是：

- 先用现有数据抓坊市输血
- 抓批量兑换码
- 抓离线挂机生产号
- 再补登录来源和设备主体识别

这样做的收益最高，误伤也最低。
