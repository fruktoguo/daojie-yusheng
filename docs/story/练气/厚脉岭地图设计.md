# 厚脉岭地图设计

## 1. 总体定位

- 地图名：`厚脉岭`
- 五行属性：土
- 关键词：掘山、承压、镇脉、开路、立基前夜

厚脉岭不该是一张普通山路图。

这张图最该有的感觉，不是“爬坡”，而是整座山脉本身把路埋死了。玩家进来以后，不是沿着现成山道一路走，而是一路凿石、掘矿、开脉，硬把通路从山体里挖出来。

所以厚脉岭这次的核心方向直接定成：

- 整张图被大量石头、矿脉和断层堵满
- 主路不是天然露出来的，而是被埋在山体下面
- 玩家推进的感觉应当是“边挖边进”，不是“边跑边刷”
- 越往深处，石层越厚，矿越密，路越窄，越像在给自己挖一条进山的活路

## 2. 核心设计判断

这张图和前两张的空间语言必须明显不同：

- 寒汐泽是 `旋进去`
- 赤陨庭是 `坠下去`
- 厚脉岭应该是 `掘进去`

也就是说：

- 不是绕
- 不是冲
- 也不是一眼能看到头

而是每推进一段，就要再掘开下一段。

土行这张图真正要立住的是三个字：

- `压`
整张图都像压在玩家身上

- `堵`
大量石头和矿脉本身就是空间结构，不只是装饰

- `开`
玩家不是顺着山走，是在山里开线

## 3. 地块语言

这张图尽量不加新地块，直接靠现有石系地块做。

主要用这套：

- `o`：大块岩石、堵路脉石、需要挖开的主石层
- `L`：灵石矿脉，兼顾资源与堵路作用
- `铁`：玄铁矿脉，作为更硬、更深的堵路点
- `崖`：整块断层、高壁、不可开的山体边界
- `^`：隆起的高坪、石坡、抬高地势
- `,`：裸露土面、碎石滩、已开出的浅层地面
- `;`：沉脉重地、压脚泥石带
- `=`：已经被掘开的稳固主路
- `:`：细窄侧路、柱间通道、补线小路

这里最重要的是：

- `o / L / 铁` 不是顺手摆几个采集点
- 它们就是整张图的“门”和“墙”

玩家看到的不是一条路边摆矿，而是：

- 路就在矿和石头后面
- 要把它们挖掉，路才真正露出来

## 4. 空间结构

厚脉岭仍然按六段来做，但六段之间不是自然衔接，而是“挖开一层，才看到下一层”。

### 1. 回渡石坪

- 最外层入口
- 这里只有少量裸露地面
- 玩家先看到被石层压满的山脚，不会直接看到深处主路
- 回城口就放在这里

这段的作用不是战斗，而是先让玩家理解：

- 这张图不是跑图
- 这张图是开图

### 2. 山脚采脉区

- 第一层挖掘区
- 大量 `o` 和少量 `L` 混合堵路
- 挖开后会形成比较宽的推进带
- 普通怪先放在已开出的小空坪和石缝边

这里应该有“刚进山，还能喘口气”的感觉，但仍然明确告诉玩家：

- 以后路只会更堵，不会更松

### 3. 坠石狭道

- 路线第一次明显收窄
- 两边大块 `崖` 卡死，中间夹着 `o / L / 铁`
- 玩家不是清出一个大平面，而是慢慢掘出一条狭长通道

这一段重点是：

- 有明显卡位感
- 有“刚挖开就只能过一人”的狭压感
- 怪物会在狭道尽头或转角后的空格里等人

### 4. 负碑缓坡

- 不再是单纯狭道，而是边挖边上坡
- 路线上会出现大块孤石、断碑、矿脉混堵
- 挖出来的路线不是一条直线，而是一段一段的台阶式推进

这一段的作用是让玩家感觉：

- 山体开始真正变重
- 不是“快到顶了”
- 而是“越挖越觉得整座山压下来了”

### 5. 石柱高坪

- 中后段主战区
- 挖开之后会露出一块块被石柱切碎的高坪空地
- 精英怪适合放在这些高坪之间的开阔口

这里不能太空。

正确感觉应该是：

- 虽然挖开后视野会稍微放大
- 但到处还是石柱、矿脉、断层
- 玩家站在高坪上，只会觉得自己进了更重的一层，不会觉得终于通了

### 6. 镇岳石府前庭

- 最深处最终压力区
- 不是宫殿式大广场
- 仍然是大块土石和残门结构，只是地势更整、更硬、更像有人在这里强行镇过山脉

最深处应该给人的感觉是：

- 这已经不是自然山体
- 而是有人试图把整条地脉压成一座门

归藏口就放在这里最深的一层。

## 5. 怪物刷新约束

这张图因为大量路都靠挖开，所以怪物生成点必须单独写死一条规则：

- 怪物刷新点附近必须保留若干单个空格
- 至少保证怪物脚下和周围有可落位的已开地面
- 不能把刷新点直接埋在整片石头或矿脉里
- 这些空格看起来应该像“石层间自然露出来的小空坪”或“已经挖穿的小口袋”

也就是说：

- 路可以被埋
- 资源可以堵路
- 但怪点周围必须有实际可生成、可站立、可开战的小空间

这条约束是地图结构的一部分，不是后面补怪时再临时修。

## 6. 资源结构

土行这张图的资源不该只是顺路捡。

正确做法是：

- `o` 负责形成最常见的堵路石层
- `L` 负责让玩家觉得“挖开不只是开路，也是在采脉”
- `铁` 负责做深层、更硬、更关键的堵点

所以资源本身承担两层作用：

1. 路网结构
2. 采集收益

玩家体验上要形成一句话：

- “我不是在路边挖矿，我是在矿里挖路。”

## 7. 本轮落地口径

如果下一轮开始做地形，这张图的正确落地顺序应该是：

1. 先做整张图的大结构
- 先把山体、断层和六段位置定下来

2. 再塞满石头和矿
- 让路先消失

3. 最后只在必要位置露出少量初始可走点
- 入口区
- 怪物出生口袋
- 若干过渡连接点

也就是：

- 不是先画路，再往路边补矿
- 而是先画山，再从山里抠路

这才是厚脉岭该有的土行味道。

## 8. 当前大图落地结构

这一版大图直接按一整张 `60x60` 去落，不再拆散图。

整体推进方向定成：

- 从右侧中段入山
- 一路斜着往左上掘
- 最深处的镇岳石府前庭压在左上内层

这样做有三个好处：

- 入口和终点不会一眼对穿
- 推进线天然带出“越掘越深”的斜压感
- 后面补怪、补灵气和补地标时，六段空间比较好分层

当前六段在大图里的位置这样定：

- `回渡石坪`
放在右侧中段，和栖真渡传送口贴在一起，只留一小块能站人的石坪

- `山脚采脉区`
放在右下偏中的第一大片矿层，主要由 `o / L` 堵住，只散露几个浅口袋

- `坠石狭道`
从右中往左上收成斜切窄道，两边用 `崖` 卡死，中间塞石层和矿块

- `负碑缓坡`
放在中部偏上的斜阶地带，用 `^` 和重矿层做出一段一段往上顶的感觉

- `石柱高坪`
放在左中区域，挖开后会是多个被石柱切开的碎坪，不做成完整大平原

- `镇岳石府前庭`
放在左上最深处，结构更整、更硬，归藏口压在这里

这一版地形还有两个硬约束已经一起落进去：

- 整图默认先被石头、灵石矿和玄铁矿塞满，开放地面只保留入口、小口袋和少量过渡缝
- 未来怪物刷新区附近都预留了单格或小块空地，不会出现怪点直接埋死在整片矿层里的情况

一句话概括当前成图逻辑：

- 先给玩家一块能下铲的地，再把后面的路全埋进山里。
