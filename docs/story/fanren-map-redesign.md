# 凡人篇地图重设计

状态：草案 v1

适用范围：

- 凡人篇主线地图
- 当前 `spawn / wildlands / bamboo_forest / black_iron_mine / ancient_ruins / beast_valley / spirit_ridge`
- 重点先解决 `云来镇` 作为主据点过于空、过于小、缺少功能区的问题

## 1. 为什么要重做

当前地图能支撑基础刷怪和传送，但还不足以支撑“凡人篇”的完整体验。

现状问题：

- `云来镇` 只有少量房子和 NPC 点位，缺少主城应有的分区、入口、店铺感和秩序感
- 镇内没有围墙、门楼、哨位、仓场这类“边镇”应有结构，世界观落点偏弱
- 地图虽然已经不是 30 格，但 `spawn` 目前仍像一张功能测试图，不像活着的镇子
- 主线地图之间更像战斗关卡切换，缺少“路怎么走、货怎么运、镇为什么还能活”的空间叙事
- 缺少能让玩家记住位置的地标，例如脚店、药铺、井口、仓栈、矿路口、祭台、残碑庭
- 没有“可探索但不必强功能化”的点位，例如补给箱、废弃屋、路边车、封存仓、旧祠

凡人篇如果要成立，地图必须先像一个“还在勉强运转的人间地带”，再像一条升级路线。

## 2. 当前实现约束

本次设计必须服从当前玩法能力，而不是假设未来一定会有复杂脚本系统。

已确认的现实约束：

- 地图为格子地图，按 JSON 静态配置加载
- 服务端 1Hz tick，长距离移动成本高，不适合做过大空地图
- 当前任务系统稳妥支持：
  - `kill`
  - `learn_technique`
  - `realm_progress`
  - `realm_stage`
  - 击杀后提交指定物品
- 当前不应把主线建立在护送、守城、潜入、复杂机关联动这类强脚本能力上
- 店铺建筑当前应优先作为“功能 NPC 所在地点”来设计，不要先假定完整商业系统

因此，地图重做目标不是做“看起来大”，而是做“有区块逻辑、有路线逻辑、有玩法承载”。

## 2.1 可正式依赖的新系统

既然地面物品、容器、时间和光照系统正在实现，凡人篇地图设计可以把它们视为正式依赖，而不是纯预留能力。

### A. 地面物品

地图设计可直接依赖以下表现：

- 怪物掉落物落在地面
- 固定场景物资落在地面
- 某些任务物通过地面拾取完成最后一步
- 路边车、废屋、仓场、井坊可出现“散落痕迹”

这会直接提升空间叙事能力，因为“这里刚发生过什么”不必只靠对白说明。

### B. 容器系统

地图设计可直接依赖以下对象：

- 仓箱
- 木柜
- 封存箱
- 废弃货箱
- 供物箱

容器的叙事价值大于数值价值。

在凡人篇里，容器应优先服务：

- 主城可信度
- 探索感
- 任务交付和搜寻感
- 人间生活痕迹

### C. 时间系统

地图设计可正式区分：

- 清晨
- 白日
- 黄昏
- 夜间

这意味着凡人篇可以把“宵禁”“巡夜”“夜路更坏”“旧井夜里不对劲”写成真实空间体验，而不是单纯文案口径。

### D. 光照系统

地图设计可正式区分：

- 灯火覆盖区
- 弱光区
- 黑暗区
- 特殊冷光或异光区

这会直接影响主城与野外的氛围分层：

- 南市要亮
- 门楼和巡道要有稳定火把
- 北井旧坊要偏暗
- 兽谷和灵岭要做出不同的危险感

## 2.2 因新系统带来的设计升级

在新增依赖成立后，凡人篇地图不应再只满足“刷怪 + 交任务”。

每张地图还应额外承担：

- 地面物资分布逻辑
- 固定容器分布逻辑
- 昼夜状态差异
- 光照覆盖层次

这四点会让“边镇还活着”和“镇外一到夜里就不一样”真正落到空间里。

## 3. 总体设计原则

### 3.1 地图尺寸原则

考虑 1Hz 移动频率，凡人篇地图不宜无限放大。

建议尺寸区间：

- 主城地图：`56x56` 到 `64x64`
- 普通野外地图：`56x56` 到 `72x72`
- 高压副区地图：`48x48` 到 `56x56`
- 不建议超过 `72x72`

理由：

- 太小会像测试场
- 太大则会导致跑图时间过长，拖慢节奏
- 地图应该通过“区块 + 岔路 + 地标”拉开体验，而不是靠纯面积灌水

### 3.2 每张地图至少要有 4 类信息

每张主线地图都应同时具备：

- 主路径：玩家第一次进入时最容易走通的路线
- 危险区：刷怪与材料获取区域
- 地标区：记忆点与剧情落点
- 转场区：通往下一张图的明确出口

### 3.3 主城地图必须承担据点功能

`云来镇` 不是出生点，而是凡人篇的“反复返回据点”。

它至少要承担：

- 任务领取和交付
- 基础功能 NPC 分布
- 区域认知锚点
- 人间层剧情承接
- 多条外线入口组织

### 3.4 空间本身要表达剧情

空间表达优先于长对白。

例如：

- 有围墙，说明镇子长期有外患
- 门楼狭窄，说明进镇资格严格
- 南市拥挤，说明流民和脚行多
- 仓场靠近门口，说明镇子命脉是运输
- 旧井被围起，说明异常早就是本地共识
- 通往青竹林的是运材路，通往荒野的是粮路

## 4. 凡人篇地图总结构

建议把凡人篇主线地图组织成以下结构：

| 地图 | 建议尺寸 | 剧情定位 | 玩法定位 |
| --- | --- | --- | --- |
| `spawn` 云来镇 | `64x64` | 主城据点，凡人篇中枢 | 接任务、交任务、换线、建立世界认知 |
| `wildlands` 荒野 | `64x64` | 粮路与匪患区 | 早期刷怪、材料、跑外线 |
| `bamboo_forest` 青竹林 | `56x56` | 由武入玄的第一张野外图 | 中前期练功、双出口分流 |
| `black_iron_mine` 玄铁矿洞 | `48x48` | 旧矿火与通脉材料区 | 中期材料、高压近战区 |
| `ancient_ruins` 断碑遗迹 | `48x48` | 旧阵纹与历史信息区 | 中期精英怪、任务物回收 |
| `beast_valley` 噬魂兽谷 | `56x56` | 裂息外溢区 | 中后期高压战斗区 |
| `spirit_ridge` 灵脊岭 | `56x56` | 凡武终点、叩门前夜 | 凡人篇终章区域 |

推荐主线空间顺序：

- `云来镇 -> 荒野 -> 青竹林 -> 玄铁矿洞 / 断碑遗迹 -> 噬魂兽谷 -> 灵脊岭`

推荐逻辑说明：

- 先让玩家明白“镇子靠外路活着”
- 再让玩家明白“林路不只是山路，而是旧线入口”
- 再进入矿洞和遗迹，得知旧镇体系
- 再去兽谷承受压力
- 最后由灵岭收束凡人篇

`天穹残宫` 不应再算凡人篇地图，应作为修仙篇开场。

## 5. 云来镇重设计

详细落图蓝本见：

- [云来镇详细格局稿](./yunlai-town-layout.md)

## 5.1 重做目标

新的 `云来镇` 必须让玩家一眼看出：

- 这是个边镇
- 这是个还在运转的边镇
- 这是个表面有秩序、实则压力很大的边镇

它不能再像“几间房子 + 几个 NPC 点”。

## 5.2 建议尺寸与总体结构

建议尺寸：`64x64`

结构建议：

- 外圈有完整围墙
- 至少两座主门：
  - 南门：通往荒野粮路
  - 东门：通往青竹林运材路
- 一座次门或封闭旧门：
  - 北侧旧关道，可作为后续灵岭线的镇内叙事锚点
- 镇内至少分成 6 个功能区

推荐分区：

1. 南门外手区
2. 南市脚行区
3. 镇中仓场区
4. 东门工匠区
5. 北井旧坊区
6. 西侧棚户与临时落脚区

## 5.3 分区设计

### A. 南门外手区

定位：

- 玩家初入镇后的第一活动区
- 外乡人、脚夫、短工、临时摊贩集中地

应有内容：

- 门楼
- 守门哨位
- 公示木牌
- 外来人登记点
- 可见但简陋的棚屋和马棚

对应叙事：

- 云来镇不排斥外人，但只接纳“有用的外人”
- 玩家第一次进镇、第一次出镇都应经过这里

对应玩法：

- 最早期主线 NPC
- 初始任务交接点
- 返回镇时最常见的进门路径

### B. 南市脚行区

定位：

- 凡人篇最重要的人间锚点
- 玩家日常最常回来的商业区

应有建筑：

- 脚店
- 小食铺
- 杂货摊
- 药铺
- 账房柜台

NPC 建议：

- 沈七娘
- 兰药师
- 跑腿差使 NPC
- 一名只负责讲市井口风的闲话 NPC

对应叙事：

- 玩家要先有落脚处，凡人篇才有“回镇”的意义
- 这里要比其他区域更挤、更杂、更亮

对应玩法：

- 支线交接集中区
- 消耗品与补给概念锚点
- 玩家每次出城前的整备区

### C. 镇中仓场区

定位：

- 镇子真正的命脉区域
- 用空间表达“云来镇是靠路和货活着”

应有内容：

- 粮仓
- 木料堆
- 运货车
- 货栈
- 巡逻通道

NPC 建议：

- 仓头
- 车把式
- 负责发“保路”任务的差役

对应叙事：

- 这里不需要太多情绪对白，空间本身就要让玩家明白：路一断，镇子就死

对应玩法：

- 荒野线、青竹林线的主任务切换中枢
- 镇内公告板或章节提示点适合放在这里

### D. 东门工匠区

定位：

- 兵器、矿料、木工和出东路的连接区

应有建筑：

- 铁匠铺
- 炉棚
- 木作间
- 运材车停靠位

NPC 建议：

- 韩铁匠
- 一名和矿路、竹林路线绑定的工匠学徒

对应叙事：

- 通往青竹林的路是运材路，不是游客步道
- 玩家去青竹林、矿洞前，应先在这里形成空间记忆

对应玩法：

- 武器、防具、材料相关任务集中区
- 竹林线和矿洞线的前置引导区

### E. 北井旧坊区

定位：

- 云来镇最关键的异常叙事区
- 凡人篇的暗线源头之一

应有内容：

- 被围起的旧井
- 旧祠
- 半废弃民居
- 巡夜岗哨

NPC 建议：

- 守夜人
- 老住户
- 一个“不讲真话但会反复提醒不要靠近井口”的 NPC

对应叙事：

- 白天这里安静
- 夜里这里令人不舒服
- 这里不该一开始就大量发任务，而应先作为气氛地标

对应玩法：

- 前期主要是剧情地标
- 中后期可以接入遭遇任务或章节对话

### F. 西侧棚户区

定位：

- 镇里最靠近“活着就行”的区域
- 用来体现边镇真实人口结构

应有内容：

- 棚屋
- 破车
- 临时柴棚
- 低价值补给点

对应叙事：

- 不是所有进镇的人都住得进脚店
- 镇内的秩序也分层

对应玩法：

- 适合放少量早期杂务任务
- 也适合放第一个“固定补给箱”或“废弃物堆”型互动点

## 5.4 围墙、门楼与防御结构

云来镇必须补上完整的边镇防御结构。

最低要求：

- 一圈可见围墙
- 两座可见主门
- 至少一段城上巡道或门楼前缓冲区
- 门内外不能直接一望到底，应有缓冲拐角或空场

这样做的价值：

- 让“门禁”和“宵禁”变得可信
- 让玩家能在空间上感受到“镇里”和“镇外”是两个世界
- 让守门兵、巡夜人、差役这种角色有自然落位

## 5.5 店铺设计原则

当前没有完整交易系统时，店铺也必须先做出来。

原因：

- 店铺不仅是交易 UI 的承载体，也是空间叙事锚点
- 玩家记住的是“去脚店找沈七娘”“去药铺找兰药师”，不是抽象坐标

建议首批必须具备的店铺：

- 脚店
- 药铺
- 铁匠铺
- 杂货棚

当前实现阶段：

- 先做成建筑 + NPC 点位 + 对话与任务入口
- 等商业系统成熟后再补买卖与修理逻辑

## 5.6 宝箱与可探索点设计

当前没有通用容器系统，不建议把“宝箱”做成核心主循环。

但可以先在地图设计层预留以下点位：

- 仓场补给箱
- 废弃屋小箱
- 巡夜封存柜
- 井坊供物箱
- 旧货车暗格

当前阶段建议定位：

- 固定奖励点
- 隐藏宝物点
- 任务交付点
- 真实容器点位
- 与地面物品联动的搜寻点

设计原则：

- 镇内容器数量少，但位置讲道理
- 不要出现新手村到处发光宝箱的廉价设计
- 镇内补给点应更像“封存物资”或“遗落货箱”
- 同一区域要同时考虑“容器里有什么”和“地上散着什么”

## 5.6.1 昼夜与光照对主城设计的影响

云来镇一旦接入时间和光照系统，主城分区就不能只按白天视角设计。

建议夜间层次：

- 南门和东门：稳定火把，保持高可见度
- 南市脚行区：灯火最密，是夜间最安全、最有人气的区域
- 镇中仓场：有巡夜灯，但会留下明显阴影区
- 东门工匠区：炉火和零散灯火形成暖光区
- 北井旧坊区：整体偏暗，只保留少量巡夜灯
- 西侧棚户区：光照不均，安全感最低

这样昼夜切换后，主城仍然是同一张图，但情绪和风险感会明显变化。

## 5.7 云来镇的传送与路线重排

建议保留“主城直接连接两条外线”的清晰结构。

推荐连接：

- 南门 -> `荒野`
- 东门 -> `青竹林`

`灵脊岭` 不建议继续直接从镇中主路裸连。

更合理的处理：

- 北门或北侧旧关道只作为后期叙事区域
- 真正进入 `灵脊岭` 的主线权限应当在凡人篇后段才开放

如果仍保留技术上的直连 portal，也应把落点做在：

- 镇内封闭旧关道尽头
- 由守卫、旧碑、栅门、残关口构成的晚期开启区域

而不是像现在这样像一条随时都能走的普通门路。

## 6. 其余地图的重做方向

## 6.1 荒野

重做目标：

- 不再只是大平地 + 刷怪点
- 要成为“粮路、匪路、沼地、猎道”并存的外线图

建议结构：

- 南部低危猎场
- 中部破路与废营
- 西部匪徒活动区
- 东部沼地妖物区
- 北部通往灵岭的旧关道入口

体验重点：

- 前期刷怪
- 材料获取
- 让玩家明白镇外不是纯怪图，而是人祸地带
- 白天和夜间要像两张不同气氛的图：
  - 白天看得见路和废营
  - 夜里更依赖火把、营火、月光和敌方伏击点

## 6.2 青竹林

重做目标：

- 成为真正的“从人间进入入玄”的过渡图

建议结构：

- 主林道
- 侧向蛇潭
- 竹灵密生区
- 翻车点
- 东北矿路口
- 东南遗迹路

体验重点：

- 林道清障
- 双出口分流
- 让玩家第一次从空间上感受到“林外另有旧体系”
- 借光照强化竹海迷向感：
  - 白天是竹影和水色
  - 黄昏后是视野被切碎的压迫感

## 6.3 玄铁矿洞

重做目标：

- 强化“矿火未灭”的旧工事感

建议结构：

- 外层废矿道
- 中层矿晶洞
- 深层钟乳区
- 旧信标区
- 炉室遗址

体验重点：

- 近战压迫感
- 视线受限
- 材料点明确
- 可通过矿灯、炉火、晶簇冷光做出分区

## 6.4 断碑遗迹

重做目标：

- 强化“旧阵、旧文、旧封镇痕迹”的辨识度

建议结构：

- 外庭碎碑区
- 中庭石卫区
- 内庭符阵区
- 侧殿残卷区
- 底层钥石区

体验重点：

- 精英怪点位清晰
- 石碑、残纹、祭台形成视觉记忆
- 让遗迹和矿洞在空间叙事上形成互补
- 夜间或暗处可用微弱异光强化“旧阵仍在工作”的感觉

## 6.5 噬魂兽谷

重做目标：

- 让它真正像“压力外溢区”，不是单纯 BOSS 场

建议结构：

- 谷口缓冲区
- 狼群盘踞区
- 血羽高地
- 裂息洼地
- 谷底核心区

体验重点：

- 路线变窄
- 怪压更高
- 玩家感受到这是镇外真正压不住的地方
- 夜间应进一步降低安全感，但不能黑到影响基本战斗可读性

## 6.6 灵脊岭

重做目标：

- 成为凡人篇终章地图，而不是普通晚期练级图

建议结构：

- 山门残道
- 虎行岭
- 鹤鸣坡
- 守关碑坪
- 叩门旧台

体验重点：

- 空旷、冷、稀薄
- 视野比兽谷更开，但敌人更强
- 空间上明确表现“已经接近门槛，不再是人间腹地”
- 光照应从“人间灯火”切换到“天光、冷光、残辉”体系

## 7. 推荐实施顺序

不要一次重画所有图，按主线体验优先级推进。

第一阶段：

- 重做 `云来镇`
- 明确围墙、门楼、分区、主功能建筑
- 把主线 NPC 的空间落点重新安排好

第二阶段：

- 重做 `荒野` 与 `青竹林`
- 把“保路 -> 开林道”的凡人篇前半体验做顺

第三阶段：

- 重做 `玄铁矿洞` 与 `断碑遗迹`
- 建立中段双图并进结构

第四阶段：

- 重做 `噬魂兽谷` 与 `灵脊岭`
- 统一凡人篇后段压迫感和终章空间感

## 8. 不该怎么做

以下方向不建议采用：

- 只放大地图面积，不增加功能区和路径结构
- 主城只做装饰房屋，不给明确功能落点
- 每张图都做成中心刷怪盆地
- 镇内到处堆宝箱，破坏边镇可信度
- 用大段对白解释世界，而不通过空间表达世界
- 让 `灵脊岭` 和 `天穹残宫` 在凡人篇里混成同一层级

## 9. 文档结论

凡人篇地图重做的第一优先级不是“多做几张新图”，而是先把 `云来镇` 做成一个真正成立的主城据点。

只要 `云来镇` 仍然像测试地图，凡人篇就很难立住。

重做完成后的凡人篇空间体验，应当让玩家自然感受到这条链路：

- 我先在镇里站住
- 我开始替镇子跑路
- 我发现路外面不只是匪和兽
- 我看见旧系统在坏
- 我替人间把压力往外顶了一段
- 我最后才走到叩门的位置

这样地图、玩法和剧情才会真正合一。
