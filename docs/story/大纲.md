# 大纲

本文档用于给后续“完全重构剧情”做现状整理。

- 口径优先级：
  1. `packages/server/data/content/quests/*.json` 任务真源
  2. `docs/story/**` 剧情与时期文档
  3. `docs/assistant/道劫余生知识库-剧情任务地图.md`、`docs/凡人期任务流程总览.md` 等整理型文档
- 原则：
  - 先整理当前仓库里“已经写出来的剧情骨架”
  - 区分“已实装主线/支线”和“后续草案”
  - 不在本文档里直接重写新剧情

## 0.5 后续重构的叙事硬约束

你已经明确的新方向是：

- 避免所谓“真相大白”
- 避免给出完整故事
- 所有任务、剧情都只从侧面进入
- 只给后果、残留、误读、传闻、口供、遗物和现场痕迹
- 整体改成碎片化叙事，而不是线性揭秘叙事

据此，后续重构时默认遵守以下规则：

1. 不设置一个会被完整说清的总答案
2. 不让任何单个 NPC 掌握全貌并把全貌讲完
3. 玩家完成任务后，拿到的是局部印证，不是结论
4. 每张地图只提供自己的伤痕、习惯、尸体、规矩和活法，不负责替世界观做总解释
5. 同一事件允许存在互相冲突的说法，且不强制裁定哪一个是真的
6. 主线不再追求“越来越接近真相”，而是“接触越来越多互相咬合但永远缺口很大的碎片”
7. 终章也不给总结陈词，只把人推到更大的缺口前

## 1. 一句话概括当前版本剧情

当前版本现有的任务链表现为：

玩家作为乱世边地中的外乡人，在云来镇一带先为立足、奔走和修行而前行，随后沿着北井异动、青竹林翻车、矿洞信标、断碑钥石、兽谷漏口、天穹残宫这些残片一路往前摸。当前版本已经明显在朝“旧灾、旧阵崩坏、道劫外泄”这类大背景收束，但按你的新要求，后续重构时不应再把这些东西组织成一条会被完整讲清的标准答案。

凡人篇解决的是：

- 你能不能先活下来
- 你能不能碰到“门”
- 你能不能从凡人的混乱里，反复撞见一些永远拼不完整的异常碎片

当前版本尚未真正展开“更高境界主线”，只在终章把目标推到 `半步筑基`。

## 3. 世界底色与叙事原则

结合 `docs/story/README.md` 与当前任务文案，现有剧情的稳定底色如下：

- 玩家不是天命主角，而是乱世里许多入局者之一
- 多玩家共在默认成立，玩家更像“同路人”
- 世界长期承受劫难冲击，但秩序仍在勉力维系
- 前期表层是武侠边镇生存，仙侠异常只以边角和余波出现
- 叙事节奏从“安身立命”推进到“撞见异常”再推进到“被更大的余波推着往前”
- 当前版本里，清玄承担的是控信息者职能；后续若按碎片化重构，这种信息控制不能再通向完整解释

当前版本最核心的叙事体验，不是刻意渲染绝望，而是：

- 劫难不可回避
- 人只能先在乱局里安身立命
- 规矩是维持地方运转的办法
- 碎片只能让人看见局部后果，而不是全貌

## 5. 分章剧情大纲

### 5.1 序章·断道入镇

关键词：

- 外乡人
- 断路
- 宵禁
- 落脚资格
- 北井异动

剧情作用：

- 建立世界底色与玩家位置
- 让玩家进入云来镇体系
- 埋下第一条异常暗线

主要推进：

1. 玩家在断道与残墙边醒来，被老关卒赶去先找落脚点
2. 沈七娘、赵老六、清玄依次把玩家纳入“先证明你有用”的边镇规则中
3. 玩家先学护身本事，再去清理鼠患、流匪，替脚店伤者换药钱
4. 无名伤者醒来，说出北井旧坊相关怪话
5. 清玄并不解释，只让玩家先继续变强，达到能在镇外活着回来的资格

本章留下的碎片：

- 云来镇并不安稳，但仍有人在尽力维持运转
- 玩家并非被选中的人，只是被筛进来的人
- 北井、旧坊、怪异伤口意味着问题早就存在，但没人肯把话讲满

本章结束状态：

- 玩家在云来镇站住脚
- 玩家认识清玄、沈七娘、老关卒等关键引路人
- 第一条“不是普通匪患”的异常线索出现

对应支线主题：

- 铁匠、药师、脚店三类底层生计
- 支线都围绕“脏活、杂活、活路”展开

### 5.2 第一章·粮路讨生

关键词：

- 粮路
- 荒野生计
- 买命钱
- 旧北关

剧情作用：

- 把“镇里求生”扩大成“镇外生路是否还能维持”
- 让玩家从内城杂务进入荒野压力场

主要推进：

1. 沈七娘把玩家推出镇门，要求去见洛猎户
2. 玩家开始处理野彘、匪徒、毒蛛，实质上是在维持云来镇的粮、药、安保
3. 文籍师叔引出“旧北关不是单纯守不住，而是里头的人先不对了”
4. 清玄继续压着话头，只要求玩家继续炼体、锻骨

本章留下的碎片：

- 云来镇并不是封闭孤点，而是被荒野粮路吊着命
- 旧北关异变和现有怪事之间存在关联，但关联只体现在口风和后果上
- 异常并未直接爆发成大灾，而是以“边地秩序逐步坏掉”的方式渗出来

本章结束状态：

- 玩家具备稳定处理镇外事务的资格
- 故事从“先站稳脚跟”推进到“边镇结构正在受损”

对应支线主题：

- 荒野怪人、野修客等“门牌外的人”
- 边地真实生存方式不全在官面秩序里

### 5.3 第二章·青竹追线

关键词：

- 运材路
- 翻车痕迹
- 先救人，再追问
- 竹隐客

剧情作用：

- 从生存线切换到调查线
- 把异常从北井与荒野扩散到更远的东线通路

主要推进：

1. 清玄让玩家前往青竹林见巡林弟子阿栖
2. 玩家先处理咬断运材线的狼群，保住林路
3. 翻车调查让玩家意识到裂口并不完全像野兽造成
4. 兰药师仍坚持先救人，说明当前版本的价值排序仍是“先保住活口”
5. 竹隐客给出方向：矿洞与遗迹里埋着更早的东西
6. 玩家被要求把经脉和胆气练到足以进更深层区域

本章留下的碎片：

- 异常已经不是镇内孤例，而是沿资源线路蔓延
- 矿洞与遗迹成为下一阶段碎片的集中地
- 清玄、兰药师等人都知道事情不对，但没人负责摊牌

本章结束状态：

- 主线从边镇危机正式进入遗迹调查
- 玩家完成从“打杂活”到“跑调查线”的角色升级

对应支线主题：

- 当前归在 `第二章_支线.json` 的支线实际上已经落到 `玄铁矿洞`、`断碑遗迹`
- 说明中段支线的章节边界是按任务链切的，不完全按地图切

### 5.4 第三章·矿洞遗火

关键词：

- 失落信标
- 遗迹钥石
- 旧阵崩坏
- 漏出来的余波

剧情作用：

- 这是当前凡人篇里碎片首次开始互相咬合的章节
- 前面零散的怪话和异常，在这里第一次显出结构，但仍不该被整理成标准答案

主要推进：

1. 玩家先去玄铁矿洞找锻炉余火，确认矿路不是自然死掉
2. 通过失落信标核心，追问矿道和镇里的断裂历史
3. 再去断碑遗迹找守纹残念，追问“为什么会坏”
4. 玩家夺回遗迹钥石，并收集断纹石片
5. 清玄正式承认：云来镇外的怪事不是偶然，而是和更早的崩坏残留有关
6. 但他仍然不让玩家立刻碰最深处，只要求玩家先入先天

本章留下的碎片：

- 旧地、旧物、旧人之间能互相对上
- 矿洞更像留下了事故痕迹
- 遗迹更像留下了守口和封存痕迹
- 问题不是单点怪谈，而是一整条旧线都烂过

本章结束状态：

- 玩家第一次拿到足够多的相邻碎片
- 玩家从异常见闻者，变成开始被异常后果追着走的人

对应支线主题：

- 矿监残魂、碑灵学士等亡灵型 NPC 强化“死人比活人更保留旧历史”
- 中段支线整体承担补旧账、认旧名、读旧碑的功能

### 5.5 第四章·兽谷问门

关键词：

- 漏口
- 兽谷核心
- 灵脊岭
- 行令
- 凡人与修士之间的窄门

剧情作用：

- 把“旧阵崩坏”从历史成因推进成正在现实外泄的风险点
- 正式建立“叩门”概念

主要推进：

1. 清玄指向噬魂兽谷，说那是最先往外漏的一道口子
2. 玩家从重伤巡山弟子口中确认，谷底那团东西不是自然长成的
3. 兽谷核心被带回后，清玄承认整条旧线都在往下塌
4. 玩家去灵脊岭见守岭老修
5. 守岭老修明确：灵脊岭守的不是普通山路，而是凡人与修士之间还没彻底断掉的窄门
6. 玩家通过行令试炼，获得“继续往前”的资格
7. 清玄要求玩家正式迈入练气境

本章留下的碎片：

- 旧阵问题已经不再只是历史遗迹，而是在现实中持续扩散
- “门”第一次被清楚说出口
- 凡武尽头与修行世界之间还残留一条并不稳固的通路

本章结束状态：

- 玩家从凡武边缘正式踏到修行门槛前
- 下一步不再是旁观异常，而是要用修为硬碰门后的东西

对应支线主题：

- 谷边埋骨、拿爪换路、霜华照脉都强调“门槛前的代价”
- 支线气质比前几章更接近修行者世界，也更强调门槛与代价

### 5.6 第五章·残宫封天

关键词：

- 天穹残宫
- 星铁雨
- 星陨金
- 封柱
- 天封核心

剧情作用：

- 这是凡人篇主线的最高处
- 前几章主要还在地面与地下摸索残痕，这一章开始碰到更高层留下的遗存

主要推进：

1. 练气后，清玄允许玩家去天穹残宫见观星残影
2. 观星残影说明封柱真正崩断前曾下过一场“星铁雨”
3. 玩家先夺回星陨金，获得凡人之手也能碰天门残物的材料
4. 玩家再深入最深处，击败噬星兽，取回天封核心
5. 清玄拿到核心后，云来镇至少暂时从下一场塌陷边缘退开

本章留下的碎片：

- 当前灾变源头和天穹残宫、封柱崩断有关
- 天上的残毁和地上的灾变显然属于同一类后果，但仍没有谁能把前因后果完整说清
- 玩家能做到的仍只是“抢回一块能续命的骨头”

本章结束状态：

- 凡人篇主线达成阶段性胜利
- 但这不是解决问题，只是把镇子从立刻暴毙里拖出来

对应支线主题：

- 残宫傀壳、猎者遗物都在强调“天上掉下来的残骸也必须被地上人重新利用”

### 5.7 终章·道劫未尽

关键词：

- 旧灾第一层皮
- 道劫未尽
- 半步筑基

剧情作用：

- 当前版本收束
- 从“阶段胜利”转成“长期任务宣告”

主要推进：

1. 天封核心带回后，云来镇暂时稳住
2. 清玄第一次把口径说死：玩家看到的只是旧灾往人间漏出的第一层
3. 若想继续追下去，先把道基立住
4. 主线用 `道基初誓` 把目标推到 `半步筑基`

本章留下的碎片：

- 当前凡人篇并不是闭环大结局，只是第一卷的阶段终点
- 眼前这场灾祸显然远没结束，但故事不给总解释，只把人继续往前推

本章结束状态：

- 当前版本正式结束
- 叙事焦点从“继续查”切到“先让自己成为能继续查的人”

## 6. 主线中的碎片堆叠链

当前剧情真正有效的地方，不是“解谜”，而是残片不断堆叠：

1. 无名伤者与北井怪话
2. 旧北关“里头的人先不对了”
3. 青竹林翻车裂口不似野兽
4. 竹隐客指向矿洞与遗迹
5. 矿洞信标说明矿路不是自然死掉
6. 遗迹钥石说明遗迹在守着更深层的东西
7. 断纹石片拼出“旧阵崩坏、余波外泄”
8. 兽谷核心说明现实仍在漏
9. 灵脊岭说明凡人与修士之间的窄门尚未彻底断绝
10. 天穹残宫与天封核心把问题抬到更高层灾变
11. 清玄在终章只留下“这事还远没完”的口风

后续若按你的要求重构，应该保留“堆叠感”，去掉“最终会被解释清楚”的预期。

## 7. 关键角色与叙事职能

| 角色 | 当前定位 | 剧情职能 |
| --- | --- | --- |
| 清玄执事 | 主引路人、控信息者 | 负责压住话头，只让玩家碰到下一层后果和残片 |
| 沈七娘 | 边镇落脚点掌柜 | 代表“先把日子撑住”的现实规则 |
| 老关卒 | 旧路守望者 | 把玩家从路边混乱推进镇内秩序 |
| 马会头 | 门槛执行者 | 用制度体现边镇的筛选逻辑 |
| 兰药师 | 救命派现实主义者 | 一再提醒“先保住活人，别急着追问” |
| 赵老六 / 洛猎户 / 阿栖 | 边地活路执行者 | 负责把抽象危机落成具体的路、粮、材、命 |
| 竹隐客 | 半脱离体制的知情人 | 负责把调查线往更深处拨一下 |
| 锻炉余火 / 守纹残念 / 碑灵学士 | 旧线残留者 | 负责提供矿洞、遗迹的历史与残响 |
| 重伤巡山弟子 | 漏口见证者 | 证明兽谷不是普通妖兽区 |
| 守岭老修 | 门槛看守者 | 正式说出“凡人与修士之间的门” |
| 观星残影 | 高层残响见证者 | 负责把问题抬到天穹残宫与封柱层 |
| 岑止 / 闻烬 | 归藏脉窟支线角色 | 连接终章后的练气期边缘内容 |

## 11. 当前剧情资料的结构问题

下面这些不是重写意见，而是已经能确认的现状问题。

### 11.1 真源、摘要、样章分散

当前剧情资料至少分在四层：

- 任务真源 `packages/server/data/content/quests/*.json`
- 总目录 `docs/story/README.md`
- 样章 `docs/story/凡人/chapter-01-duandao-ruzhen.md`
- 汇总文档 `docs/assistant/道劫余生知识库-剧情任务地图.md`、`docs/凡人期任务流程总览.md`

后续重构如果不先定“谁才是剧情真源”，很容易出现文档口径继续漂移。

### 11.2 深度剧情文档只完整写到了第一章样章

`docs/story/凡人/README.md` 里当前只明确挂出了第一章样章，第二章到终章没有同层级的剧情长文档。

结果是：

- 已实装剧情骨架主要存在于 quest JSON
- 文档层对后几章的“主题、情绪、角色关系、场景写法”沉淀不够均匀

### 11.3 六个凡人期支线文件都存在缺少 `chapter` 字段的问题

当前缺 `chapter` 的文件：

- `序章_支线.json`
- `第一章_支线.json`
- `第二章_支线.json`
- `第三章_支线.json`
- `第四章_支线.json`
- `第五章_支线.json`

这不影响玩法推进，但会让后续做章节整理、自动目录、剧情统计时边界不够稳定。

### 11.4 中段支线的“章节名”和“实际地图”已经有交叉

例如：

- `第二章_支线.json` 里的任务已经落到 `玄铁矿洞`、`断碑遗迹`
- `第三章_支线.json` 继续补矿洞与遗迹

这说明当前中段支线更像按“剧情段”切文件，而不是按主线章节严格切。

### 11.5 终章支线已经提前长进练气期边缘

`终章_支线.json` 的三条支线已经发生在 `归藏脉窟`：

- `守口旧话`
- `五炁落账`
- `混脉试手`

这会带来一个现状问题：

- 主线口径说“当前只收束到半步筑基”
- 但支线已经让玩家提前摸到练气期终图边缘

后续重构时需要决定，这到底算“当前卷尾声外延”，还是“下一卷预告内容”。

### 11.6 当前剧情最强的是残片之间的咬合，不是角色弧光

现有剧情的优势是：

- 线索推进清楚
- 地图推进清楚
- 危机层层抬高很稳定

但按你刚定的新方向，后续不该继续强化“大因果链”，而应该转成强化“局部残片的密度、回声和彼此冲突”。

现有剧情相对薄的地方是：

- 玩家与关键 NPC 的关系弧线还不算强
- 派系、立场、人物之间的冲突层次还不多
- 更多是“谁负责放下一句信息”，少一些“谁因为这场灾变付出了不可逆的代价”

这也是后续完全重构时最值得重点处理的部分。

## 12. 从现状里抽出来的可保留骨架

如果后续要完全重构，当前版本里最值得保留的骨架大致有六条：

1. 外乡人不是主角天选，而是从边地秩序里被筛出来的人
2. 前期表层是武侠边镇立足，异常只从边角和后果里露出来
3. 地图推进和剧情推进高度绑定，空间感明确
4. 每一章都能留下有辨识度的残片、遗物、口风和现场后果
5. 清玄“压着话头、逼你先站稳脚跟”的推进方式可以保留，但不能再通向总解释
6. 终章不做总结，只把人推到一个更大的缺口前

## 13. 这份整理文档的用途

后续完全重构剧情时，可以直接把本文档当成三份底稿来用：

1. 当前剧情内容盘点
2. 当前剧情结构问题列表
3. 当前剧情里值得保留的骨架清单

这样下一步重构时，就可以在“不丢现有优势”的前提下，决定哪些章节重写、哪些角色重做、哪些谜团顺序要改、哪些支线要并线或拆线。

## 14. 重构版章节结构

这一版不是把现有章节简单润色，而是把它改造成更适合碎片化叙事的章节骨架。

总原则：

- 每章不负责回答问题，只负责留下更清晰的后果
- 每章至少有 `3` 组来源不同的碎片，彼此能咬合，但不能互相盖棺定论
- 每章至少有 `1` 个“地方是如何继续运转”的现实任务簇，避免剧情只剩调查
- 每章都要有一个“玩家看见了，但没人愿意正面解释”的场面
- 章节收束不做总结，只把玩家推到下一处更大的缺口

### 14.1 第一卷定位

- 卷名暂仍可沿用 `外乡人`
- 范围仍覆盖 `realmLv 1-18`
- 核心不是“查明道劫”，而是“在劫难边缘逐渐学会怎么活、怎么走、怎么承受”

这一卷重构后的章节节奏建议为：

1. 入镇
2. 立足
3. 追索
4. 对照
5. 叩门
6. 见天
7. 留白

### 14.2 序章·入镇

章节职能：

- 让玩家进入世界
- 建立“人都在躲劫，但没人能完全躲开”的基调
- 第一次让玩家接触无法解释的后果

本章应保留的碎片源：

- 路边伤者的怪异伤口
- 守门人对北井旧坊的回避
- 脚店夜里听见的怪话

本章不该做的事：

- 不解释伤口来源
- 不把北井写成明确副本入口
- 不让任何人告诉玩家“你卷进大事了”

本章任务簇建议：

- 入镇资格簇
  - 跑腿、挂名、换住处、补药钱
- 夜里见闻簇
  - 守夜、听错、记错、转述
- 门边杂务簇
  - 清鼠、收拾、搬运、巡看

章节收束画面：

- 玩家进了镇，但没有更安全
- 玩家只知道有些地方别人不让碰、不让问

### 14.3 第一章·立足

章节职能：

- 把“活下来”变成“如何持续过日子”
- 把劫难写成对粮路、药路、人手和规矩的长期消耗

本章应保留的碎片源：

- 荒野沿路残缺的车辙与弃置货袋
- 不同 NPC 对旧北关的不同说法
- 药铺与脚店对同一名伤者截然不同的态度

本章不该做的事：

- 不把荒野匪患写成单纯反派清剿
- 不把旧北关写成一个马上要进去解谜的秘密点

本章任务簇建议：

- 粮路维持簇
  - 猎物、短差、押路、护送
- 药材换命簇
  - 取药、换药、带口信、补缺货
- 口风拼接簇
  - 问三个人，得到三个不完全一致的旧事版本

章节收束画面：

- 镇子还在运转，但每一条维持秩序的线都在发紧

### 14.4 第二章·追索

章节职能：

- 让玩家第一次主动去追某个痕迹
- 但追到的不是答案，而是更多不一致的现场

本章应保留的碎片源：

- 运材车翻覆后的裂口
- 青竹林里不合常理的遗留物
- 竹隐客含糊其辞但指向更深处的话

本章不该做的事：

- 不让竹隐客变成剧情讲解员
- 不把“翻车”写成一个有标准真相的案件

本章任务簇建议：

- 林路清障簇
  - 清狼、修路、接回失散脚夫
- 翻车查验簇
  - 看轮印、看木料、看断绳、带回碎片
- 对错转述簇
  - 玩家把现场说给不同 NPC，每个人只认一部分

章节收束画面：

- 玩家第一次感觉到，不同地方留下的是同一种破损痕迹

### 14.5 第三章·对照

章节职能：

- 把矿洞与遗迹并列起来
- 让玩家意识到“不同地方留下的是能互相印证的后果”

本章应保留的碎片源：

- 失落信标核心
- 遗迹钥石
- 断纹石片
- 矿监残魂、守纹残念、碑灵学士这些各执一词的旧留者

本章不该做的事：

- 不要把矿洞负责“前因”、遗迹负责“后果”写得过分工整
- 不要让清玄在本章末尾下一个世界观定义

本章任务簇建议：

- 矿路残灯簇
  - 抢回核心、修复照明、看见旧矿道一瞬亮起
- 遗迹守门簇
  - 取钥石、认纹路、听残念各说半句
- 交叉比对簇
  - 把来自两地的碎片交给不同 NPC，各自得出不同判断

章节收束画面：

- 玩家手里第一次同时握着几块彼此能对上的东西
- 但谁也不给结论

### 14.6 第四章·叩门

章节职能：

- 把“异常残留”推进成“现实门槛”
- 明确玩家继续往前，不只是因为好奇，而是因为已经被局势推到门边

本章应保留的碎片源：

- 兽谷核心
- 血羽
- 灵岭行令
- 守岭老修对“门”的老说法

本章不该做的事：

- 不把“门”解释成单一设定名词
- 不把灵脊岭写成明确的升阶考场

本章任务簇建议：

- 谷口回收簇
  - 收尸、收骨、带回还认得出的东西
- 漏口逼近簇
  - 处理不断冒出来的兽潮、怪羽、异变痕迹
- 行令试身簇
  - 不是考试，而是一连串“你扛不扛得住”的现场活

章节收束画面：

- 玩家知道前面确实有门
- 但门后是什么，仍然没人讲清

### 14.7 第五章·见天

章节职能：

- 把视野从地表和遗迹抬到更高处
- 让玩家看见“劫难留下的不只是地上的坏处，还有更高处掉下来的残余”

本章应保留的碎片源：

- 星陨金
- 封柱残片
- 观星残影的话
- 噬星兽守着的核心区

本章不该做的事：

- 不让观星残影把天穹残宫的来龙去脉说完
- 不把天封核心写成万能答案或万能修复件

本章任务簇建议：

- 星铁回收簇
  - 猎者遗物、散落星铁、残兵旧器
- 高处残影簇
  - 看见旧景的片段，但每次都只是一瞬
- 核心夺回簇
  - 不是救世终战，而是一次勉强抢回关键遗存的行动

章节收束画面：

- 玩家知道自己只是从更大的劫难里抢回了一小块还能用的东西

### 14.8 终章·留白

章节职能：

- 收束当前卷
- 不回答问题
- 只确认玩家已经不能退回最初那个“只求过一夜”的状态

本章应保留的碎片源：

- 清玄不肯说满的话
- 云来镇恢复运转时留下的新规矩
- 通往下一阶段的门槛要求

本章不该做的事：

- 不做世界观总总结
- 不做“原来一切都是因为……”式结论
- 不做煽情收官

本章任务簇建议：

- 交付后续簇
  - 把该交的东西交掉，把该立的约立下
- 镇中回响簇
  - 同一事件在不同 NPC 嘴里留下不同余音
- 出门准备簇
  - 不是领奖，而是为下一段路做准备

章节收束画面：

- 镇子暂时稳住
- 玩家被要求继续走，但前面没有被说明白

## 15. 碎片化任务簇模板

这一节用于指导后续把具体 quest 重写成碎片化任务。

### 15.1 一个任务簇的标准结构

每个任务簇建议只承担一个很小的叙事功能：

1. 看见一个后果
2. 处理一个现实问题
3. 带回一个物件、口信或见闻
4. 把这个碎片交给不同人，得到不一致回应

不要让单个任务同时承担：

- 世界观解释
- 角色剖白
- 章节总结
- 真相揭晓

### 15.2 可重复使用的碎片来源

- 伤口
- 遗物
- 缺页
- 旧账
- 残碑
- 错写的口供
- 说不全的话
- 不同版本的地名
- 被人回避的禁忌称呼
- 一闪而过的旧景
- 物件和现场对不上的细节

### 15.3 任务文本的写法约束

- 多写“你看到什么”“别人怎么处理这件事”
- 少写“这一切意味着什么”
- 多写地方如何继续运转
- 少写宏大判断
- 多写人对同一件事的不同态度
- 少写作者替角色下结论

### 15.4 NPC 的用法约束

- NPC 负责说立场，不负责说全貌
- NPC 负责转述、误读、隐瞒、回避、偏信
- NPC 可以记得细节，但不要掌握最终解释
- 关键 NPC 之间要互相打架，不要口径整齐

### 15.5 章节收束的写法约束

每一章结尾只允许给出三类收束：

- 现实层收束
  - 例如路通了、药够了、门开了、镇子稳住了
- 视野层收束
  - 例如玩家看见了更高层留下的后果
- 门槛层收束
  - 例如玩家获得继续往前走的资格

不允许给出：

- 世界观总结
- 完整前因后果
- 唯一正确答案
- “本章谜底揭晓”

## 16. 可落地任务线清单

这一节直接服务于后续 quest JSON 重写。

原则：

- 只使用当前真源里已经存在的任务目标类型
- 不额外发明协议字段
- 每条任务线都要能拆成一串小任务，而不是一段大剧情说明

当前可直接复用的目标类型：

- `talk`
- `kill`
- `kill_collect`
- `submit_item`
- `learn_technique`
- `realm_stage`

### 16.1 序章·入镇任务线

建议任务簇数量：`3` 组，合计 `7-9` 条任务

#### A. 入镇挂名线

作用：

- 先把玩家放进镇内秩序
- 让“落脚资格”成为第一层门槛

推荐任务序列：

1. `talk`
   - 去见脚店掌柜，拿到临时挂名机会
2. `talk`
   - 带口信给门边熟人，确认今晚还能不能进店
3. `submit_item`
   - 交住宿押钱、路上捡回的凭物，或替人带到的证明物

本线产出的碎片：

- 镇子如何筛人
- 有些人能进门，有些人不行
- 北井旧坊被人刻意回避

#### B. 门边杂务线

作用：

- 让玩家先在现实问题里动起来
- 不把开场做成纯对话

推荐任务序列：

1. `kill`
   - 清门边鼠患或驱散近门混乱
2. `kill_collect`
   - 清掉趁乱摸来的匪徒，收回腰牌、药钱或门边失物
3. `submit_item`
   - 把收回的东西交给掌柜、门头或药师

本线产出的碎片：

- 镇子外头并不稳
- 很多规矩都和“今晚能不能撑过去”有关

#### C. 夜里见闻线

作用：

- 留下第一批真正有记忆点的异常碎片

推荐任务序列：

1. `talk`
   - 夜里看护伤者、听见含糊其辞的话
2. `talk`
   - 把听来的话说给第二个人，对方只认一半
3. `learn_technique`
   - 学一门基础护身技法，作为“你先别问，先学会护身”的现实回应

本线产出的碎片：

- 怪异伤口
- 北井怪话
- 大家都知道有问题，但没人把话说满

### 16.2 第一章·立足任务线

建议任务簇数量：`3-4` 组，合计 `8-10` 条任务

#### A. 粮路维持线

作用：

- 把日常维持写成剧情主轴

推荐任务序列：

1. `talk`
   - 去荒野找能说得上话的人
2. `kill_collect`
   - 打猎、收肉、收牙、收能下锅的材料
3. `submit_item`
   - 交给镇里对应的人，各人反应不同

本线产出的碎片：

- 镇子靠外面的路和货活着
- 所有人都在围着同一条线使力

#### B. 药路续命线

作用：

- 强化“劫难不是大爆炸，而是慢慢拖垮一切”

推荐任务序列：

1. `talk`
   - 药师不肯白给药，只给一个方子
2. `kill_collect`
   - 取药材、毒物、丝囊、胆块之类的实物
3. `submit_item`
   - 交药后得到的不是答案，只是伤者又多撑一天

本线产出的碎片：

- 药铺和脚店对同一件事的看法完全不同
- 人先被保住，事却仍说不清

#### C. 旧关口风线

作用：

- 提供第一组互相冲突的旧事版本

推荐任务序列：

1. `talk`
   - 问老关卒
2. `talk`
   - 问文籍师叔
3. `talk`
   - 问另一个不在正面秩序里的人

本线产出的碎片：

- 同一个旧北关，至少有三种说法
- 玩家拿到的是传闻差异，不是定论

#### D. 境界站稳线

作用：

- 把练体、锻骨写成“继续往前的资格”

推荐任务序列：

1. `learn_technique`
2. `realm_stage`

本线产出的碎片：

- 修行不是奇遇奖励，而是继续做事的门槛

### 16.3 第二章·追索任务线

建议任务簇数量：`3` 组，合计 `7-9` 条任务

#### A. 林路清障线

作用：

- 让玩家进入青竹林，不是为了探秘，而是为了先把路理出来

推荐任务序列：

1. `talk`
   - 去见巡林人
2. `kill`
   - 清掉反复咬路的怪物
3. `submit_item`
   - 带回路上捡到的工具、断绳、遗落物

本线产出的碎片：

- 林路坏得不只是“怪多”
- 有些损坏像是旧事留下的

#### B. 翻车查验线

作用：

- 让玩家第一次主动查看现场痕迹

推荐任务序列：

1. `talk`
   - 去翻车点查看
2. `submit_item`
   - 交轮片、断绳、裂木、遗留布片这类现场碎片
3. `talk`
   - 分别给不同 NPC 看，每人只认一部分

本线产出的碎片：

- 车不是普通翻的
- 但没有谁能或愿意给出完整说法

#### C. 深路指向线

作用：

- 把故事推向矿洞和遗迹，但仍不正面解释

推荐任务序列：

1. `kill_collect`
   - 为了救人或补药，取深处材料
2. `talk`
   - 去找竹隐客或相似角色
3. `realm_stage`
   - 提升到足以进更深层区域

本线产出的碎片：

- 深处显然埋着更久的事
- 玩家还没资格直接碰它

### 16.4 第三章·对照任务线

建议任务簇数量：`4` 组，合计 `9-12` 条任务

#### A. 矿路残灯线

作用：

- 用矿洞的“还能不能重新亮一下”做视觉记忆点

推荐任务序列：

1. `talk`
   - 去见矿洞幸存者、守火人或旧匠
2. `kill_collect`
   - 抢回信标核心或照明组件
3. `submit_item`
   - 交回后让旧矿路短暂亮一次

本线产出的碎片：

- 矿路不是自然烂掉的
- 有些旧设施曾经服务于更大的体系

#### B. 遗迹守门线

作用：

- 让遗迹呈现“有人一直在守某种东西”的感觉

推荐任务序列：

1. `talk`
   - 去见守门残念或碑灵
2. `kill_collect`
   - 取钥石、碎纹、守门残件
3. `submit_item`
   - 把纹路、钥石交给不同的人看

本线产出的碎片：

- 遗迹不是废了就空了
- 还有“守着”的意志残留

#### C. 旧账对照线

作用：

- 让矿洞和遗迹的碎片互相咬合

推荐任务序列：

1. `submit_item`
   - 交矿册、石片、残图、账页
2. `talk`
   - 问矿监残魂、碑灵学士、清玄
3. `talk`
   - 三方给出不一致判断

本线产出的碎片：

- 同一条旧线有不同版本
- 玩家能感到它们是同一件事，但无法拼成定稿

#### D. 先入先天线

作用：

- 把“知道得差不多了”转换成“你还得先练”

推荐任务序列：

1. `learn_technique`
2. `realm_stage`

本线产出的碎片：

- 越往前，门槛越像承受力，而不是等级数字

### 16.5 第四章·叩门任务线

建议任务簇数量：`3` 组，合计 `8-10` 条任务

#### A. 谷口回收线

作用：

- 先让玩家处理“已经发生了什么”

推荐任务序列：

1. `talk`
   - 去见还活着的人
2. `kill_collect`
   - 收骨、收羽、收回残留标识物
3. `submit_item`
   - 把收回来的东西交还给镇里或岭上相关人

本线产出的碎片：

- 这里不是普通猎场
- 每次漏出来的东西都会留下可辨认后果

#### B. 漏口逼近线

作用：

- 强化“问题正在现实里继续扩散”

推荐任务序列：

1. `kill`
   - 压下不断冒出的异变怪群
2. `kill_collect`
   - 收兽谷核心、血羽、异常残块
3. `talk`
   - 交给不同人看，得到不同旧说法

本线产出的碎片：

- “门”的说法开始反复出现
- 但谁也没法把门后讲清

#### C. 行令试身线

作用：

- 不做考试感，而做承担感

推荐任务序列：

1. `talk`
   - 去岭上见守门人
2. `kill_collect`
   - 完成数个与守路有关的现场活
3. `submit_item`
   - 交行令或证明物
4. `realm_stage`
   - 迈入下一阶段

本线产出的碎片：

- 行令不是奖励，而是承担后才拿得到的通行凭据

### 16.6 第五章·见天任务线

建议任务簇数量：`3` 组，合计 `7-9` 条任务

#### A. 星铁回收线

作用：

- 把“高处留下的后果”落成玩家能接触的实物

推荐任务序列：

1. `talk`
   - 去见残宫里的见证者
2. `kill_collect`
   - 从猎者、傀壳、旧械处取回星陨金或残片
3. `submit_item`
   - 交回后触发下一步许可

本线产出的碎片：

- 天上掉下来的东西，不是奇宝，而是灾后遗存

#### B. 残影见闻线

作用：

- 让玩家看见更高层旧景，但每次只是一闪而过

推荐任务序列：

1. `talk`
   - 触发残影见闻
2. `submit_item`
   - 交某种能触发回响的残片或纹片
3. `talk`
   - 听到不完整旧景，不做解释

本线产出的碎片：

- 旧景只会掠过，不会复述成完整历史

#### C. 核心夺回线

作用：

- 作为凡人篇最高强度的行动收束

推荐任务序列：

1. `kill`
   - 清最深处的阻挡者
2. `kill_collect`
   - 夺回核心
3. `submit_item`
   - 交给镇中主引路人

本线产出的碎片：

- 玩家不是解决劫难，只是抢回一件还能续用的关键遗存

### 16.7 终章·留白任务线

建议任务簇数量：`2-3` 组，合计 `4-6` 条任务

#### A. 交付与回响线

作用：

- 让全卷后果在镇里形成回声

推荐任务序列：

1. `submit_item`
   - 交核心、交信物、交旧账
2. `talk`
   - 分别听几个人怎么理解这一路发生的事

本线产出的碎片：

- 同一件事在不同人眼里意义完全不同

#### B. 继续上路线

作用：

- 把“终章”写成门槛，而不是结论

推荐任务序列：

1. `learn_technique`
   - 学下一阶段需要的基础法门
2. `realm_stage`
   - 提升到下一阶段门槛

本线产出的碎片：

- 当前卷结束，事情没结束
- 玩家获得的是继续往前走的资格

#### C. 终章支线挂接线

作用：

- 专门处理 `归藏脉窟` 这类已经提前露头的内容

推荐处理方式：

- 作为“尾声外延”单独归组
- 不并入凡人篇主线正文
- 文本上明确这是下一阶段的边缘回响，不是本卷谜底补完

## 17. 后续落地顺序

如果继续往代码真源推进，建议按下面顺序做：

1. 先确定“重构后每章保留几组任务簇”
2. 再确定“每组任务簇对应哪些旧 quest 可以删、并、改”
3. 再写新的章节文案大纲
4. 最后才改 `packages/server/data/content/quests/*.json`

最稳妥的做法不是一次性全改，而是：

1. 先拿 `序章 + 第一章` 做完整样板
2. 确认碎片化写法可用后，再平推到后续章节
