# 开天门与仙道前置流程规范

## 1. 目标

- 将 `开天门` 从“`叩仙门 -> 练气前期` 的一次性界面”升级为一套可复用的仙道前置步骤规范。
- 明确 `开天门`、后续可能出现的 `结丹`、`结婴` 等步骤，统一遵守“前置未完成时，先做前置，再谈正常突破”的顺序规则。
- 明确“整套重洗但保留额外概率”的重置语义，避免 GM、道具、未来系统各写一套不一致逻辑。

## 2. 核心原则

- `开天门` 不再只绑定“当前正在 `叩仙门` 且刚好满经验”这一瞬时状态。
- 只要玩家境界达到或超过 `叩仙门`，且当前角色还存在未完成的开天门流程，就必须先完成开天门，之后才能继续后续仙道突破。
- 所有仙道前置步骤都应当由服务端按固定顺序检查。
- 同一时刻只允许玩家处理“顺序链中最靠前的未完成前置步骤”。
- `开天门`、`斩灵根`、`补灵根`、`逆天改命`、`重置天门` 都走服务端即时处理，不进入地图 1Hz tick 队列。

## 3. 仙道前置顺序规范

### 3.1 顺序链

- 当前版本先定义第一条仙道前置链：
  - `开天门`
- 后续若增加：
  - `结丹`
  - `结婴`
  - 其他仙道关键步骤
- 则统一追加到同一条顺序链中，由服务端按顺序依次检查。

### 3.2 服务端判定原则

- 服务端在以下场景都必须检查仙道前置顺序链：
  - 左上角主突破按钮展示
  - 点击主突破按钮
  - 任何会推进大境界的行为
- 若顺序链中存在未完成前置：
  - 主按钮显示该前置步骤，而不是普通突破目标
  - 点击后进入该前置步骤对应流程
- 只有当前置链全部满足后，才允许继续走普通境界突破。

### 3.3 当前版本的开天门判定

- 玩家境界 `realmLv >= 18`
- 且当前角色尚未完成本轮开天门
- 则视为“需要先开天门”

这里的“尚未完成本轮开天门”指：

- 当前没有正式生效的天门结果
- 或执行过重置天门后，已回到待开门状态

## 4. 开天门页面与交互

### 4.1 页面结构

- 弹层中心为丹田主节点。
- 五行节点均匀分布在丹田周围。
- 每个五行节点与丹田之间有一条可交互连线。
- 中心主按钮有两种文案：
  - `开天门`
  - `入天门`
- 当当前已开出结果但尚未确认时，底部显示 `逆天改命`。

### 4.2 五行交互

- 悬浮未斩断连线时，提示态为 `斩`。
- 悬浮已斩断连线时，提示态为 `补`。
- 点击未斩断连线：
  - 二次确认后执行 `斩灵根`
  - 消耗当前境界经验上限的 `10%`
- 点击已斩断连线：
  - 二次确认后执行 `补灵根`
  - 消耗当前境界经验上限的 `10%`
- 最多只允许斩断 `4` 条，保证至少保留 `1` 条灵根。

### 4.3 斩补对当前结果的影响

- 只要当前已经开出一套结果，再执行 `斩灵根` 或 `补灵根`：
  - 当前结果立刻失效
  - `入天门` 退回 `开天门`
  - `逆天改命` 暂时隐藏
  - 需要按新的保留灵根配置重新开门

## 5. 开天门、入天门、逆天改命

### 5.1 开天门

- 点击 `开天门` 后二次确认。
- 确认后按当前剩余灵根配置随机出一套候选结果。
- 该结果只写入开天门暂存状态，不算正式完成。
- 只有后续确认 `入天门`，才算本轮开天门真正落定。

### 5.2 入天门

- 只有已经拥有一套候选结果时，中心按钮才显示为 `入天门`。
- 点击 `入天门` 后二次确认。
- 确认后：
  - 将当前候选结果写入正式灵根
  - 标记本轮开天门完成
- `入天门` 本身不直接代替正常境界突破。
- 完成后，玩家仍需按正常突破条件继续推进后续境界。

### 5.3 逆天改命

- 只有已经拥有一套候选结果时，底部才显示 `逆天改命`。
- 点击后需二次确认。
- 消耗当前境界经验上限的 `25%`。
- 本次操作不会直接重抽结果，而是：
  - 清掉当前候选结果
  - 保留当前斩根配置
  - 让“平均品质额外加成”永久增加 `+2`
  - 回到重新 `开天门` 的状态

### 5.4 额外加成的语义

- 这里增加的是“平均品质加成”，不是“额外触发另一套独立结算”。
- 正确语义是：
  - 先按当前概率表抽出基础平均品质
  - 再叠加累计的额外平均品质加成
  - 最终平均品质统一封顶为 `100`
- 当本次结果中出现多个 `100` 时：
  - 第一个 `100` 直接保留
  - 从第二个 `100` 开始，按 `averageBonus` 做额外保留检定
  - 检定失败的 `100` 统一降为 `99`
- 也就是说：
  - `averageBonus = 20` 通常仍然不会直接把结果顶到 `100`
  - 但当累计加成足够高时，平均品质可以自然被推到 `100`
  - 真正的“五条全 `100`”仍然会比“总值非常接近满值”更难出现

## 6. 重置天门规范

## 6.1 重置来源

- GM 面板重置天门
- 道具触发重置天门
- 未来其他明确声明“重置天门”的系统能力

以上来源必须共用同一套服务端重置逻辑。

## 6.2 重置语义

`重置天门` 的标准定义为“整套重洗，但保留累计额外概率”。

重置时应当清除：

- 当前已斩灵根配置
- 当前已开出的候选结果
- 当前是否已入天门
- 当前正式生效的灵根结果

重置时必须保留：

- 开天门累计的 `averageBonus`

### 6.3 禁止行为

- 禁止 GM 重置把 `averageBonus` 清零。
- 禁止碎灵丹重置与 GM 重置逻辑不一致。
- 禁止未来其他“重置天门”来源绕过统一服务端方法。

## 7. 碎灵丹规范

### 7.1 道具定义

- 新增道具：`碎灵丹`
- 类型：`consumable`
- 效果：执行一次标准 `重置天门`

### 7.2 使用效果

- 使用后立刻执行标准重置天门逻辑。
- 结果是：
  - 整套重洗
  - 但保留累计 `averageBonus`
- 若玩家当前境界已经达到或超过 `叩仙门`，则重置后应当立刻重新具备开天门入口。

### 7.3 掉落

- `坠星宫主` 新增 `碎灵丹` 掉落
- 掉落概率：`1%`

## 8. 数值规则

## 8.1 灵根值范围

- 已斩灵根固定为 `0`
- 保留灵根单项范围固定为 `1 - 100`
- 总值上限由保留条数决定：
  - `1` 灵根：`1 - 100`
  - `2` 灵根：`2 - 200`
  - `3` 灵根：`3 - 300`
  - `4` 灵根：`4 - 400`
  - `5` 灵根：`5 - 500`

## 8.2 随机方式

- 先按剩余灵根数 `n` 抽一个平均品质值 `A`
- 总值 `T = A * n`
- 再按当前保留灵根数对应的偏科程度，把总值 `T` 分配给每条保留灵根
- 平均品质统一按“基础区间结果 + `averageBonus`”计算
- 最终平均品质统一封顶为 `100`
- 不再为 `100` 单独走另一套 bonus 分支判断
- 但若结果中出现多个 `100`，则对第二个及之后的 `100` 追加一次基于 `averageBonus` 的保留检定；检定失败时降为 `99`

## 8.3 当前初始平均目标

- `5` 灵根：平均约 `25`
- `4` 灵根：平均约 `30`
- `3` 灵根：平均约 `40`
- `2` 灵根：平均约 `50`
- `1` 灵根：平均约 `70`

## 9. 持久化与数据落点

必须正式持久化到数据库的状态包括：

- 当前已斩灵根配置
- 当前已开出的候选结果
- 当前是否已入天门
- 当前正式灵根结果
- 当前累计 `averageBonus`

其中：

- `averageBonus` 必须作为正式真源持久化
- 禁止只存在内存
- 禁止只存在浏览器存储
- 禁止在 GM 重置、碎灵丹重置后被误清空

## 10. 服务端实现约束

- `开天门` 系列操作继续走服务端即时处理链，不进入地图 tick 队列。
- 主突破按钮展示与点击逻辑，必须统一走“仙道前置顺序链”判定，而不是把 `开天门` 写死在单个境界分支里。
- `重置天门` 必须有单一服务端入口，供 GM、道具和未来系统复用。
- 后续若增加 `结丹`、`结婴` 等步骤，不再另写一套独立“抢按钮”逻辑，而是统一接入顺序链。

## 11. 当前版本落地结论

- 玩家境界只要达到或超过 `叩仙门`，且本轮开天门未完成，就应始终能重新进入开天门流程。
- `重置天门` 的结果是“整套重洗，但保留累计额外概率”。
- `逆天改命` 每次让 `averageBonus` 增加 `+2`，且该值长期保留。
- `碎灵丹` 是标准重置天门道具，`坠星宫主` 以 `1%` 概率掉落。
- 未来仙道关键步骤统一走“顺序链阻塞”模型，先做最前面的未完成步骤，再做后续突破。
