# 开天门设计

## 1. 目标

- 将 `叩仙门 -> 练气前期` 的突破改造成一套独立的 `开天门` 流程。
- `开天门`、`斩灵根`、`补灵根`、`逆天改命` 都走服务端即时处理，不进入地图 1Hz tick 队列。
- 前端使用单实例详情弹层承载整套操作，保持可关闭、可重复进入。

## 2. 入口规则

- 当前境界为 `叩仙门`，且本次角色在该境界中曾经达到过满经验时，左上角原突破按钮改为 `开天门`。
- 该入口在当前角色仍处于 `叩仙门` 时持续显示，不因之后经验被消耗而消失。
- 玩家离开 `叩仙门` 后，入口状态重置，由服务端根据新境界重新判定。

## 3. 页面结构

- 弹层中心为一个圆形丹田主节点。
- 五行节点均匀分布在中心周围五个方向。
- 每个五行节点通过一条带光效的连线连接至丹田。
- 初始阶段：
  - 中心按钮文案为 `开天门`
- 已完成一次开天门随机后：
  - 中心按钮文案改为 `入天门`
  - 底部新增 `逆天改命`

## 4. 五行交互

- 鼠标悬浮在“某五行节点与丹田之间的连线”上时：
  - 若该灵根尚未斩断，则提示态为 `斩`
  - 若该灵根已斩断，则提示态为 `补`
- 点击未斩断连线：
  - 二次确认后执行 `斩灵根`
  - 消耗 `叩仙门满经验值的 10%`
- 点击已斩断连线：
  - 二次确认后执行 `补灵根`
  - 消耗 `叩仙门满经验值的 10%`
- 最多只允许斩断 4 条，保证至少保留 1 条灵根。
- 一旦已开天门结果存在，后续再执行 `斩` 或 `补`：
  - 当前已生成的灵根结果立刻失效
  - 中心按钮退回 `开天门`
  - `逆天改命` 暂时隐藏
  - 需按新的灵根保留配置重新随机

## 5. 开天门 / 入天门 / 逆天改命

### 5.1 开天门

- 不管是否斩过灵根，都可以点击中心主节点。
- 点击 `开天门` 后二次确认。
- 确认后在当前剩余灵根中随机出一套正式灵根值。
- 该结果立即写入正式战斗数值来源，并持久化。

### 5.2 入天门

- 只有已经拥有一套开天门结果时，中心按钮才显示为 `入天门`。
- 点击 `入天门` 后二次确认。
- 确认后才真正完成 `叩仙门 -> 练气前期` 的境界跃迁。

### 5.3 逆天改命

- 只有已经拥有一套开天门结果时，底部才显示 `逆天改命`。
- 点击后需二次确认。
- 消耗 `叩仙门满经验值的 30%`。
- 在当前剩余灵根配置下重新随机一次。
- 新结果直接覆盖旧结果，并同步覆盖战斗数值中的五行灵根值。
- 玩家可关闭页面后继续刷经验，再回来反复执行该操作。

## 6. 数值规则

## 6.1 灵根值范围

- 已斩灵根固定为 `0`
- 保留灵根单项范围固定为 `1 - 100`
- 因此总值上限由保留条数决定：
  - 1 灵根：`1 - 100`
  - 2 灵根：`2 - 200`
  - 3 灵根：`3 - 300`
  - 4 灵根：`4 - 400`
  - 5 灵根：`5 - 500`

## 6.2 随机目标

- 单灵根上限必须是 `100`
- 五灵根总值必须有极低概率达到 `500`
- 最低值不能为 `0`
- 灵根越少：
  - 单项越容易高
  - 总值上限越低
- 灵根越多：
  - 总值上限越高
  - 单项平均值越低

## 6.3 随机方式

- 先按剩余灵根数 `n` 抽取一个平均品质值 `A`
- `A` 取值范围为 `1 - 100`
- 总值 `T = A * n`
- `A = 100` 为独立极低概率事件，目标概率暂定为 `0.01%`
- 抽到总值后，再按剩余灵根数对应的“偏科程度”把总值分配给每一条保留灵根

## 7. 数据落点

- `是否已在叩仙门满过一次`
- `当前已斩灵根配置`
- `当前已开出的灵根结果`
- `当前是否已开天门但尚未入天门`

以上状态都属于正式运行时数据，最终真源必须落数据库，不能只放内存、浏览器存储或 runtime JSON。

## 8. 即时执行约束

- `斩灵根`
- `补灵根`
- `开天门`
- `逆天改命`
- `入天门`

以上操作均由网关直接进入服务端即时执行链，执行完成后立刻推送玩家自身的属性、行动和提示消息更新。

## 9. 本轮实现范围

- 先实现文档与客户端 UI 原型。
- 本轮 UI 允许先用客户端本地状态模拟开天门流程，用于确认布局、交互和随机手感。
- 正式经验扣除、正式持久化与服务端即时结算在后续步骤接入。
