# 持久化链路

## 概述

持久化链路负责将游戏运行时状态安全地写入数据库，确保玩家数据不丢失。这是 MMO 游戏最关键的链路之一。

## 架构

```
┌─────────────────┐
│   Runtime       │  运行时状态（内存）
│   (权威态)       │
└────────┬────────┘
         │ 标记脏数据
         ▼
┌─────────────────┐
│  Flush Ledger   │  脏数据账本
│   (待刷盘队列)   │
└────────┬────────┘
         │ 定时/触发刷盘
         ▼
┌─────────────────┐
│  Persistence    │  持久化服务
│   Service       │
└────────┬────────┘
         │ 写入
         ▼
┌─────────────────┐
│  PostgreSQL     │  数据库（真源）
└─────────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `persistence/flush-ledger.service.ts` | 脏数据账本管理 |
| `persistence/flush-wakeup.service.ts` | 刷盘唤醒调度 |
| `persistence/player-flush-ledger.service.ts` | 玩家刷盘账本 |
| `persistence/player-persistence-flush.service.ts` | 玩家刷盘执行 |
| `persistence/player-persistence.service.ts` | 玩家数据持久化 |
| `persistence/durable-operation.service.ts` | 强事务操作 |
| `persistence/outbox-dispatcher.service.ts` | Outbox 事件分发 |

## 持久化分层

### 第一层：内存态（Runtime）

- 服务端权威状态
- 高频读写
- 进程重启丢失

### 第二层：Redis 态

- 在线玩家缓存
- 实时态数据
- 跨进程共享

### 第三层：PostgreSQL 态

- 持久化真源
- 玩家资产
- 审计日志

## 刷盘机制

### 脏数据标记

```typescript
// 当玩家状态变更时标记脏
flushLedger.markDirty(playerId, 'inventory');
flushLedger.markDirty(playerId, 'vitals');
flushLedger.markDirty(playerId, 'progression');
```

### 刷盘触发条件

| 触发条件 | 说明 |
|----------|------|
| 定时刷盘 | 每 30 秒检查脏数据 |
| 玩家下线 | 立即刷盘 |
| 关键操作 | 交易、邮件等立即刷盘 |
| 服务关闭 | 优雅关闭时全量刷盘 |

### 刷盘流程

```
FlushWakeupService.tick()
  │
  ├─▶ 检查脏数据账本
  │     - 获取待刷盘玩家列表
  │     - 按优先级排序
  │
  ├─▶ 批量刷盘
  │     - 收集玩家快照
  │     - 批量写入数据库
  │
  └─▶ 清理账本
        - 标记已刷盘
        - 更新时间戳
```

## 玩家数据域

### 域划分

| 域 | 说明 | 刷盘频率 |
|----|------|----------|
| vitals | 生命值、灵气 | 高频 |
| inventory | 背包物品 | 中频 |
| progression | 修为、经验 | 中频 |
| equipment | 装备状态 | 低频 |
| quests | 任务进度 | 低频 |
| placement | 位置信息 | 下线时 |

### 域刷盘服务

```typescript
// player-domain-persistence.service.ts
class PlayerDomainPersistenceService {
  // 按域刷盘，减少写入量
  async flushDomain(playerId: string, domain: string): Promise<void>;

  // 全量刷盘
  async flushAll(playerId: string): Promise<void>;
}
```

## Durable Operation

对于关键操作（交易、邮件、市场），使用强事务保证：

```typescript
// durable-operation.service.ts
class DurableOperationService {
  async execute<T>(
    operationId: string,
    operation: () => Promise<T>,
    rollback?: () => Promise<void>
  ): Promise<T>;
}
```

### 特点

- 幂等性保证
- 失败重试
- 回滚支持
- 审计记录

## Outbox 模式

对于需要异步处理的事件，使用 Outbox 模式：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Runtime    │────▶│   Outbox    │────▶│  Consumer   │
│  写入事件   │     │   (队列)    │     │  异步处理   │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 应用场景

- 战斗审计日志
- 交易记录
- 邮件发送
- 统计数据

## 数据恢复

### 玩家数据恢复

```
服务启动
  │
  ├─▶ 加载玩家快照
  │     - 从 PostgreSQL 读取
  │     - 验证数据完整性
  │
  ├─▶ 恢复运行时状态
  │     - 初始化内存结构
  │     - 恢复临时状态
  │
  └─▶ 重放未完成操作
        - 检查 Outbox
        - 重试失败操作
```

### 地图数据恢复

```
MapPersistenceService.loadInstance()
  │
  ├─▶ 加载地图模板
  │
  ├─▶ 加载实例状态
  │     - 建筑状态
  │     - 资源点状态
  │
  └─▶ 恢复动态实体
        - 怪物刷新
        - NPC 状态
```

## 故障处理

### 刷盘失败

```typescript
try {
  await flushPlayer(playerId);
} catch (error) {
  // 1. 记录错误日志
  logger.error('Flush failed', { playerId, error });

  // 2. 保留脏标记，下次重试
  flushLedger.keepDirty(playerId);

  // 3. 如果连续失败，告警
  if (consecutiveFailures > 3) {
    alertService.critical('Flush repeatedly failed', { playerId });
  }
}
```

### 数据不一致

```bash
# 运行一致性检查
pnpm --filter @mud/server ts-node src/tools/player-consistency-check.ts

# 修复不一致数据
pnpm --filter @mud/server ts-node src/tools/player-data-repair.ts
```

## 监控指标

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| 刷盘延迟 | < 100ms | > 500ms |
| 脏数据积压 | < 100 | > 1000 |
| 刷盘失败率 | < 0.1% | > 1% |
| 数据库写入 QPS | < 1000 | > 5000 |

## 最佳实践

### 1. 避免热路径 IO

```typescript
// 错误：tick 内直接写库
onTick() {
  await db.save(player); // 阻塞 tick
}

// 正确：标记脏数据，异步刷盘
onTick() {
  flushLedger.markDirty(playerId);
}
```

### 2. 批量写入

```typescript
// 错误：逐条写入
for (const player of players) {
  await db.save(player);
}

// 正确：批量写入
await db.batchSave(players);
```

### 3. 关键操作立即刷盘

```typescript
// 交易完成后立即刷盘
async completeTrade(trade: Trade) {
  // 1. 更新内存状态
  applyTrade(trade);

  // 2. 立即刷盘双方数据
  await Promise.all([
    flushPlayer(trade.buyerId),
    flushPlayer(trade.sellerId),
  ]);
}
```
