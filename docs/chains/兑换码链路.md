# 兑换码链路

## 概述

兑换码链路负责兑换码的创建、分发和兑换流程。支持批量生成、分组管理和频率限制。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  创建分组   │────▶│  生成兑换码 │────▶│  玩家兑换   │────▶│  发放奖励   │
│  (GM 操作)  │     │  (批量生成) │     │  (校验+限频)│     │  (Durable)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/redeem/redeem-code-runtime.service.ts` | 兑换码运行时核心 |
| `persistence/redeem-code-persistence.service.ts` | 兑换码持久化 |

## 详细流程

### 1. 创建分组

```
RedeemCodeRuntimeService.createGroup(name, rewards, count)
  │
  ├─▶ 校验名称唯一
  ├─▶ 校验奖励物品有效
  ├─▶ 批量生成兑换码
  │     - 36 位大写字母数字
  │     - 字符表: 0-9A-Z
  │     - crypto.randomBytes 生成
  │     - 单次最多 500 个
  └─▶ 全量持久化到 PostgreSQL
```

### 2. 玩家兑换

```
RedeemCodeRuntimeService.redeemCodes(playerId, codes)
  │
  ├─▶ 频率限制检查
  │     - 同一玩家 3 秒内不可重复兑换
  │     - 缓存 TTL 60 秒，最多 10000 玩家
  │
  ├─▶ 逐码校验
  │     - 状态: active / used / destroyed
  │     - 单次最多 5 个码
  │
  ├─▶ 校验背包容量
  │
  ├─▶ 发放奖励
  │     - 物品: DurableOperationService.grantInventoryItems
  │     - 灵石: mutatePlayerWallet（独立钱包事务）
  │
  ├─▶ 标记码为 used
  │
  └─▶ 持久化 + 发通知 + 日志
```

### 3. 追加码

```
appendCodes(groupId, count)
  │
  ├─▶ 生成新码追加到内存
  └─▶ 持久化
```

### 4. 销毁码

```
destroyCode(codeId)
  │
  ├─▶ 仅 active 状态可销毁
  ├─▶ 标记 destroyed
  └─▶ 持久化
```

## 数据结构

### PostgreSQL 表

| 表名 | 用途 |
|------|------|
| `server_redeem_code_state` | 全局 revision |
| `server_redeem_code_group` | 分组信息 |
| `server_redeem_code` | 兑换码明细 |

## 关键约束

- **互斥**: 所有写操作通过 `runExclusive` 互斥锁串行化
- **频率限制**: 同一玩家 3 秒冷却
- **数量限制**: 单次兑换最多 5 码，单次生成最多 500 码
- **灵石识别**: `spirit_stone` 走钱包路径，不占背包格
- **回滚**: 发物失败时完整回滚（恢复 inventory/revision/dirtyDomains）
- **启动加载**: 启动时 `loadDocument` 全量加载到内存
