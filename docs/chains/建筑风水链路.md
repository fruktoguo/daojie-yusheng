# 建筑风水链路

## 概述

建筑风水链路负责建筑放置、建造施工、拆除以及房间检测和风水评分的完整流程。建筑系统支持幂等操作和审计追踪。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  放置建筑   │────▶│  建造施工   │────▶│  房间检测   │────▶│  风水评分   │
│  (幂等校验) │     │  (tick推进) │     │  (BFS填充)  │     │  (多维评分) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/world/world-runtime-building.service.ts` | 建筑运行时（函数式 API） |
| `runtime/building/fengshui-calculator.service.ts` | 风水计算 |
| `runtime/building/room-detection.service.ts` | 房间检测（BFS） |
| `runtime/building/building-topology-index.service.ts` | 拓扑索引（TypedArray） |
| `runtime/building/building-content.repository.ts` | 建筑内容编译 |

## 放置建筑流程

```
handleBuildPlaceIntent(requestId, playerId, ...)
  │
  ├─▶ 幂等检查（requestId 去重，缓存上限 1000）
  ├─▶ 查找 CompiledBuildingDef
  ├─▶ 解析材料消耗
  ├─▶ instance.placeBuildingInstance()
  │     - 碰撞检测
  │     - 占位注册
  ├─▶ 扣除材料
  ├─▶ 记录操作审计
  └─▶ 返回结果（缓存供幂等重放）
```

## 建造施工流程

```
dispatchStartBuildingConstruction()
  │
  ├─▶ 设置 player.buildingJob
  │     - remainingTicks
  │     - 目标建筑信息
  │
  ├─▶ 每 tick: tickBuildingConstruction()
  │     - remainingTicks--
  │
  └─▶ 完成后
        - 清除 job
        - 奖励建造经验
```

## 拆除流程

```
handleBuildDeconstructIntent()
  │
  ├─▶ 校验所有权
  ├─▶ instance.deconstructBuildingInstance()
  └─▶ 释放占位
```

## 房间检测

```
RoomDetectionService.detectRooms()
  │
  ├─▶ BFS 洪水填充
  ├─▶ 判断封闭性（是否触碰地图边缘）
  ├─▶ 计算属性
  │     - 面积、周长
  │     - 门窗数量
  │     - 屋顶覆盖率
  │
  ├─▶ 过滤条件
  │     - 面积 > 256 且屋顶覆盖率 < 60% → 半户外，不计入
  │     - 必须有门或窗才算有效房间
  │
  └─▶ 生成 RoomInstance
```

## 风水评分

```
FengShuiCalculatorService.calculate(roomAggregate)
  │
  ├─▶ 多维度评分
  │     - 形状分
  │     - 封闭分
  │     - 气分
  │     - 煞分
  │     - 舒适度分
  │     - 五行分（相生 +45，相克 -60/-40）
  │     - 阵法分
  │
  ├─▶ 输出
  │     - 总分（FENGSHUI_SCORE_MIN ~ FENGSHUI_SCORE_MAX）
  │     - 等级
  │     - 原因列表
  │
  └─▶ 房间角色推断（inferRoomRole）
        - 基于特征 trait 打分
        - 置信度 ≥ 60 且领先优势 ≥ 30
```

## 关键约束

- **幂等**: 相同 requestId 直接返回缓存结果
- **拓扑索引**: TypedArray（Uint32Array/Uint8Array）按 cellIndex 存储，动态 2 的幂次扩容
- **持久化**: 建筑状态随 world instance 持久化，非独立表
- **审计**: 操作日志在内存（上限 1000 条）
- **generic 房间**: 分数上限 520
