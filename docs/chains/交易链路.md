# 交易链路

## 概述

交易链路涵盖玩家间物品交易的完整流程，包括坊市挂单、一口价购买、求购匹配和拍卖竞价。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  创建挂单   │────▶│  撮合匹配   │────▶│  执行交易   │────▶│  结果通知   │
│  (卖/买/拍) │     │  (价格匹配) │     │  (原子操作) │     │  (双方同步) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/market/market-runtime.service.ts` | 坊市运行时核心 |

## 交易类型

### 一口价出售（Sell）

```
卖家创建挂单
  │
  ├─▶ 校验物品
  │     - 物品存在且可交易
  │     - 数量足够
  │
  ├─▶ 锁定物品
  │     - 从背包移至坊市仓库
  │     - 标记为挂单中
  │
  ├─▶ 创建挂单
  │     - 设置价格、数量、过期时间
  │     - 写入数据库
  │
  └─▶ 广播挂单
        - 通知坊市面板更新
```

### 一口价购买（Buy Now）

```
买家点击购买
  │
  ├─▶ 校验挂单
  │     - 挂单存在且有效
  │     - 未被其他人购买
  │
  ├─▶ 校验买家
  │     - 灵石足够
  │     - 背包有空间
  │
  ├─▶ 执行交易（原子操作）
  │     - 扣除买家灵石
  │     - 物品转入买家背包
  │     - 灵石转入卖家账户
  │     - 关闭挂单
  │
  ├─▶ 记录交易
  │     - 写入交易历史
  │     - 审计日志
  │
  └─▶ 通知双方
        - 买家：物品到账
        - 卖家：灵石到账
```

### 求购（Buy Order）

```
买家创建求购单
  │
  ├─▶ 校验灵石
  │     - 灵石足够
  │
  ├─▶ 锁定灵石
  │     - 从账户冻结
  │
  ├─▶ 创建求购单
  │     - 设置物品、价格、数量
  │
  └─▶ 等待匹配
        - 有卖家挂单时自动撮合
```

### 拍卖（Auction）

```
卖家创建拍卖
  │
  ├─▶ 设置起拍价、一口价（可选）
  │
  ├─▶ 设置拍卖时长
  │
  └─▶ 开始拍卖

竞价过程
  │
  ├─▶ 买家出价
  │     - 必须高于当前最高价
  │     - 锁定出价灵石
  │     - 退还上一出价者灵石
  │
  ├─▶ 延时机制
  │     - 结束前 30 秒内出价
  │     - 延长拍卖时间
  │     - 最多延长 1 小时
  │
  └─▶ 拍卖结束
        - 最高出价者获得物品
        - 卖家获得灵石
```

## 撮合逻辑

### 价格匹配

```typescript
// 买单匹配卖单
function matchBuyOrder(buyOrder: MarketOrder): MarketOrder[] {
  return openSellOrders
    .filter(sell =>
      sell.itemId === buyOrder.itemId &&
      sell.price <= buyOrder.price
    )
    .sort((a, b) => a.price - b.price); // 价格优先
}

// 卖单匹配买单
function matchSellOrder(sellOrder: MarketOrder): MarketOrder[] {
  return openBuyOrders
    .filter(buy =>
      buy.itemId === sellOrder.itemId &&
      buy.price >= sellOrder.price
    )
    .sort((a, b) => b.price - a.price); // 价格优先
}
```

### 撮合优先级

1. 价格优先（买高卖低）
2. 时间优先（先挂先成交）

## 原子性保证

交易执行使用 Durable Operation 保证原子性：

```typescript
async executeTrade(order: MarketOrder, buyer: Player) {
  await durableOperation.execute(
    `trade:${order.orderId}:${buyer.id}`,
    async () => {
      // 1. 扣除买家灵石
      await deductCurrency(buyer.id, order.totalPrice);

      // 2. 转移物品
      await transferItem(order.sellerId, buyer.id, order.itemData);

      // 3. 增加卖家灵石
      await addCurrency(order.sellerId, order.totalPrice);

      // 4. 关闭挂单
      await closeOrder(order.orderId);

      // 5. 记录交易
      await recordTrade(order, buyer.id);
    },
    async () => {
      // 回滚逻辑
      await rollbackTrade(order, buyer.id);
    }
  );
}
```

```typescript
interface MarketStorage {
```

### 仓库操作

| 操作 | 说明 |
|------|------|
| 存入 | 从背包移入仓库 |
| 取出 | 从仓库移入背包 |
| 挂单 | 仓库物品上架 |
| 退回 | 挂单取消/过期时退回仓库 |

## 手续费

```typescript
const MARKET_FEE_RATE = 0.05; // 5% 手续费

function calculateFee(price: number): number {
  return Math.floor(price * MARKET_FEE_RATE);
}

// 卖家实际收入 = 成交价 - 手续费
const sellerIncome = price - calculateFee(price);
```
