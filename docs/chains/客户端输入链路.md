# 客户端输入链路

## 概述

客户端输入链路负责键盘和鼠标事件的捕获、节流和分发。将用户操作转化为游戏意图发送给服务端。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  原生事件   │────▶│  输入处理   │────▶│  意图转化   │────▶│  Socket 发送│
│  (DOM)      │     │  (节流/映射)│     │  (游戏语义) │     │  (C2S)      │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `client/src/input/keyboard.ts` | 键盘事件处理与快捷键映射 |
| `client/src/input/mouse.ts` | 鼠标事件处理与地图交互 |
| `client/src/input/throttle.ts` | 输入节流工具 |
| `client/src/game-map/interaction/` | 地图交互逻辑 |

## 键盘输入

```
keyboard.ts
  │
  ├─▶ 监听 keydown / keyup
  ├─▶ 快捷键映射
  │     - 方向键 → 移动意图
  │     - 数字键 → 技能快捷栏
  │     - ESC → 取消/关闭
  │     - 其他功能键
  │
  └─▶ 转化为游戏指令
        - C2S.Move（方向移动）
        - C2S.UseSkill（技能使用）
        - UI 操作（面板开关等）
```

## 鼠标输入

```
mouse.ts
  │
  ├─▶ 监听 click / mousemove / mousedown / mouseup
  │
  ├─▶ 地图点击
  │     - 坐标转换（屏幕 → 地图格）
  │     - 目标识别（玩家/怪物/NPC/地块）
  │     - C2S.MoveTo（点击移动）
  │     - C2S.Attack（点击攻击）
  │     - NPC 交互
  │
  ├─▶ UI 交互
  │     - 面板按钮点击
  │     - 物品拖拽
  │     - 右键菜单
  │
  └─▶ Hover 效果
        - 地块高亮
        - 实体 tooltip
```

## 节流机制

```
throttle.ts
  │
  ├─▶ 防止高频重复发送
  ├─▶ 移动指令节流
  └─▶ 技能指令冷却检查
```

## 地图交互

```
game-map/interaction/
  │
  ├─▶ 点击目标解析
  │     - 命中检测（哪个实体被点击）
  │     - 优先级：NPC > 怪物 > 玩家 > 地块
  │
  ├─▶ 交互分发
  │     - NPC → 打开对话/商店/任务
  │     - 怪物 → 攻击/观察
  │     - 玩家 → 观察/交易
  │     - 地块 → 移动/采集
  │
  └─▶ 拖拽与选区
        - 框选（GM 模式）
        - 物品拖放
```

## 关键约束

- **客户端只收集意图**: 不做合法性判断
- **节流**: 防止高频重复发送相同指令
- **坐标转换**: 屏幕坐标必须正确转换为地图格坐标
- **触控适配**: 手机端考虑触控命中区域和安全区
- **焦点保护**: 输入处理不打断当前焦点/输入框
