# 任务链路

## 概述

任务链路负责 NPC 任务的接取、推进、提交和奖励发放。任务系统由事件驱动（击杀、交互、学技能），不参与 tick 循环。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  接取任务   │────▶│  推进进度   │────▶│  提交任务   │────▶│  发放奖励   │
│  (NPC交互)  │     │  (事件驱动) │     │  (NPC交互)  │     │  (自动接续) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/world/world-runtime-quest-state.service.ts` | 任务状态刷新、击杀/学技能进度推进 |
| `runtime/world/world-runtime-npc-quest-write.service.ts` | NPC 任务交互写路径 |
| `runtime/world/world-runtime-quest-runtime-facade.service.ts` | 门面层统一转发 |
| `runtime/world/query/world-runtime-quest-query.service.ts` | 查询层：进度计算、ready 判定 |

## 详细流程

### 1. 接取任务

```
dispatchAcceptNpcQuest(playerId, questId)
  │
  ├─▶ 校验 NPC 邻近
  ├─▶ 校验任务可用（未接取、前置完成）
  ├─▶ cloneQuestState(quest, 'active')
  ├─▶ 推入 player.quests.quests
  └─▶ markQuestStateDirty
```

### 2. 推进进度

按任务类型分支：

```
kill 类：
  advanceKillQuestProgress(playerId, monsterId)
    → progress +1
    → 自动 refreshQuestStates

talk 类：
  dispatchInteractNpcQuest()
    → 匹配 targetNpcId + targetMapId
    → progress = required

learn_technique 类：
  advanceLearnTechniqueQuest()
    → progress = required
```

### 3. 状态刷新

```
refreshQuestStates()
  │
  ├─▶ 遍历所有 active 任务
  ├─▶ 通过 query 服务重算 progress
  └─▶ progress >= required → status = 'ready'
```

### 4. 提交任务

```
dispatchSubmitNpcQuest(playerId, questId)
  │
  ├─▶ 校验 status = 'ready'
  ├─▶ 校验 submitNpcId 匹配
  ├─▶ 消耗 requiredItem（如有）
  ├─▶ 发放奖励
  │     - 物品入背包
  │     - spirit_stone 入钱包
  │     - 背包空间预检（buildNextQuestInventorySnapshots）
  ├─▶ status = 'completed'
  └─▶ tryAcceptNextQuest()（自动接续）
```

## NPC 交互优先级

```
1. ready 提交 > 2. talk 推进 > 3. available 接取 > 4. active 进度提示 > 5. 默认对话
```

## 关键约束

- **事件驱动**: 不参与 tick 循环，由击杀/交互/使用物品触发
- **背包预检**: 提交前做背包空间预检，空间不足直接拒绝
- **灵石分离**: spirit_stone 走钱包路径，不占背包格
- **不可重复**: 同一任务不可重复接取
- **自动接续**: 完成后自动尝试接取后续任务
