# 建议系统链路

## 概述

建议系统链路负责玩家建议的创建、投票、回复和状态同步。所有写操作通过 mutationQueue 串行化执行。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  创建建议   │────▶│  投票/回复  │────▶│  串行写入   │────▶│  状态同步   │
│  (玩家提交) │     │  (互动)     │     │  (队列)     │     │  (广播)     │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/suggestion/suggestion-runtime.service.ts` | 建议运行时核心 |
| `http/native/native-gm-suggestion-query.service.ts` | GM 建议查询 |
| `persistence/suggestion-persistence.service.ts` | 建议持久化 |

## 详细流程

### 创建建议

```
SuggestionRuntimeService.createSuggestion()
  │
  ├─▶ 校验内容
  │     - 标题限 50 字
  │     - 正文限 500 字
  │
  ├─▶ 通过 mutationQueue 串行执行
  ├─▶ 持久化到数据库
  └─▶ 同步到客户端
```

### 投票

```
vote(playerId, suggestionId, direction)
  │
  ├─▶ 校验建议存在
  ├─▶ 更新投票计数
  └─▶ 持久化 + 同步
```

### 回复

```
reply(playerId, suggestionId, content)
  │
  ├─▶ 校验建议存在
  ├─▶ 添加回复
  └─▶ 持久化 + 同步
```

## 排序方式

| 排序 | 说明 |
|------|------|
| 时间排序 | 按创建时间倒序 |
| 热度排序 | 按投票数排序 |

## 关键约束

- **串行化**: 所有写操作通过 mutationQueue 串行执行
- **字数限制**: 标题 50 字，正文 500 字
- **启动回填**: 模块初始化时从持久化回填建议列表
- **持久化**: 通过 SuggestionPersistenceService 落库
