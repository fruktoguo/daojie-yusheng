# 物品装备链路

## 概述

物品装备链路负责物品的使用、装备穿戴/卸下、地面物品丢弃/拾取以及背包管理的完整流程。所有操作由玩家指令触发，不参与 tick 循环。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  物品使用   │     │  装备穿戴   │     │  地面物品   │
│  (分支分发) │     │  (属性重算) │     │  (丢弃/拾取)│
└─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/world/world-runtime-use-item.service.ts` | 物品使用分发 |
| `runtime/world/world-runtime-equipment.service.ts` | 装备穿戴/卸下 |
| `runtime/world/world-runtime-item-ground.service.ts` | 地面物品丢弃/拾取 |
| `runtime/player/player-runtime.service.ts` | 背包核心操作 |
| `runtime/player/inventory-lock.helpers.ts` | 背包锁定空间（escrow） |

## 物品使用流程（dispatchUseItem）

按优先级分支：

```
1. formationDiskTier → 提示通过布阵页面使用
2. useBehavior === 'create_sect' → 转发宗门创建
3. mapUnlockIds → 解锁地图（支持地图组批量解锁）
4. respawnBindMapId → 绑定复活点
5. tileResourceGains / tileAuraGainAmount → 地块资源增益
   - 禁止在安全区/出生点/NPC 附近使用
6. 默认 → playerRuntimeService.useItem()
   - 消耗物品
   - 学技能任务推进
```

## 装备流程

### 穿戴

```
equipItem(playerId, slotIndex)
  │
  ├─▶ 从背包取出物品
  ├─▶ 检查 equipSlot 匹配
  ├─▶ 检查 craftPanel 锁定
  ├─▶ 旧装备回背包（合并或新槽）
  ├─▶ 新装备进入装备槽
  ├─▶ 属性重算
  └─▶ itemInstanceId 乐观一致性校验
```

### 卸下

```
unequipItem(playerId, equipSlot)
  │
  ├─▶ 从装备槽取出
  ├─▶ 回背包合并
  ├─▶ 属性重算
  └─▶ itemInstanceId 一致性校验
```

## 背包核心机制

### 合并策略

```
canMergeItemStack + createItemStackSignature
  - 签名: itemId + enhanceLevel
  - 同签名堆叠 count
```

### 容量管理

| 约束 | 说明 |
|------|------|
| `inventory.capacity` | 背包格数上限 |
| `MAX_ITEM_COUNT` | 单格数量硬顶 |
| `lockedItems` | escrow 锁定空间（强化/市场期间） |
| `inventory.revision` | 每次变更递增，触发脏标记 |

### 锁定空间（Escrow）

```
强化/市场期间：
  物品移入 player.inventory.lockedItems
  → 不参与正常操作
  → 完成后归还
```

## 地面物品

### 丢弃

```
dropItem(playerId, slotIndex, count)
  │
  ├─▶ 从背包移除
  └─▶ 在玩家脚下生成地面物品
```

### 拾取

```
pickupItem(playerId, groundItemId)
  │
  ├─▶ 校验距离
  ├─▶ 校验背包空间
  ├─▶ 物品入背包（合并或新槽）
  └─▶ 移除地面物品
```

## 关键约束

- **不参与 tick**: 所有操作由玩家指令触发
- **地块资源限制**: 禁止在安全区/出生点/传送点/NPC 附近使用
- **批量使用**: 需物品 `allowBatchUse === true`
- **craftPanel 锁定**: 正在炼制时不可操作对应槽位
- **itemInstanceId**: 装备类物品必须有（lazy 分配 UUID）
- **revision 追踪**: 每次变更触发 `markPlayerDirtyDomains(['inventory'])`
