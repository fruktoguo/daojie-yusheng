# 客户端渲染链路

## 概述

客户端渲染链路负责 Canvas 2D 地图渲染、相机管理、文本测量和图块缓存。渲染层只做显示，不承担任何游戏逻辑。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  服务端同步 │────▶│  状态投影   │────▶│  渲染帧循环 │────▶│  Canvas 绘制│
│  (delta)    │     │  (runtime)  │     │  (rAF)      │     │  (分层)     │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `client/src/renderer/camera.ts` | 相机管理（视口、缩放、平移） |
| `client/src/renderer/text.ts` | 文本渲染 |
| `client/src/renderer/text-measure-cache.ts` | 文本测量缓存 |
| `client/src/renderer/tile-sprite-cache.ts` | 图块精灵缓存 |
| `client/src/renderer/types.ts` | 渲染类型定义 |
| `client/src/game-map/renderer/` | 地图渲染器 |
| `client/src/game-map/camera/` | 地图相机 |
| `client/src/game-map/viewport/` | 视口管理 |
| `client/src/game-map/scene/` | 场景管理 |

## 渲染分层

| 层级 | 内容 | 更新频率 |
|------|------|----------|
| 静态地形层 | 地块、地形纹理 | 跨图/首次 |
| 建筑层 | 建筑、阵法 | 变更时 |
| 动态实体层 | 玩家、怪物、NPC | 每帧 |
| 选择/Hover 层 | 选中高亮、悬停提示 | 交互时 |
| Overlay 层 | 技能范围、路径预览 | 交互时 |

## 相机系统

```
Camera
  │
  ├─▶ 视口计算（viewport）
  │     - 可见区域裁剪
  │     - 只渲染视口内实体
  │
  ├─▶ 缩放（zoom）
  │     - 支持多级缩放
  │     - 缩放中心跟随
  │
  └─▶ 平移（pan）
        - 跟随玩家移动
        - 边界约束
```

## 缓存机制

### 图块精灵缓存

```
TileSpriteCache
  │
  ├─▶ 按 tileId 缓存已渲染的图块图像
  ├─▶ 避免每帧重复绘制相同图块
  └─▶ 地图切换时清理
```

### 文本测量缓存

```
TextMeasureCache
  │
  ├─▶ 缓存 measureText 结果
  ├─▶ 避免每帧重复测量
  └─▶ 字体变更时失效
```

## 关键约束

- **只做显示**: 渲染层不承担移动合法性、碰撞、战斗结算
- **分层更新**: 静态层不因动态实体变化重绘
- **视口裁剪**: 只渲染可见区域内的实体
- **表现插值**: 移动插值只影响显示，不污染权威坐标
- **每帧避免**: 全量解析协议、重复全图查询、大量短命对象
