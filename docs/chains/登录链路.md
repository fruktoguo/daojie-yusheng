# 登录链路

## 概述

登录链路负责玩家身份验证、会话建立和角色数据加载。这是玩家进入游戏的第一个关键链路。

## 链路流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ Client  │────▶│  Auth   │────▶│ Session │────▶│ Runtime │
│ 登录请求 │     │ 身份验证 │     │ 会话建立 │     │ 数据加载 │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `auth/player-token-verify.ts` | JWT token 验证 |
| `auth/password-hash.ts` | 密码哈希处理 |
| `auth/account-validation.ts` | 账号格式校验 |
| `http/native/native-auth.controller.ts` | 登录 HTTP 入口 |
| `persistence/player-persistence.service.ts` | 玩家数据持久化 |
| `persistence/player-identity-persistence.service.ts` | 玩家身份持久化 |
| `runtime/player/player-runtime.service.ts` | 玩家运行时管理 |

## 详细流程

### 1. 客户端发起登录

```typescript
// 客户端
POST /api/auth/login
{
  "username": "player123",
  "password": "xxx",
  "deviceId": "device-uuid"
}
```

### 2. 身份验证

```
NativeAuthController.login()
  │
  ├─▶ 校验账号格式（account-validation.ts）
  │     - 长度限制
  │     - 字符限制
  │     - 敏感词过滤
  │
  ├─▶ 查询账号记录（player-identity-persistence.service.ts）
  │     - 从 PostgreSQL 读取
  │     - 检查账号是否存在
  │
  ├─▶ 验证密码（password-hash.ts）
  │     - bcrypt 比对
  │     - 防时序攻击
  │
  └─▶ 签发 Token（player-token-verify.ts）
        - JWT HS256 签名
        - 包含 playerId、过期时间
```

### 3. Token 结构

```typescript
// JWT Payload
{
  "sub": "player-uuid",      // 玩家 ID
  "iat": 1234567890,         // 签发时间
  "exp": 1234654290,         // 过期时间
  "deviceId": "device-uuid"  // 设备标识
}
```

### 4. 会话建立

```
客户端携带 Token 建立 WebSocket 连接
  │
  ├─▶ Socket.IO 握手
  │     - 验证 Token
  │     - 检查设备 ID
  │
  ├─▶ 顶号检测
  │     - 检查是否有其他设备在线
  │     - 踢出旧连接
  │
  └─▶ 注册 Session
        - 建立 playerId → socket 映射
        - 初始化消息队列
```

### 5. 角色数据加载

```
PlayerRuntimeService.loadPlayer()
  │
  ├─▶ 从 PostgreSQL 读取快照
  │     - server_player_snapshot 表
  │     - 包含位置、属性、背包等
  │
  ├─▶ 恢复运行时状态
  │     - 初始化 PlayerRuntimeState
  │     - 恢复 buff、冷却等
  │
  ├─▶ 加入地图实例
  │     - 根据保存位置加入实例
  │     - 触发 AOI 广播
  │
  └─▶ 发送首包
        - 玩家完整状态
        - 地图静态数据
        - 周围实体信息
```

## 数据结构

### 玩家快照表

```sql
CREATE TABLE server_player_snapshot (
  player_id varchar(100) PRIMARY KEY,
  template_id varchar(120) NOT NULL,
  instance_id varchar(160),
  persisted_source varchar(32) NOT NULL,
  seeded_at bigint,
  saved_at bigint NOT NULL,
  updated_at timestamptz NOT NULL DEFAULT now(),
  payload jsonb NOT NULL
);
```

### 快照 Payload

```typescript
interface PlayerSnapshotPayload {
  placement: {
    instanceId: string;
    templateId: string;
    x: number;
    y: number;
    facing: number;
  };
  vitals: {
    hp: number;
    maxHp: number;
    qi: number;
    maxQi: number;
  };
  progression: {
    foundation: number;
    combatExp: number;
    // ...
  };
  inventory: {
    capacity: number;
    items: InventoryItem[];
  };
  // ...
}
```

## 错误处理

| 错误码 | 说明 | 处理 |
|--------|------|------|
| INVALID_CREDENTIALS | 账号或密码错误 | 返回通用错误，不暴露具体原因 |
| ACCOUNT_LOCKED | 账号被锁定 | 提示联系客服 |
| TOKEN_EXPIRED | Token 过期 | 引导重新登录 |
| DEVICE_MISMATCH | 设备不匹配 | 需要重新验证 |
| SESSION_CONFLICT | 会话冲突（顶号） | 踢出旧连接 |

## 安全措施

### 密码安全

- bcrypt 哈希，cost factor = 12
- 防时序攻击的比对
- 登录失败次数限制

### Token 安全

- HS256 签名
- 短有效期（默认 24 小时）
- 设备绑定

### 会话安全

- 单设备在线（顶号机制）
- 心跳检测
- 异常断开清理

## 断线重连

```
客户端检测到断开
  │
  ├─▶ 使用现有 Token 重连
  │     - 验证 Token 有效性
  │     - 检查会话是否存在
  │
  ├─▶ 恢复会话
  │     - 重新注册 socket
  │     - 恢复消息队列
  │
  └─▶ 同步状态
        - 发送增量更新
        - 恢复 UI 状态
```

## 监控指标

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| 登录成功率 | > 99% | < 95% |
| 登录延迟 | < 500ms | > 2000ms |
| Token 验证延迟 | < 10ms | > 100ms |
| 顶号次数 | < 100/小时 | > 500/小时 |
