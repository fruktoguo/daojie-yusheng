# 分身机器人链路

## 概述

分身机器人链路负责 Actor Blueprint 的注册、缓存和应用。支持 bot 客户端登录时根据蓝图 ID 恢复行为配置，也为分身/宠物玩法提供基础设施。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  蓝图注册   │────▶│  LRU 缓存   │────▶│  Bot 登录   │────▶│  蓝图应用   │
│  (外部传入) │     │  (TTL淘汰)  │     │  (blueprintId)│   │  (运行时)   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/actor/actor-blueprint-registry.service.ts` | 蓝图注册表（LRU + TTL） |
| `runtime/actor/actor-persistence-policy.service.ts` | Actor 持久化策略 |
| `runtime/actor/ephemeral-actor-identity.service.ts` | 临时 Actor 身份管理 |

## 蓝图注册表

```
ActorBlueprintRegistryService
  │
  ├─▶ 容量上限: 1000（LRU 淘汰最旧条目）
  ├─▶ TTL: 30 分钟（过期惰性删除 + 周期 GC）
  ├─▶ 服务重启即全部失效
  │
  ├─▶ register(blueprint) → blueprintId
  │     - 生成唯一 ID（ACTOR_BLUEPRINT_ID_PREFIX + randomBytes）
  │     - 存入 LRU 缓存
  │
  └─▶ get(blueprintId) → ActorBlueprint | undefined
        - 命中则刷新 LRU 位置
        - 过期则惰性删除返回 undefined
```

## Bot 登录流程

```
Bot 客户端登录
  │
  ├─▶ 携带 blueprintId 连接
  ├─▶ 从注册表获取蓝图
  ├─▶ 应用蓝图到 player runtime
  │     - 属性配置
  │     - 行为配置
  │     - AI 策略
  └─▶ 进入正常游戏循环
```

## 当前阶段

- **第 1 批**: 注册表仅承担"接受外部传入的合法蓝图 + 缓存"
- **第 2 批**: 接入克隆逻辑（fromPlayer 蓝图生成）

## 关键约束

- **内存态**: 蓝图纯内存存储，重启丢失
- **LRU 淘汰**: 超过 1000 条淘汰最旧
- **TTL 过期**: 30 分钟未访问自动过期
- **ID 前缀**: 使用 `ACTOR_BLUEPRINT_ID_PREFIX` 标识
