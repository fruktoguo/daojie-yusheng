# NPC 商店链路

## 概述

NPC 商店链路负责 NPC 商品查询和购买结算。读写路径分离，购买通过 DurableOperation 保证原子性。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  商品查询   │────▶│  购买请求   │────▶│  校验结算   │────▶│  物品发放   │
│  (读路径)   │     │  (写路径)   │     │  (扣款)     │     │  (Durable)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/world/world-runtime-npc-shop.service.ts` | 写路径：购买结算 |
| `runtime/world/query/world-runtime-npc-shop-query.service.ts` | 读路径：商品查询 |

## 商品查询（读路径）

```
WorldRuntimeNpcShopQueryService
  │
  ├─▶ 根据 NPC ID 查找商品列表
  ├─▶ 过滤可购买商品（境界/等级限制）
  └─▶ 返回商品信息（名称、价格、库存）
```

## 购买流程（写路径）

```
WorldRuntimeNpcShopService.dispatchBuyFromNpc()
  │
  ├─▶ 校验位置
  │     - 玩家与 NPC 邻近
  │
  ├─▶ 校验 NPC
  │     - NPC 存在且有商店
  │
  ├─▶ 校验商品
  │     - 商品存在且可购买
  │     - 数量合法
  │
  ├─▶ 校验买家
  │     - 货币充足（灵石等）
  │     - 背包有空间
  │
  ├─▶ 扣款
  │     - 扣除对应货币
  │
  ├─▶ 物品发放
  │     - 支持堆叠合并
  │     - 实例 ID 分配（装备类）
  │
  └─▶ DurableOperationService 持久化提交
```

## 关键约束

- **读写分离**: 查询和购买走不同服务
- **位置校验**: 必须在 NPC 附近才能购买
- **原子性**: 购买通过 DurableOperation 保证
- **堆叠合并**: 可堆叠物品自动合并
- **实例 ID**: 装备类物品分配唯一 itemInstanceId
