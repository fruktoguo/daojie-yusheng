# 观察面板显示逻辑统计

本文档统计当前观察面板的显示归类，用于后续继续统一观察体验。

统计时间：2026-05-29

## 总口径

观察面板主内容按三类显示：

1. 地块信息
2. 地面物品
3. 角色信息，仅有角色时显示

右侧辅助气机卡保持原有逻辑，不并入主内容区。

## 数据流

观察面板链路：

1. 客户端执行观察：`createMainObserveStateSource().show(x, y)`。
2. 客户端先用本地可见 tile 和本地 AOI 实体渲染一次。
3. 客户端发送 `C2S.RequestTileDetail`，载荷 `{ x, y }`。
4. 服务端 `WorldGatewayReadModelHelper.handleRequestTileDetail()` 调用 `worldRuntimeService.buildTileDetail(playerId, { x, y })`。
5. 服务端 `WorldRuntimeDetailQueryService.buildTileDetail()` 校验视野，读取权威地块、地面、传送点、安全区、容器、建筑、玩家、怪物、NPC。
6. 服务端发送 `S2C.TileDetail`。
7. 客户端 `handleTileDetail(data)` 只接受当前正在观察的同地图同坐标详情，然后重渲染。

## 服务端 TileDetail 字段

主类型：`TileDetailView`

| 字段 | 来源 | 当前用途 |
|------|------|----------|
| `x` / `y` | 请求坐标 | 面板副标题 |
| `type` | `instance.getEffectiveTileType()` | 地块信息：地貌 |
| `terrainType` | `instance.getTileLayerState()` | 地块信息：底层地形 |
| `surfaceType` | `instance.getTileLayerState()` | 地块信息：地表铺装 |
| `structureType` | `instance.getTileLayerState()` | 地块信息：地上结构 |
| `movementCost` | `instance.getTileTraversalCost()` | 地块信息：移动点数消耗，常驻 |
| `walkable` | `instance.isWalkable()` | 仅不可通行时显示“通行限制：无法通行” |
| `blocksSight` | `instance.isTileSightBlocked()` | 仅遮挡时显示“视线遮挡：会阻挡” |
| `playerOverlap` | `instance.isPlayerOverlapTile()` | 仅可重叠时显示“可重叠：玩家可重叠” |
| `qiDrainPerTick` | `instance.getTileQiDrainPerTick()` | 大于 0 时显示 |
| `interactableKinds` | `tileLayerState.interactableKinds` | 有交互物时显示 |
| `hp` / `maxHp` | `instance.getTileCombatState()` | 有结构时显示结构生命，否则显示地物/壁垒稳固 |
| `aura` / `resources` | `listTileResources()` + aura 投影 | 地块信息和右侧气机卡 |
| `safeZone` | `instance.getSafeZoneAtTile()` | 合并进地块信息 |
| `portal` | `instance.getPortalAtTile()` | 合并进地块信息 |
| `ground` | `instance.getTileGroundPile()` | 地面物品区 |
| `entities` | 同格实体详情 | 客户端再按角色/非角色归类 |
| `error` | 视野或坐标错误 | 观察回响区 + toast |

## entities 组装

服务端 `buildTileDetail()` 当前会向 `entities` 加入：

1. 传送点 `portal`
2. 容器 `container`
3. 地面物品堆 `ground`
4. 建筑 `building`
5. 玩家 `player`
6. 怪物 `monster`
7. NPC `npc`

客户端只把 `player`、`monster`、`npc`、`crowd` 归入角色信息。其余实体归入地块信息或地面物品，不再进入角色信息。

当前仍不由 TileDetail 组装的实体：

| 目标 | 当前来源 |
|------|----------|
| 人群 `crowd` | 客户端本地 AOI 聚合实体 |
| 阵法 `formation` | 客户端本地 AOI 实体；如果同格出现，会作为地上结构名称归入地块信息 |

## 地块信息

地块信息区块常驻显示。

### 基础行

| 行名 | 显示规则 |
|------|----------|
| 地貌 | 优先 structure，其次 surface，其次 terrain，最后 tile type |
| 底层地形 | `terrainType`，fallback 为地貌 |
| 地表铺装 | `surfaceType`，没有显示“无” |
| 地上结构 | `structureType` + 同格建筑/阵法名；没有显示“无” |
| 移动点数消耗 | 常驻；显示 `{cost} 点/格`，不再因为不可通行而隐藏 |

### 条件行

| 行名 | 显示条件 |
|------|----------|
| 通行限制 | `walkable === false` |
| 视线遮挡 | `blocksSight === true` |
| 可重叠 | `playerOverlap === true` |
| 每息灵力消耗 | `qiDrainPerTick > 0` |
| 交互物 | `interactableKinds.length > 0` |
| 地上陈设 | 同格存在容器 |
| 结构生命 | 有地上结构/建筑/阵法，并且有 hp/maxHp |
| 地物稳固 | 无结构但 tile combat 有 hp/maxHp |
| 壁垒稳固 | wall 类型且有 hp/maxHp |
| 驻足气息 | 仅同格存在角色时显示角色名 |
| 最近变动 | `modifiedAt` 存在 |
| 异状 | `hiddenEntrance` 存在 |
| 安全区 | `safeZone` 存在 |
| 灵气 | detail aura 大于 0 |
| 气机 | detail resources 中存在非中性灵气资源 |
| 地面物品 | detail 有 ground 时显示物品种数摘要 |
| 界门去向 | detail 有 portal |
| 触发方式 | detail 有 portal |
| 方向 | detail 有 portal |

### 望气信息

玩家开启望气时，不再单独显示“房间风水”区块，而是把望气结果归入地块信息。

显示内容：

- 房间
- 面积
- 围合
- 门窗
- 风水
- 幸运
- 风水分项：形制、围合、灵气、煞气、舒适/用途、五行、阵法、完整性
- 风水加减摘要：最多 12 条非 0 原因

没有房间信息时显示：

- 房间：此处不在房间内
- 风水：平 0
- 幸运：+0

## 地面物品

地面物品区块常驻显示。

数据来源优先级：

1. detail `ground.items`
2. 本地可见 ground pile
3. 空数组

显示规则：

| 条件 | 显示 |
|------|------|
| 有物品 | 每个物品一行：名称 + 数量 |
| detail 有 ground 但 items 为空 | 此地无物可取 |
| detail 没有 ground 且本地也没有物品 | 地面无物 |

地面物品不再进入角色信息。

## 角色信息

角色信息区块仅在同格存在角色时显示。

归入角色信息的 kind：

| kind | 标签 |
|------|------|
| `player` | 修士 |
| `monster` | 妖兽 |
| `npc` | 人物 |
| `crowd` | 人群 |

不再归入角色信息的 kind：

| kind | 归类 |
|------|------|
| `building` | 地块信息：地上结构 / 结构生命 |
| `formation` | 地块信息：地上结构 |
| `container` | 地块信息：地上陈设 |
| `portal` | 地块信息：界门去向 / 触发方式 / 方向 |
| `ground` | 地面物品 |

### 角色排序

客户端排序权重：

| kind | 权重 |
|------|------|
| `crowd` | 0 |
| `player` | 1 |
| `npc` | 3 |
| `monster` | 4 |

容器等非角色实体即使参与总实体排序，也不会进入角色信息区块。

### 角色卡

非 crowd 角色卡显示：

- 名称
- kind 标签
- observation verdict
- 生命、灵力
- observation lines 中非生命/灵力行
- Buff / Debuff
- 怪物完整洞察后的掉落按钮

crowd 角色卡显示固定的人群说明，不展开单个玩家血条、buff 和详情。

## 各目标当前显示

### 地块

显示在地块信息中：

- 地貌、底层地形、地表铺装、地上结构
- 移动点数消耗
- 不可通行、遮挡视线、可重叠
- 地块/结构生命
- 安全区、隐藏入口、变动、交互物
- 灵气、气机、望气信息

### 玩家

显示在角色信息中。

服务端字段：

- 名称：`resolveObservedPlayerName()`
- 生命：`hp/maxHp`
- 灵力：`qi/maxQi`
- 观察：`buildPlayerObservation()`
- Buff：`projectVisiblePlayerBuffs()`

观察清晰度按观看者神识与目标神识比例计算。看自己固定完整。

### 怪物

显示在角色信息中。

服务端字段：

- 名称、血脉层次、生命、灵力
- 观察：`buildMonsterObservation()`
- Buff：`cloneVisibleBuff()`
- 掉落预览：仅完整洞察时下发

### NPC

显示在角色信息中。

观察内容：

- 身份
- 商号
- 话语
- 委托

### 人群

显示在角色信息中。

来源是客户端本地 AOI 聚合实体。若同格存在 crowd，本地兜底会过滤 player，避免人群和单人重复。

### 建筑

显示在地块信息中。

服务端通过 `MapInstanceRuntime.getBuildingsAtTile()` 查同格建筑，并生成 building entity：

- 名称
- 状态
- 生命
- 藏书，若有

客户端把 building 归入“地上结构”，生命显示为“结构生命”。

### 容器

显示在地块信息中。

客户端把 container 归入“地上陈设”，不再显示为角色卡。

### 传送点

显示在地块信息中：

- 界门去向
- 触发方式
- 方向

不再单独显示传送点区块，也不进入角色信息。

### 地面物品

显示在地面物品区块。

不再进入角色信息。

### 阵法

当前主要来自客户端本地 AOI。若同格显示为实体，客户端归入“地上结构”。阵法半径、范围、所有者等 RenderEntity 扩展字段当前还未展开到观察面板。

## 已知后续统一点

1. `portal`、`ground` 已被服务端放进 TileDetail entities，类型层已扩展，但 domain label 表仍不需要给它们做角色标签，因为它们不进入角色区。
2. 阵法详情仍未由服务端 TileDetail 权威返回，只能依赖本地 AOI。
3. 容器当前只显示名称归类，刷新倒计时和搜索状态后续可决定是否归入地块信息。
4. 结构生命优先显示 tile combat 或 building hp/maxHp；没有生命字段时不强造。
5. 右侧气机卡仍按原有 `buildObservedResourceAsideCards()` 逻辑独立渲染。
