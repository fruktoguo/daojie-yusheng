# 导航移动链路

## 概述

导航移动链路负责玩家从点击目标位置到角色逐格移动到达的完整流程。这是 MMO 中最高频的操作链路之一。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  客户端     │────▶│  导航意图   │────▶│  寻路计算   │────▶│  逐格移动   │
│  点击目标   │     │  入队(覆盖) │     │  A* 路径    │     │  tick 推进  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                    │
                                                                    ▼
                                                            ┌─────────────┐
                                                            │  AOI 广播   │
                                                            │  位置同步   │
                                                            └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/world/world-runtime-navigation.service.ts` | 导航意图管理、寻路调度 |
| `runtime/world/world-runtime-movement.service.ts` | 逐格移动执行、占位检测 |
| `runtime/instance/map-instance.runtime.ts` | 地图实例 tick 中执行移动队列 |
| `network/world-gateway.ts` | 接收客户端移动请求 |

## 详细流程

### 1. 客户端发起导航

```
客户端点击地图目标格
  │
  ├─▶ 发送 navigation intent
  │     - 目标坐标 (x, y)
  │     - 目标实例 ID
  │
  └─▶ 服务端接收意图
        - 覆盖式：同一玩家一息内多次修改目标，以最后一次为准
```

### 2. 导航意图物化（materializeNavigationCommands）

```
World Tick 主循环第 2 步
  │
  ├─▶ 遍历所有 pending navigation intents
  │
  ├─▶ 对每个意图执行 A* 寻路
  │     - 基于地图碰撞层
  │     - 考虑当前占位
  │     - 生成路径点序列
  │
  └─▶ 将路径写入玩家移动队列
        - moveQueue: 待执行的路径点列表
```

### 3. 逐格移动执行

```
MapInstance.tickOnce() → 执行移动队列
  │
  ├─▶ 从 moveQueue 取下一格
  │
  ├─▶ 占位检测
  │     - 目标格是否可通行
  │     - 目标格是否已被占据（玩家不可重叠）
  │     - 碰撞体检测
  │
  ├─▶ 成功：更新玩家坐标
  │     - 释放旧格占位
  │     - 占据新格
  │     - 更新 facing 朝向
  │
  └─▶ 失败：中断移动
        - 清空 moveQueue
        - 通知客户端移动中断
```

### 4. AOI 广播

```
移动完成后
  │
  ├─▶ 视野范围内玩家收到位置更新
  │     - 增量同步：只发变化的实体位置
  │     - protobuf 编码
  │
  ├─▶ 进入新视野的实体：发送 add 事件
  │
  └─▶ 离开视野的实体：发送 remove 事件
```

## 关键约束

- **权威**: 服务端是移动的唯一权威，客户端只做表现插值
- **频率**: 每 tick（1Hz）推进一格
- **占位**: 玩家不可重叠，占位检测由服务端保证
- **覆盖式意图**: 一息内多次修改寻路目标，只按最后目标寻路
- **持久化**: 玩家位置通过 flush 机制定期落库

## 边界红线

- 客户端不做移动合法性判断，只做表现预测
- 寻路不在 tick 热路径中做全图扫描
- 移动不触发数据库 IO，位置变更通过脏标记延迟刷盘
- 占位索引必须是 O(1) 查询
