# 排行榜链路

## 概述

排行榜链路负责按多维度聚合玩家数据生成榜单，支持在线和离线玩家，结果做短缓存避免高频重算。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  数据采集   │────▶│  聚合排序   │────▶│  缓存存储   │────▶│  客户端查询 │
│  (运行态)   │     │  (多维度)   │     │  (TTL刷新)  │     │  (按需拉取) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/player/leaderboard-runtime.service.ts` | 排行榜运行时核心 |

## 详细流程

### 数据来源

```
LeaderboardRuntimeService
  │
  ├─▶ 在线玩家：直接读取运行态
  │
  └─▶ 离线玩家：通过 PlayerDomainPersistenceService 补读快照
```

### 榜单类型

| 榜单 | 排序依据 |
|------|----------|
| 境界榜 | 境界等级 + 修炼进度 |
| 战力榜 | 综合战力评分 |
| 六维榜 | 各属性值 |
| 击杀榜 | 击杀数统计 |

### 缓存策略

| 数据 | TTL |
|------|-----|
| 榜单数据 | 10 分钟 |
| 世界摘要 | 30 秒 |

## 关键约束

- **缓存优先**: 避免高频重算，榜单 10 分钟刷新一次
- **离线补读**: 支持离线玩家数据，通过持久化服务读取
- **不参与 tick**: 按需查询，不在 tick 中计算
- **世界摘要**: 30 秒 TTL 的轻量级全服概览
