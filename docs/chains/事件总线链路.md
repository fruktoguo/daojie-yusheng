# 事件总线链路

## 概述

运行时事件总线是 tick 内事件收集和分发的核心基础设施。它在 tick 内收集各域运行时事件（通知、战斗表现、面板 patch、进度反馈等），tick 末尾统一 flush 交给 WorldSyncService 组包推送。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  事件产生   │────▶│  收集合并   │────▶│  Tick 末尾  │────▶│  组包推送   │
│  (各域服务) │     │  (限流/覆盖)│     │  flushTick  │     │  (Sync服务) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

| 文件 | 职责 |

| `runtime/event-bus/runtime-event-bus.service.ts` | 事件总线核心服务 |

## 事件类型

| 事件类型 | 说明 |
|----------|------|
| 通知（Notice） | 玩家可见的文字/结构化通知 |
| 战斗表现（CombatEffect） | 伤害数字、技能特效 |
| 面板 Patch（PanelPatch） | UI 面板增量更新 |
| 活跃任务进度（ActiveJobProgress） | 制作/采集进度 |
| AOI 表现事件（AoiPresentation） | 视野内表现事件 |
| 玩家状态 Delta（PlayerStateDelta） | 玩家状态变化 |
| 玩家反馈（PlayerFeedback） | 操作反馈 |
| GM 推送标记 | GM 面板数据推送 |

## 详细流程

### 1. 事件产生

```
各域服务在 tick 内产生事件：
  │
  ├─▶ 战斗服务 → CombatEffect
  ├─▶ 制作服务 → ActiveJobProgress
  ├─▶ 任务服务 → Notice
  ├─▶ 物品服务 → PanelPatch
  └─▶ 其他服务 → PlayerFeedback / PlayerStateDelta
```

### 2. 收集与合并

```
RuntimeEventBusService 语义化方法集
  │
  ├─▶ 按玩家 ID 分桶收集
  ├─▶ 按规则合并/覆盖/限流/丢弃
  │     - 同类通知合并
  │     - 面板 patch 覆盖（最新为准）
  │     - 超频事件限流
  └─▶ 存入 PlayerEventQueue / InstanceEventQueue
```

### 3. Tick 末尾 Flush

```
WorldTickService 调用 flushTick()
  │
  ├─▶ drain PlayerEventQueue → PlayerDrainResult
  ├─▶ drain InstanceEventQueue → InstanceDrainResult
  └─▶ 交给 WorldSyncService 组包推送
```

## 关键约束

- **不落库**: 事件总线纯内存，不持久化
- **不跨进程**: 单进程内事件分发
- **不持有 socket**: 只收集事件，不直接推送
- **tick 边界**: 事件在 tick 内收集，tick 末尾统一 flush
- **合并策略**: 同类事件按规则合并，避免爆发式推送
