# 通天塔链路

## 概述

通天塔链路负责通天塔副本实例的层级管理、怪物波次生成和调度。玩家逐层挑战，每层有独立的怪物配置和波次机制。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  进入通天塔 │────▶│  层级初始化 │────▶│  波次生成   │────▶│  通关/失败  │
│  (实例创建) │     │  (spawn配置)│     │  (怪物调度) │     │  (清理回收) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/world/world-runtime-tongtian-tower.service.ts` | 通天塔运行时核心 |
| `persistence/tongtian-tower-persistence.service.ts` | 通天塔持久化 |
| `data/content/tongtian-tower.json` | 通天塔配置 |
| `data/content/monsters/通天塔.json` | 通天塔怪物配置 |

## 详细流程

### 1. 进入通天塔

```
玩家进入通天塔
  │
  ├─▶ 创建/获取通天塔实例
  ├─▶ 确定当前层级（layer）
  └─▶ 初始化层级状态
```

### 2. 层级状态管理

```
每层独立状态：
  │
  ├─▶ spawn 计时
  ├─▶ 空闲销毁计时
  ├─▶ 活跃波次状态
  └─▶ 怪物存活计数
```

### 3. 波次生成

```
按层配置生成怪物波次
  │
  ├─▶ 普通怪物 spawn
  ├─▶ 精英怪物 spawn（特定波次）
  ├─▶ 支持多人参与同一波次
  └─▶ 波次全灭 → 下一波次/通关
```

### 4. 空闲清理

```
cleanupIdleTongtianInstances()
  │
  ├─▶ 检测无玩家的通天塔实例
  ├─▶ 空闲超时 → 销毁实例
  └─▶ 释放资源
```

## Tick 中的行为

```
World Tick 主循环第 10 步：
  cleanupIdleTongtianInstances()
  → 清理空闲通天塔实例
```

## 关键约束

- **独立实例**: 每个通天塔层级可能有独立地图实例
- **波次机制**: 怪物按波次生成，全灭后推进
- **空闲回收**: 无玩家时自动销毁实例
- **持久化**: 通过 TongtianTowerPersistenceService 保存进度
- **配置驱动**: 层级和怪物配置从 JSON 加载
