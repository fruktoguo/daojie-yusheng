# 配置编辑器链路

## 概述

配置编辑器链路负责游戏内容（怪物、技能、地图、物品等）的可视化编辑、校验和发布。编辑器产出的配置经过 schema 校验后供服务端启动期加载。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  内容编辑   │────▶│  Schema校验 │────▶│  文件导出   │────▶│  服务端加载 │
│  (可视化UI) │     │  (类型检查) │     │  (JSON)     │     │  (启动期)   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `config-editor/src/pages/monsters/` | 怪物配置编辑页 |
| `config-editor/src/pages/techniques/` | 技能/功法配置编辑页 |
| `config-editor/src/pages/maps/` | 地图配置编辑页 |
| `config-editor/src/pages/files/` | 文件管理页 |
| `config-editor/src/pages/service/` | 服务配置页 |
| `config-editor/src/lib/api.ts` | API 通信层 |
| `config-editor/src/lib/format.ts` | 格式化工具 |

## 编辑页面

| 页面 | 编辑内容 |
|------|----------|
| monsters | 怪物模板（属性、技能、掉落、AI 参数） |
| techniques | 技能/功法模板（效果、消耗、冷却） |
| maps | 地图模板（地形、NPC、传送点、刷怪点） |
| files | 配置文件管理（导入/导出/版本） |
| service | 服务端运行时配置 |

## 编辑流程

### 1. 内容编辑

```
编辑器 UI
  │
  ├─▶ 表单编辑（属性、数值、文本）
  ├─▶ 实时预览
  ├─▶ 草稿保存（本地）
  └─▶ 关联引用检查
```

### 2. Schema 校验

```
提交前校验
  │
  ├─▶ 类型校验（shared schema）
  ├─▶ 引用完整性（物品ID、技能ID 存在性）
  ├─▶ 数值范围检查
  └─▶ 错误提示
```

### 3. 文件导出

```
校验通过后
  │
  ├─▶ 序列化为 JSON
  ├─▶ 写入 packages/server/data/content/
  └─▶ 版本标记
```

### 4. 服务端加载

```
服务端启动期
  │
  ├─▶ 读取 data/content/ 下的 JSON 文件
  ├─▶ 解析并注册到各 TemplateRegistry
  │     - ItemTemplateRegistry
  │     - MonsterTemplateRegistry
  │     - TechniqueTemplateRegistry
  │     - 等等
  ├─▶ Object.freeze 冻结模板
  └─▶ 运行期只读访问
```

## 关键约束

- **编辑器阶段校验**: 内容错误尽量在编辑器/导入期暴露
- **schema 一致**: 编辑器、shared schema、服务端加载保持同一契约
- **草稿与发布分离**: 编辑器草稿和正式运行配置区分真源
- **启动期解析**: 配置解析在启动期完成，运行期直接读预解析结构
- **可审计**: 影响玩家资产的配置变更可追溯
