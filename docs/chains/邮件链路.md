# 邮件链路

## 概述

邮件链路负责游戏内邮件的发送、接收、附件领取和过期清理。邮件是玩家间和系统向玩家传递物品/通知的核心渠道。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  发送邮件   │────▶│  邮箱存储   │────▶│  领取附件   │────▶│  过期清理   │
│  (系统/GM)  │     │  (内存+PG)  │     │  (Durable)  │     │  (定时归档) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/mail/mail-runtime.service.ts` | 邮件运行时核心 |
| `runtime/mail/mail-runtime.types.ts` | 邮件类型定义 |
| `persistence/mail-persistence.service.ts` | 邮件持久化 |

## 详细流程

### 1. 发送邮件

```
MailRuntimeService.createDirectMail() / appendMail()
  │
  ├─▶ 生成 mailId（playerId + 时间戳 + revision 编码）
  ├─▶ 构建邮件对象（标题、正文、附件列表）
  ├─▶ 插入玩家邮箱内存
  └─▶ persistMailboxMutation() 增量 upsert 到 PostgreSQL
```

### 2. 欢迎邮件

```
MailRuntimeService.ensureWelcomeMail()
  │
  ├─▶ 检查 welcomeMailDeliveredAt
  ├─▶ 首次登录自动发送
  │     - 模板: mail.welcome.v1
  │     - 附件: 2×小回复丹 + 8×灵石
  └─▶ 标记已发送
```

### 3. 读信与标记已读

```
markRead(playerId, mailId)
  │
  ├─▶ 设置 firstSeenAt / readAt
  ├─▶ 递增 mailVersion 和 revision
  └─▶ 持久化
```

### 4. 领取附件

```
claimAttachments(playerId, mailId)
  │
  ├─▶ 校验背包容量
  ├─▶ DurableOperationService 强事务
  │     - session fencing（runtimeOwnerId + sessionEpoch）
  │     - 物品入背包 / 灵石入钱包
  │     - 标记 claimedAt
  └─▶ 持久化
```

### 5. 删除邮件

```
deleteMails(playerId, mailIds)
  │
  ├─▶ 校验：必须已领取附件
  ├─▶ 设置 deletedAt（软删）
  └─▶ 持久化
```

### 6. 过期清理

```
cleanupExpiredMails()
  │
  ├─▶ 批量软删过期邮件
  ├─▶ 归档到 player_mail_archive 表
  └─▶ 30 天后物理清除（purgeSoftDeletedMails）
```

## 数据结构

### PostgreSQL 表

| 表名 | 用途 |
|------|------|
| `player_mail` | 邮件主表 |
| `player_mail_attachment` | 附件表 |
| `player_mail_counter` | 计数器 |
| `player_recovery_watermark` | 恢复水位 |
| `player_mail_archive` | 归档表 |
| `player_mail_attachment_archive` | 附件归档 |

## 关键约束

- **串行化**: 同玩家邮箱写操作通过 `runSerializedMailboxWrite` Promise 链串行
- **互斥锁**: `pg_advisory_xact_lock(7101, hashtext(playerId))` 保证数据库层互斥
- **缓存上限**: 内存 LRU 缓存最多 5000 玩家邮箱
- **附件领取**: 走 DurableOperation + session fencing，防跨节点重复领取
- **删除约束**: 有未领取附件的邮件不允许删除
- **重试**: 写入失败自动重试（最多 3 次，仅死锁/序列化失败）
