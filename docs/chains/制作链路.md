# 制作链路

## 概述

制作链路涵盖炼丹、炼器（锻造）和装备强化三大制作系统的完整流程。所有制作任务通过统一的 TechniqueActivityPipeline 管线管理生命周期，在 tick 中逐息推进，支持中断、暂停、队列排队和自动恢复。

## 链路流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  发起制作   │────▶│  资源校验   │────▶│  Tick 推进  │────▶│  产出结算   │
│  (意图提交) │     │  材料扣除   │     │  (逐息进度) │     │  (成功/失败)│
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
       │                                       │
       ▼                                       ▼
┌─────────────┐                        ┌─────────────┐
│  队列排队   │                        │  中断暂停   │
│  (已有任务) │                        │  (10息恢复) │
└─────────────┘                        └─────────────┘
```

## 核心文件

| 文件 | 职责 |
|------|------|
| `runtime/craft/craft-panel-runtime.service.ts` | 核心引擎：任务创建、tick 推进、结算 |
| `runtime/craft/pipeline/technique-activity-pipeline.service.ts` | 管线骨架：统一 start/tick/interrupt/cancel |
| `runtime/craft/pipeline/technique-activity-queue.service.ts` | 队列管理：排队、休眠、唤醒 |
| `runtime/world/world-runtime-alchemy.service.ts` | 炼丹/炼器写路径门面 |
| `runtime/world/world-runtime-enhancement.service.ts` | 强化写路径门面 |
| `runtime/world/world-runtime-craft-tick.service.ts` | Tick 编排器：每帧分发 |
| `runtime/world/world-runtime-craft-interrupt.service.ts` | 统一中断器 |
| `runtime/world/world-runtime-craft-mutation.service.ts` | Mutation 刷新：面板推送、持久化 |

## 策略模式

每种技艺实现 `TechniqueActivityStrategy` 接口：

| 策略 | kind | pauseTicks | 说明 |
|------|------|-----------|------|
| AlchemyStrategy | alchemy | 10 | 炼丹 |
| ForgingStrategy | forging | 10 | 炼器 |
| EnhancementStrategy | enhancement | 10 | 强化 |
| GatherStrategy | gather | 0 | 采集（条件型） |
| BuildingStrategy | building | 0 | 建造（条件型） |

## 炼丹流程

### 1. 发起炼丹

```
WorldRuntimeAlchemyService.dispatchStartAlchemy()
  │
  ├─▶ CraftPanelRuntimeService.startAlchemy()
  │     - 查找丹方（alchemyCatalog）
  │     - 校验材料选择
  │     - 若已有活跃任务 → 入队列
  │     - 扣除材料 + 灵石
  │     - 计算成功率、单炉时间
  │     - 创建 alchemyJob，phase='preparing'
  │
  └─▶ flushCraftMutation()（面板推送 + 持久化）
```

### 2. Tick 推进

```
phase 状态机：
  preparing → (preparationTicks 耗尽) → brewing → (每炉结算) → 完成/下一炉
  任何 phase → (中断) → paused → (pausedTicks 耗尽) → 恢复原 phase
```

### 3. 单炉结算

- 按 `successRate` 随机判定每件产出
- 成功：物品入背包（满则掉地）
- 计算技能经验
- 全部完成：清除 job，启动队列下一任务

## 强化流程

### 1. 发起强化

```
WorldRuntimeEnhancementService.dispatchStartEnhancement()
  │
  ├─▶ 校验：类型=equipment、未达上限(20)、保护物合法
  ├─▶ 扣材料 + 灵石
  ├─▶ 装备提取 → 分配 itemInstanceId → 锁入 lockedItems（escrow）
  ├─▶ 创建 enhancementJob，phase='enhancing'
  └─▶ 记录强化历史
```

### 2. 强化结算

```
remainingTicks = 0 时结算：
  │
  ├─▶ Math.random() < successRate → 成功：等级 +1
  ├─▶ 失败 + 有保护：消耗保护物，等级 -1（不低于 0）
  ├─▶ 失败 + 无保护：等级归零
  │
  ├─▶ 未达目标等级 → advanceEnhancementJob 继续下一阶
  └─▶ 达到目标 → finishEnhancementJob 解锁装备归还
```

## 中断机制

**触发条件**：移动、出手、打坐、手动取消

```
WorldRuntimeCraftInterruptService.interruptCraftForReason()
  │
  ├─▶ 非条件型（炼丹/炼器/强化）：
  │     phase → 'paused'，增加 10 息暂停
  │     下次 tick 递减 pausedTicks，归零后恢复
  │
  └─▶ 条件型（采集/建造）：
        直接清理 job → sleepToQueue() 休眠入队列尾部
```

## 关键约束

- **权威**: 所有结算由服务端执行
- **频率**: 每 tick 推进一息
- **持久化**: 强化通过 DurableOperation 保证原子性
- **锁定**: 强化中装备进入 escrow，不可交易/使用/卸下
- **队列**: 已有活跃任务时新任务自动入队
- **材料**: 发起时扣除，不在结算时扣除
