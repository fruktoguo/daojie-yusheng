# 《道劫余生》客户端 UI 美学与高频交互重构白皮书

本白皮书针对《道劫余生》当前客户端（`packages/client`）在视觉呈现、响应式布局、高频交互以及性能同步上存在的“不够美观”、“反馈迟滞”、“缺乏微动效”等问题，进行了全方位、深层源码级的检索与剖析，并基于“商业级 MMO 研发标准”和“Wow 级视觉美学要求”制定了详细的重构与美化方案。

---

## 一、 宏观视觉主轴与设计哲学

《道劫余生》的核心视觉基调是 **“水墨宣纸修仙风”**（通过 `tokens.css` 中的 `--paper-bg`、`--wash-ink` 及 `Ma Shan Zheng`、`Zhi Mang Xing` 毛笔字体现）。然而，目前的实现手段较为单一，偏向传统的扁平化网页排版，导致视觉质感较为生硬、冰冷，缺乏“仙凡有别”的灵动意境与沉浸感。

### 1. 色彩空间的微调与高雅化
当前 `--stamp-red` (`#c53c3c`)、`--success-ink` (`#2e6d46`) 等颜色偏向网页设计的“通用红绿”，略显俗气。
*   **优化建议**：引入符合中国传统色彩学的优雅 HSL 配色方案。
    *   **朱砂红（印章色）**：从 `#c53c3c` 升级为 `hsla(356, 65%, 45%, 0.95)`，色彩更具饱和沉淀感。
    *   **青黛/松石绿（成功/正面状态）**：从 `#2e6d46` 升级为 `hsla(168, 55%, 32%, 0.95)`，温润如玉。
    *   **太虚金（特殊/高贵）**：引入 `hsla(38, 75%, 55%, 0.95)` 作为高阶境界或珍稀装备的宝光色。

### 2. 质感升华：磨砂宣纸（Glassmorphism & Ink Splash）
*   **优化建议**：不再采用单纯透明或全实色的面板底色。利用现代 CSS 混合模式与毛玻璃滤镜，打造出“宣纸半透，水墨晕染”的磨砂质感：
    ```css
    .layout-section {
      background: var(--panel-shell-bg);
      backdrop-filter: blur(12px) saturate(180%);
      border: var(--panel-frame-border);
      box-shadow: var(--panel-shell-shadow);
      /* 引入微弱的宣纸纤维纹理底图，采用 multiply 混合模式 */
      background-image: url('/public/assets/textures/paper-fiber.png');
      background-blend-mode: multiply;
    }
    ```

## 二、 核心痛点检索清单与具体优化方案

### 1. 登录与注册模块（`login-overlay` / `index.html#L13-83`）
*   **当前问题**：
    *   表单输入框与“入”字印章（`login-stamp`）为纯静态，且登录/注册选项卡（`login-mode-tabs`）切换极为生硬（直接 `hidden` 替换），毫无仙侠开门之仪式感。
    *   第三方链接按钮（QQ、GitHub）在登录框底部显得格格不入。
*   **具体重构方案**：
    *   **墨迹晕染仪式感**：在 `login-overlay` 背景中加入一个低消耗的 Canvas 粒子特效，模拟缓慢飘动的“天地灵气”与“水墨雾霭”。
    *   **印章物理按压与光晕**：为 `.login-stamp` 添加三维悬停物理缩进与水墨波纹扩散动画。
    *   **书页折叠过渡**：切换登录/注册时，采用 3D 书页翻转或 `clip-path` 渐变动画，避免生硬的 display 切换。
    ```css
    /* 登录印章微动效 */
    .login-stamp {
      transition: transform 0.3s cubic-bezier(0.25, 1, 0.5, 1), box-shadow 0.3s ease;
      cursor: pointer;
    }
    .login-stamp:hover {
      transform: scale(1.05) translateZ(5px);
      filter: drop-shadow(0 0 8px var(--stamp-red));
    }
    .login-stamp:active {
      transform: scale(0.95);
    }
    ```

### 2. 主框架折叠轨道与拐角修饰（`layout.css#L70-176`）
*   **当前问题**：
    *   `.layout-section::before` 与 `::after` 目前只是用普通 `border` 拼凑的 `18px` L型黑色硬直角线，缺乏艺术设计感。
    *   侧栏折叠按钮（`.layout-collapse-toggle`）采用简单的字符 `<` 和 `>`，且折叠时侧栏内部元素直接瞬间消失，布局过渡极其冷硬。
*   **具体重构方案**：
    *   **古典拐角修饰**：将 L 型线框替换为精致的“云纹/如意纹”或“太极阴阳拐角”SVG 遮罩，让每个面板边缘宛如一幅装裱精细的字画。
    *   **如意挂件折叠钮**：将折叠按钮改造成挂在侧栏边缘的“竹简玉佩”或“如意玉扣”悬浮图标，悬停时产生温润的发光和轻微摆动。
    *   **视差伸缩动画**：折叠侧栏时，内部子元素同步执行 `opacity: 0 -> 1` 且配合 `transform: translateX(-20px) -> 0` 的视差滑动，极大缓和视觉冲击。

### 3. HUD 数值进度条与境界突破（`hud.css`）
*   **当前问题**：
    *   修为、生命值、灵力进度条（`#hud-cultivate-bar`、`#hud-hp-bar`、`#hud-qi-bar`）均是死板的纯单色填充，无法体现修仙者体内“气血沸腾、灵力如潮”的生命流动感。
    *   境界突破（`#hud-breakthrough`）只在修为满时冷冰冰地取消 `hidden` 显现，毫无破境飞升的尊贵与震撼。
*   **具体重构方案**：
    *   **灵力流光进度条**：为所有进度条填充背景添加 CSS 渐变流光与气泡微动效：
    ```css
    .hud-resource-fill {
      background: linear-gradient(90deg, var(--stamp-red) 0%, #e06c75 100%);
      background-size: 200% 100%;
      animation: pulseFlow 3s infinite linear;
      position: relative;
      overflow: hidden;
    }
    .hud-resource-fill::after {
      content: '';
      position: absolute;
      inset: 0;
      background: linear-gradient(90deg, transparent, rgba(255, 255, 255, 0.25), transparent);
      transform: translateX(-100%);
      animation: barShine 2s infinite ease-in-out;
    }
    @keyframes pulseFlow {
      0% { background-position: 0% 50%; }
      100% { background-position: 200% 50%; }
    }
    @keyframes barShine {
      100% { transform: translateX(100%); }
    }
    ```
    *   **金丹环绕突破钮**：当修为满盈、可供突破时，突破按钮展示“金丹自转/五彩祥云环绕”的呼吸式发光特效；点击瞬间触发全屏轻微的“天劫震颤”和“墨迹喷溅”，并在破境成功后，境界文本（如“练气一层”）播放金光灌体淡入，极大提升爽感。

### 4. 属性雷达图与卡片交互（`attr-panel.ts#L582-794`）
*   **当前问题**：
    *   六维/灵根雷达图（`AttrRadarPaneSnapshot`）线条僵硬，顶点标志（Node Dots）只是普通圆点，无动态呼应。
    *   属性卡片（`.attr-mini`）过于扁平直白。
*   **具体重构方案**：
    *   **太极游鱼雷达顶点**：将雷达图的顶点圆点升级为旋转的“太极微标”或闪烁的“灵气火花”。
    *   **阴阳渐变填充**：雷达覆盖面积（`areaPoints`）使用半透明的古典五行渐变填充（如金用淡黄、水用幽蓝），并在悬停某个顶点时，对应的属性卡片和雷达分轴同步产生高亮和轻微的脉冲放大。
    *   **玉简式 3D 卡片**：当鼠标滑过属性卡片（`.attr-mini`）时，加入极小的 3D 偏转（CSS `perspective` & `rotateY`）与金色包边，营造出“翻阅仙家玉简”的厚重交互感。

### 5. 背包行囊格与悬浮提示框（`inventory-panel.ts`, `equipment-tooltip.ts`）
*   **当前问题**：
    *   背包格（`.inventory-cell`）中，凡、玄、地、天等不同阶别的顶级丹药或装备，视觉区别仅在于极细的彩边，整体堆砌感严重，无法彰显宝物的珍稀度。
    *   浮动提示框（`FloatingTooltip`）排版高度拥挤，海量文字信息一股脑塞入，重点属性（攻击、加成）缺乏视觉层级。
*   **具体重构方案**：
    *   **宝光四溢的物品背景**：根据物品阶别，为物品格子引入动态渐变宝光。
        *   *地阶/地级*：暗金磨砂底色，并带有微弱的金芒呼吸。
        *   *天阶/天级*：玄青渐变底色，并在格子周边环绕流动太极包边。
    *   **气泡扫光特效**：鼠标滑过稀有物品时，卡片内部闪过一道精致的斜向“扫光（Shine）”，反馈极佳。
    *   **Tooltip 视觉重构分区**：
        1. 头部引入典雅的“如意如歌”云纹水平分割线。
        2. 基础属性（如攻击力）字号放大 1.25 倍，加粗，使用“朱砂红”或“深褐色”突出。
        3. 附加百分比属性（如灵气吸收速率）包装为精致的圆角小标签，双列排版，减少纵向高度。
        4. 晦涩的机制计算详情默认折叠，点击“查看感悟”方可展开，保证主视觉界面的清爽。

### 6. 高频数据同步与 1Hz Tick 迟滞优化（高频交互红线）
*   **当前问题**：
    *   在坊市买卖、淬炼装备、技能栏切换等高频操作中，由于服务端采用 `1Hz`（一秒一息）的调度，客户端在点击按钮后，到服务端回包并重新渲染 DOM 之间，有长达 0.5 秒 - 1.2 秒的“交互真空区”。在此期间，按钮无反应、界面死板，给玩家带来极强的卡顿与迟滞感。
    *   更新时采用 `replaceChildren` / `replaceElementHtml` 重新拼装整个 DOM 列表，导致页面发生闪烁（Flicker），重置了滚动条和选区，打断了玩家的操作流。
*   **具体重构方案**：
    *   **乐观 UI 与意图悬挂状态（Optimistic UI & Intention Pending）**：
        当玩家点击“突破”、“淬炼”或“坊市购买”时，客户端立刻执行**前台乐观预测表现**：
        1. 按钮立刻缩下并进入“气旋凝聚/太极流转”的 pending 状态，屏蔽二次点击。
        2. 物品格子呈现轻微淡出或被微弱金锁链锁定的等待态，给予玩家直观的“意图已传达，正在推演天机”的物理反馈。
        3. 若 1.5 秒内未收到网络更新，看门狗计时器触发自动回滚并弹出优雅的“天机阻滞”飘字通知。这能够完美消化 1Hz tick 固有的延迟感。
    *   **局部补丁更新（Fine-grained DOM Patching）**：
        放弃整体 `replaceChildren`。通过记录组件 key，只对属性数值、物品数量等具体 DOM 的 `innerText` 进行定向修改；对于列表数据，只 patch 新增/删除的节点，确保玩家在滚动或输入时的焦点绝对不被打断。
    *   **骨架屏（Skeleton Screen）过渡**：
        在坊市加载或大列表更新期间，弃用粗暴的“加载中...”文字，展示具有水墨宣纸渐消动画的骨架占位图，体验丝滑无比。

## 三、 炼丹强化面板与 NPC 对话系统交互美化

### 1. 太极八卦炉微动效与破境炸裂表现（`craft-alchemy-view.ts`、`craft-enhancement-view.ts`）
*   **当前问题**：炼丹与淬炼面板目前只是僵硬的灰黑卡片加普通的 HTML `width` 进度条，几乎没有任何图形化反馈，对于“丹成、法器晋升”或“炉爆、灵气碎裂”这样刺激、心跳的游戏瞬间缺乏表现力。
*   **重构方案**：
    *   **太极八卦炉旋转微动效**：将工作台中央的进度条重构为环状旋转的“太极八卦炼丹炉”。当炼制处于 `brewing` 状态时，炉周环绕旋转的五彩灵子流光，速度与当前的 `brewTicks` 挂钩；火候合理时炉鼎绽放出半透明金色微光。
    *   **成丹/淬炼成功的震撼动效**：炼制或淬炼成功的瞬间，在面板正中央触发带有全屏水墨震颤与“金丹成型，霞光瑞气”的超高画质 CSS Clip-Path 成功印记；若失败，则播放“炉鼎开裂，烟雾缭绕”的震动及焦黑消融动效，赋予玩法极强的感官心跳刺激。

### 2. 水墨坊市商店与 NPC 灵动立绘对话（`npc-shop-modal.ts`、`npc-quest-modal.ts`）
*   **当前问题**：NPC 任务、商店、对话弹窗为冷直的双栏信息表，NPC 仅为一段无情感的文字描述，与大世界探索的仙侠沉浸感严重割裂。
*   **重构方案**：
    *   **宣纸画轴半身立绘（Portrait Mask Overlay）**：右侧详情底色采用半透宣纸图层，其背景通过 CSS `mask-image` 与淡入动画，隐约展露出该 NPC 特有的水墨风半身立绘，如同在画卷中慢慢浮现。
    *   **毛笔蘸墨打字机动效**：NPC 的对话文字在展现时，应用带有微弱毛刷边缘的毛笔书法打字机动效（Calligraphy Text Spreader），文字字迹如毛笔蘸墨在宣纸上渐次晕染渗出，并辅以极其轻微的“玉简碰撞”音效，极大充实叙事沉浸感。

## 四、 Canvas 游戏场景地图水墨化与动态美化建议

游戏地图采用 Canvas 渲染（`#game-canvas`）。当前地图虽然实用，但地形格子和边界相对硬朗直白，缺乏修仙者“神识扫过，天地灵气流转”的奇幻和唯美。

### 1. 迷雾水墨晕染滤镜（Ink Bleeding Fog of War）
*   **当前问题**：未探索区域的战争迷雾是冰冷的直角黑色/半透明灰色块，交界处呈生硬的马赛克状，与“太虚修真”的主题极其割裂。
*   **重构方案**：
    *   在 Canvas 渲染迷雾交界处时，引入**水墨墨迹晕染（Ink Bleeding）**算法。
    *   利用一个小的离屏 Canvas（Offscreen Canvas）生成 Perlin 动态噪声遮罩（Noise Mask），让迷雾边缘渲染成如同在宣纸上自然墨水洇湿、扩散、不规则晕染的柔和边缘，仙气凛然。

### 2. 天地五行灵气微粒系统（Aura Particles System）
*   **当前问题**：灵气浓度（Qi Density/Efficiency）只能通过数值卡片读取，地图场景死板，无法从视觉上“看到”福地洞天和灵泉的灵气流动。
*   **重构方案**：
    *   在 Canvas 地图上层叠加一层超低能耗、柔和飘动的 **天地微粒系统（Particle System）**。
    *   当玩家行至“聚灵阵”、“灵脉深处”或“名山洞府”时，微光粒子（灵子）呈现汇聚并缓慢向上螺旋升腾的微光动效（例如地灵之气用赭石金，木系灵力用幽碧翡翠绿）；
    *   当玩家使用心法吐纳吸纳灵力时，周围 5x5 的灵子粒子平滑向角色中心汇聚淡出，完美实现视觉上的“吸纳天地之造化”。

### 3. 时辰昼夜交替与玄虚天气系统（Circadian & Weather Mask）
*   **当前问题**：虽然系统有严格的时辰变化（子、丑、寅、卯等十二时辰），但主 Canvas 地图光影完全恒定，毫无时间流转的史诗感。
*   **重构方案**：
    *   根据服务端同步的当前时辰（由 `map-current-time` 获取），为 Canvas 地图应用一层温润、平滑过渡的 HSL 混合滤镜。
    *   **子夜（深夜）**：地图整体笼罩在玄青/松石暗蓝滤镜中（`hsla(210, 40%, 12%, 0.45)`），玩家视野半径缩减，且篝火和聚灵阵散发出暖橘发光效果（使用 Canvas `globalCompositeOperation = 'screen'` 进行发光混合）。
    *   **正午（白昼）**：洒下轻微的曦黄微光（`hsla(45, 60%, 95%, 0.12)`），树影与山岩的投影随太阳角度微调，极大增强真实感。

## 五、 多端融合与响应式精细化设计

### 1. 手机端触控热区（Safe-Area & Touch Hit）
*   **当前问题**：
    *   在手机模式下，底部的 `.mobile-ui-tab-bar` 在全面屏手机上极易被底部手势横条遮挡，且高频功能切换按钮热区偏小，误触率高。
*   **具体重构方案**：
    *   **安全区适配**：利用 CSS 的 `env(safe-area-inset-bottom)` 全面适配刘海屏与底部手势区。
    *   **44px 黄金触控标准**：所有手机版页签和核心动作按钮（如“战斗settings”中的使用、替换）的热区均扩展至最小 `44px * 44px`，并增大按钮之间的 `gap`。

### 2. 滑动手势与局部惯性滚动
*   **当前问题**：
    *   手机模式在各个面板（如行囊、行动）之间只能硬性点击底部 Tab，不支持滑动手势，操作效率低。
*   **具体重构方案**：
    *   引入无开销的原生 CSS 触控滑块体验（`scroll-snap-type: x mandatory`），支持左右轻扫手势直接在“概览 -> 属性 -> 世界 -> 行囊 -> 行动”之间实现视差滑动切页。
    *   所有长列表强制开启 iOS 原生流畅滚动质感：
    ```css
    .tab-content, .section-body {
      overflow-y: auto;
      -webkit-overflow-scrolling: touch; /* 惯性滚动 */
    }
    ```

## 六、 总结与落地方案步骤

本白皮书提出的视觉与交互升级方案，遵循**不改变服务端权威数据流**和**不修改现有游戏机制协议**的红线原则，百分之百在客户端（`packages/client`）表现板中通过 CSS 与轻量级原生 TypeScript 状态机完成。

### 建议落地三步走：
1.  **第一步：底层色彩与 UI 质感加固**：
