# 测试服导入存档后启动恢复 IO 爆炸

## 现象

- 测试服 `yuohira` 在导入存档后重启，启动数分钟内 SSH 连接可短暂建立，随后 22 端口 banner 阶段超时。
- 云监控显示系统盘读流量、读 IOPS、IO 繁忙比率从 2026-05-22 17:00 左右持续飙升，IO busy 长时间接近满载。
- Docker/containerd 被 mask 后宿主机恢复稳定；恢复 Docker Swarm 后服务启动链路会再次拖垮机器。
- 单独启动 PostgreSQL 和 Redis 稳定，数据库体量约 180 MB，不是数据库进程独立启动即爆。
- 单独启动 API 诊断容器后，容器内存和 Block I/O 在短时间内暴涨，`/live` 连接 reset。

## 关键数据

- `player_flush_ledger` 行数约 1429，总大小约 527.8 MB，实际数据约 2.9 MB，索引约 35.7 MB，主要空间来自历史 payload/TOAST 膨胀。
- `instance_flush_ledger` 行数约 780，总大小约 25.0 MB。
- `player_inventory_item` 行数约 31235，总大小约 45.4 MB。
- `server_market_order` 行数约 19173，总大小约 38.8 MB。
- 实例目录规模约 247，离线/恢复账本规模不大，但启动期会按实例逐个执行分域恢复。

## 根因判断

这次事故不是单一的 SSH 防火墙或端口扫描问题，而是服务启动链路在导入存档后同时命中两个放大器。

1. 普通 API 启动调用 `rebuildPersistentRuntimeAfterRestore({ restoreOfflinePlayers: false })`，该路径原本更接近“导入数据库后全量重建”的恢复语义。它会扫描实例目录、重置运行时、创建默认公共实例，并对所有持久实例调用 `restorePublicInstancePersistence()`，逐实例读取 tile/resource/damage/temporary/ground/container/monster/event/overlay/building/room/fengshui/checkpoint/formation 等分域数据。
2. flush ledger 消费默认批量过大。`FlushTaskRuntimeService` 默认 player/instance 各最多 claim 1024 条，player/instance 并行度各 16。导入存档后 ledger 表带着数百 MB 历史 payload/TOAST 膨胀时，一旦 worker 或 inline consumer 开始消费，会把磁盘 IO 压力继续放大。

因此，`ledger` 积压是重要风险，但不是唯一启动早期爆点。必须同时修复普通启动的全量实例恢复和 backlog 消费节流。

## 临时处置

- mask Docker/containerd，阻止重启后自动恢复 Swarm。
- 临时挪走 `/var/lib/docker/swarm`，让 Docker 以非 Swarm 状态启动，避免 stack 自动拉起。
- 停止 `daojie-ccr-auto-update.timer`，避免自动更新器反复尝试恢复容器。

## 修复目标

- 普通服务启动默认走轻量恢复：只重建运行时基础结构、默认公共实例、必要的实例目录 claim/lease，不在 API 启动阶段全量 hydrate 所有持久实例分域。
- 显式数据库导入/恢复路径保留全量恢复能力，但需要通过明确调用或环境开关进入，避免普通重启误用。
- flush ledger 默认消费上限必须生产友好：小批量、低并发、可通过环境变量显式提高。
- 保留持久化真源和恢复能力，不把地图实例、玩家资产或运营数据转移到内存临时态。

## 验证计划

- 扩展启动生命周期 smoke，证明普通启动传入轻量恢复模式。
- 扩展世界运行时生命周期 smoke，证明轻量启动不调用 `restorePublicInstancePersistence()`，显式恢复仍调用。
- 编译 `@mud/server`，确保 TypeScript 主线可构建。

## 线上恢复建议

- 修复镜像发布前不要恢复 Swarm 自动启动。
- 发布后先以单 API 诊断方式验证 `/live` 和 `/ready`，观察磁盘读 IOPS、IO busy、容器内存，再恢复 Swarm。
- ledger 历史空间应通过已有 flush-ledger-retention 或专用清理链清理 completed rows，不应直接清理真实落盘数据表。
