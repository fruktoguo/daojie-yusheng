# 引导内容定制

本文档用于持续收集和整理游戏内“引导”内容，面向玩家首次接触、功能解锁、阶段目标和操作路径。引导强调“下一步做什么、为什么做、做到什么程度”，不承担百科条目索引职责。

## 维护口径

- 引导内容以“低打扰、少文字、明确行动”为目标。
- 每段引导优先写清：触发条件、玩家目标、操作步骤、完成条件、失败或跳过处理。
- 新手引导不直接授予未经服务端确认的资源、位置、战斗结果或资产状态。
- 客户端只展示引导状态和操作提示；涉及奖励、任务进度、解锁条件时以服务端权威为准。
- 与百科重复的解释只保留必要短句，详细说明链接到百科条目。

## 引导状态

| 状态 | 含义 |
|------|------|
| 待补充 | 仅有引导方向，缺少流程 |
| 草案 | 已有流程文本，但未确认触发与完成条件 |
| 已确认 | 流程、条件、文案口径已确认 |
| 需对齐实现 | 设计已定，需要核对代码、配置或协议 |

```markdown
- 所属阶段：
- 触发条件：
- 前置条件：
- 完成条件：
- 可跳过：是 / 否
- 关联百科条目：

#### 玩家目标

#### 引导步骤

1. 
2. 
3. 

#### 屏幕提示文案

#### 奖励或解锁

#### 异常与回退

#### 待确认问题
```
