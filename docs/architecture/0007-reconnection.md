# ADR-0007: 断线重连机制

- **状态**: 已采纳
- **日期**: 2025-02-01

## 决策

实现客户端自动重连 + 服务端状态保持机制，短时间断线后可无缝恢复。

## 核心参数

| 参数 | 值 |
|------|-----|
| 心跳间隔 | 5s |
| 心跳超时 | 15s |
| 会话保持时间 | 5min |
| 最大重连尝试 | 5 次 |
| 重连间隔策略 | 指数退避 1s → 2s → 4s → 8s → 16s |

## 断线期间服务端行为

- 玩家运行时保持在内存，不立即销毁
- 战斗中断线：AI 托管（自动防御/逃跑）
- 非战斗断线：角色原地保持
- 会话超时后：执行正式下线流程（flush 数据、释放资源）

## 重连流程

1. 客户端检测断线，开始自动重连
2. 重连成功后发送 session token
3. 服务端验证 token，恢复会话
4. 发送当前完整状态（首包）
5. 恢复正常同步

## 约束

- 重连期间不允许其他客户端使用同一账号登录（踢旧连接）
- 重连后必须重发完整视野首包
- 战斗 AI 托管不能做出玩家不可能做的操作
- 会话保持期间玩家仍占用地图实例资源

## 验证方式

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| 断线率 | < 1% | > 5% |
| 重连成功率 | > 95% | < 80% |
| 平均重连时间 | < 3s | > 10s |
| 会话超时率 | < 5% | > 20% |
