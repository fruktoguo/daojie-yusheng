# 持久化设计

本文说明 `packages/server` 主线的持久化调度、真源边界、刷盘域拆分和当前未收口风险。表结构清单已拆到 [持久化表结构现状.md](./持久化表结构现状.md)，本文件不再重复铺完整字段。

## 2026-04-28 目标口径：数据层一次性商业级硬切

当前计划已从“兼容迁移、旧链路兜底、逐步弱化”调整为“一次性硬切”。本设计文档同时采用以下目标口径：

- 正式运行时只认 PostgreSQL 结构化表和明确的 checkpoint 表为持久化真源。
- `persistent_documents` 不再承担任何业务真源；旧 scope 只能作为停服前离线导出或转换输入。
- `server_player_snapshot`、旧地图快照、旧邮箱 JSON、市场兼容文档不再作为恢复、checkpoint、fallback 或双写镜像。
- durable 条件不满足时，资产类命令必须失败，不允许 direct runtime fallback。
- 高频实例域只允许通过 ledger worker 写 row-level/key-level delta；周期服务不能常规刷高频域。
- 分域恢复失败要显式失败或阻断启动，不能静默回退 starter snapshot、fallback 出生点或旧 JSON。

统计口径：

- 当前主线只看 `packages/server`，不把 `legacy/*` 当运行时代码。
- 数据库真源来自 `packages/server/src/persistence/**`、实例运行时 domain flush 入口、durable operation 和低频业务专表。
- 本轮按代码路径、静态门禁和重点 with-db smoke 整理；完整商用验收仍要继续跑 `release gate:with-db / shadow / acceptance / full`、迁移演练和容量压测。

## 一致性等级

| 等级 | 内容 | 要求 |
|------|------|------|
| L0 静态 | 地图/怪物/物品/NPC/任务模板 | 不在运行时修改，不参与在线刷盘 |
| L1 强持久化 | 货币、背包、装备、邮件领取、市场成交、任务奖励、位置迁移、实例所有权、GM 写操作 | 成功返回前必须提交；带幂等 operation_id；有审计日志 |
| L2 最终一致 | vitals、quest 进度、technique、buff、设置、地块资源、地面掉落、容器 | 1-5s 批量提交；允许 worker 合批；可恢复可重试 |
| L3 可重建 | 普通怪物、临时特效、AOI cache、路径缓存 | 不入正式真源；重启后重建 |

## 总体口径

持久化系统按“状态拥有者 + 写入强度 + 恢复要求”拆分：

| 层级 | 真源 | 运行时副本 | 写入方式 |
| --- | --- | --- | --- |
| 账号/玩家身份 | PostgreSQL `server_player_identity / player_identity` | auth store / runtime session | 创建、登录、迁移时直接写 |
| 玩家快照兜底 | 无正式真源角色 | 无 | 硬切后删除恢复和 checkpoint 用途 |
| 玩家分域 | PostgreSQL `player_*` | 玩家 runtime 状态 | domain save / durable operation |
| 邮件和资产操作 | PostgreSQL `player_mail* / durable_operation_log / outbox_event / asset_audit_log*` | 邮件服务、outbox dispatcher | 事务内强一致写 |
| 地图实例目录 | PostgreSQL `instance_catalog` | `WorldRuntimeService` 实例注册表 | 租约、归属、生命周期直接写 |
| 地图实例分域 | PostgreSQL `instance_*` | `MapInstanceRuntime` | dirty domain 后台刷盘 |
| 低频业务态 | PostgreSQL 专表 | 对应服务内存缓存 | 低频结构化 upsert |

硬边界：

- 需要“下次还在”的状态，正式真源必须是 PostgreSQL。
- Redis、内存、Socket 投影只能是在线态、缓存或调度辅助，不是最终真源。
- 高频运行态不能回到“大 JSON 整包每轮写库”的模式。
- 玩家/实例的 dirty 域必须按域回标，不能一个域落库成功后清空整实例 dirty。
- 旧快照、旧文档、旧 fallback 只允许存在于离线迁移工具，不允许成为正式 runtime provider。

## 当前调度拓扑

```text
运行时变更
  -> markPersistenceDirtyDomains / markPersistenceDirtyDomains(player)
  -> 内存 dirty domain
  -> interval flush 或 domain worker
  -> PostgreSQL 分域表
  -> recovery watermark / flush ledger 回标
```

当前实例持久化同时存在两条调度链；硬切后的正式部署口径是 worker 为高频域主路径，周期链只做低频 checkpoint、shutdown force 和诊断兜底：

| 调度链 | 入口 | 用途 |
| --- | --- | --- |
| 周期兜底刷盘 | `MapPersistenceFlushService` | 5 秒扫描 dirty 实例，按域写入结构化表；对 `time`、`monster_runtime` 额外降频；正式容量环境应继续收窄为 checkpoint/shutdown/diagnostic |
| ledger worker | `Instance*FlushWorker` + `FlushLedgerService` | 把具体域写入 `instance_flush_ledger`，worker claim 后独立刷盘 |

当前已经修复实例 worker 认领非零 `ownership_epoch` 的问题，worker claim 不再固定卡在 epoch 0；claim 后仍会用 runtime 当前 `ownershipEpoch` 做写入前校验。

## MapPersistenceFlushService

主周期配置来自 `packages/server/src/persistence/map-persistence-flush.service.ts`：

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| `MAP_PERSISTENCE_FLUSH_INTERVAL_MS` | `5000` | interval 刷盘周期 |
| `MAP_PERSISTENCE_FLUSH_BATCH_SIZE` | `16` | 每批实例数 |
| `MAP_PERSISTENCE_FLUSH_PARALLELISM` | `3` | 批内并发 |
| `SERVER_PERSISTENCE_FLUSH_SLOW_THRESHOLD_MS` | `100`，最小 100 | 普通慢刷盘告警阈值 |
| `SERVER_PERSISTENCE_FLUSH_SLOW_BACKOFF_MS` | `5000`，最小 1000 | 慢刷后退避 |
| `SERVER_MAP_TIME_CHECKPOINT_INTERVAL_MS` | `300000` | `time` checkpoint interval 降频 |
| `SERVER_MAP_TIME_CHECKPOINT_FLUSH_BATCH_SIZE` | `16` | 每轮 interval 最多刷多少个到期 `time` checkpoint，避免 100+ 实例同轮集中写库 |
| `SERVER_MAP_MONSTER_RUNTIME_FLUSH_INTERVAL_MS` | `60000` | `monster_runtime` interval 降频 |
| `SERVER_MAP_MONSTER_RUNTIME_SLOW_THRESHOLD_MS` | `1000` | 包含 `monster_runtime` 时的慢刷阈值 |

关键行为：

- interval 每 5 秒触发一次，但如果已有 `flushPromise`、恢复冻结、退避窗口未结束，会跳过。
- `time` 和 `monster_runtime` 在 interval 路径会被 `selectFlushDomainEntries` 过滤，未到期不刷；到期 `time` checkpoint 还会按批次摊开，默认每轮最多 16 个实例。
- shutdown / 手动 flush 不受 interval 降频限制，会尽量刷当前脏域。
- 慢刷告警里的 `durationMs` 是整轮 flush cycle 耗时，不是单条 SQL，也不是单个实例。
- 如果持久化域包含 `monster_runtime`，慢刷阈值会抬到 `SERVER_MAP_MONSTER_RUNTIME_SLOW_THRESHOLD_MS` 和普通阈值的较大值。

## 实例 dirty domain

实例运行时通过 `MapInstanceRuntime.markPersistenceDirtyDomains(...)` 标记脏域。当前主要域：

| domain | 触发来源 | 真源表 |
| --- | --- | --- |
| `tile_cell` | 临时/运行时地块类型变化 | `instance_tile_cell` |
| `tile_resource` | 地块资源变化 | `instance_tile_resource_state` |
| `tile_damage` | 地块受损、摧毁、复生状态变化 | `instance_tile_damage_state` |
| `temporary_tile` | 临时地块创建、变化、过期 | `instance_temporary_tile_state` |
| `ground_item` | 地面物品堆变化 | `instance_ground_item` |
| `container_state` | 容器条目、刷新计时、翻找状态变化 | `instance_container_state / entry / timer` |
| `overlay` | portal、规则、动态覆盖层变化 | `instance_overlay_chunk` |
| `monster_runtime` | 高价值怪物位置、生命、仇恨、复生、buff 等变化 | `instance_monster_runtime_state` |
| `time` | 实例 tick / 时间 checkpoint | `instance_checkpoint` |

`WorldRuntimePersistenceStateService.flushInstanceDomains(...)` 负责把 domain 分发到 `InstanceDomainPersistenceService` 的具体写入方法。落库成功后只回标本次成功的 domain，并写 `instance_recovery_watermark`。

如果出现未支持的新 domain，硬切后不得写入 `domain_fallback_checkpoint` 并继续运行；应直接暴露错误、阻断对应实例刷盘或阻断启动。`domain_fallback_checkpoint` 只可作为过渡期诊断信号，不是正式商用路径。

## Flush Ledger 与 Worker

`FlushLedgerService` 维护两个账本：

| 表 | 主键 | 用途 |
| --- | --- | --- |
| `player_flush_ledger` | `(player_id, domain)` | 玩家 domain 异步刷盘账本 |
| `instance_flush_ledger` | `(instance_id, domain, ownership_epoch)` | 实例 domain 异步刷盘账本 |

账本字段表达：

- `latest_version`：运行时看到的最新版本。
- `flushed_version`：已经完成持久化的版本。
- `dirty_since_at`：首次变脏时间。
- `next_attempt_at`：失败后下次允许 claim 的时间。
- `claimed_by / claim_until`：worker 租约。

实例 worker 当前模式：

| worker | domain | idle | claim limit | 当前定位 |
| --- | --- | --- | --- | --- |
| `InstanceResourceFlushWorker` | `tile_resource` | `2000ms` | `32` | 资源域独立刷盘 |
| `InstanceTileDamageFlushWorker` | `tile_damage` | `2000ms` | `32` | 地块损坏域独立刷盘 |
| `InstanceGroundItemFlushWorker` | `ground_item` | `2000ms` | `32` | 地面物品域独立刷盘 |
| `InstanceContainerFlushWorker` | `container_state` | `2000ms` | `32` | 容器域独立刷盘 |
| `InstanceOverlayFlushWorker` | `overlay` | `2000ms` | `32` | overlay 域独立刷盘 |
| `InstanceMonsterRuntimeFlushWorker` | `monster_runtime` | `2000ms` | `32` | 高价值怪物运行态独立刷盘 |

worker 每轮先把 runtime dirty domain upsert 到 ledger，再 claim 自己负责的 domain，随后调用 `worldRuntimeService.flushInstanceDomains(instanceId, [domain])`。claim 时会校验 `ownership_epoch`，避免旧节点或旧租约写入。

当前缺口：

- worker 和周期兜底仍可能同时观察同一 dirty domain，需要继续确认 wakeup / claim / 回标路径不会重复刷同一批。
- `monster_runtime` 应优先由 worker 或更低频机制处理，周期兜底只做 shutdown 和异常兜底。
- 阵法仍未纳入统一 `instance_flush_ledger` domain。

## 玩家分域持久化

玩家状态已经按 `player_*` 表拆分，真源不再依赖单一玩家大 JSON。重要域：

| domain | 真源表 | 说明 |
| --- | --- | --- |
| identity | `server_player_identity / player_identity` | 登录身份和玩家 ID 映射 |
| snapshot fallback | 无 | 硬切后删除；玩家恢复只从 `player_*` 分域装配 |
| presence | `player_presence` | 在线、世界内、session epoch、转移状态 |
| anchor / position | `player_world_anchor / player_position_checkpoint` | 复活点、安全点、当前位置 checkpoint |
| vitals / progression / attr | `player_vitals / player_progression_core / player_attr_state` | 生命、灵力、成长、属性、境界 |
| wallet | `player_wallet` | 钱包余额、冻结余额、版本 |
| inventory / equipment / market storage | `player_inventory_item / player_equipment_slot / player_market_storage_item` | 物品实例、装备、市场仓储 |
| skill / buff / quest / preferences | `player_technique_state / player_persistent_buff_state / player_quest_progress / player_combat_preferences` | 功法、持久 buff、任务、战斗偏好 |
| craft / active job / logbook | `player_profession_state / player_alchemy_preset / player_active_job / player_logbook_message` | 职业、炼丹预设、当前作业、日志 |
| recovery | `player_recovery_watermark` | 玩家各域恢复水位 |

强一致资产操作不应只写玩家 domain 表，还要经过：

- `durable_operation_log`
- `outbox_event`
- `asset_audit_log`
- 必要时写 `dead_letter_event / outbox_consumer_dedupe`

## 性能设计

SQL 写入策略：

- 不做“每个小变化立刻一条 SQL”。
- 不做“整实例大 JSON 高频重写”。
- 优先按 domain 合并，再按主体中等批量写。
- 热路径只标脏，不在 tick 内等待数据库 IO。

具体到地图实例：

- `tile_resource / tile_damage` 适合按 diff / state rows 写入。
- `container_state` 应拆成 state、entry、timer 三类，避免 entries 混进低频 payload。
- `monster_runtime` 是高频运行态，只持久化高价值怪物；普通低价值怪物重启后可按模板重建。
- `time` checkpoint 是恢复辅助，不应跟随每秒 tick 写库。
- `ground_item` 必须按稳定物品 ID 或 tile/key 级 delta 写入；不能以“数量暂少”为理由保留整实例 replace 主路径。

## 恢复链

启动/恢复时的目标顺序：

1. 通过 `instance_catalog` 确定实例目录、租约和归属。
2. 从实例分域表恢复 tile、container、overlay、monster、event、formation 等状态。
3. 读取 `instance_recovery_watermark` 判断各域恢复水位。
4. 只有在分域真源缺失或迁移场景下，才使用 `server_map_aura_v1` 旧快照兜底。

硬切后第 4 条删除：分域真源缺失时恢复失败，旧快照只能在停服离线转换阶段被读取。

玩家恢复时：

1. 身份从 `server_player_identity / player_identity` 恢复。
2. 玩家分域从 `player_*` 表恢复。
3. 不读取 `server_player_snapshot`；玩家运行态直接从 `player_*` 分域装配。
4. 邮件、钱包、背包等资产域必须以分表和 durable operation 结果为准。

## Shutdown 与失败处理

当前必须维持的语义：

- 应用关闭前调用 map flush 强刷，尽量刷完当前 dirty domain。
- worker claim 到的账本项只有落库成功后才能 mark flushed。
- 租约不匹配、实例已非 persistent、节点不可写时，允许跳过并回标，避免旧账本无限重试。
- 落库失败只记录并保留 dirty/ledger 状态，等待下一轮重试。
- 恢复冻结期间禁止普通 interval flush，避免恢复过程中旧运行态覆盖新真源。

## 当前风险与待收口

| 项 | 状态 | 建议 |
| --- | --- | --- |
| `monster_runtime` 慢刷告警 | 已有 interval 降频和 worker，但仍可能被周期兜底带入慢刷 | 让 worker 成为主路径，周期兜底只处理 shutdown/超时未 claim |
| `time` checkpoint | 已降频到默认 300 秒 | 保持低频，不和普通 dirty 域混刷 |
| `ground_item` | 表已拆，已有独立 worker 入口 | 继续用 worker/backlog 指标验证 row-level 写入不会退化 |
| `container_state` | 已拆 state/entry/timer | 继续保证 entries 不回流到 state payload |
| `tile_damage` | 只有 `respawn_left_ticks`，无 wall-clock `respawn_at_ms` | 离线推进需要补 `respawn_at_ms` 或统一时间语义 |
| 药草刷新 | 还没有明确 `deadlineAtMs` 语义 | 若要离线推进，需要从 tick 剩余改为绝对时间 |
| 阵法 | `instance_formation_state` 已拆表 | 纳入统一 domain/ledger worker |
| 宗门 | 已迁到 `server_sect` | 后续可继续把成员/建筑/资产拆成更细子表 |
| 旧地图快照 | 正式 runtime 已禁用读写 | 只保留离线转换输入 |
