# ADR-0004: 持久化分层策略

- **状态**: 已采纳
- **日期**: 2025-02-01

## 决策

采用分层持久化：

| 层次 | 存储 | 用途 | 写入时机 |
|------|------|------|----------|
| 内存 | 进程内存 | 运行时状态 | 实时 |
| Redis | Redis | 在线态、会话、缓存 | 实时/异步 |
| PostgreSQL | 数据库 | 持久化真源 | 异步批量 |

## 约束

- PostgreSQL 是所有"下次还在"数据的唯一真源
- Redis 只做缓存和在线态，不是真源
- tick 内通过 outbox 队列异步写入数据库
- 玩家下线时必须完成 flush
- 强资产操作（货币、背包、装备、邮件领取、市场成交）必须走即时事务，不能只靠异步 flush

## 验证方式

- [ ] tick 内无同步数据库调用
- [ ] 玩家下线后数据完整落库
- [ ] 服务器重启后玩家数据完整恢复
