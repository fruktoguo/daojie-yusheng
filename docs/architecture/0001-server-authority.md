# ADR-0001: 服务端权威模型

- **状态**: 已采纳
- **日期**: 2025-01-15

## 背景

作为商业级 MMO，需要确保游戏状态的一致性和公平性。客户端可能被篡改，网络可能延迟或丢包，多玩家操作可能冲突。

## 决策驱动因素

- 防止作弊：客户端不可信
- 状态一致性：多玩家看到的世界必须一致
- 冲突解决：多玩家同时操作同一资源时需要裁定
- 可审计：所有状态变更必须可追溯

## 考虑的选项

### 选项 A: 客户端权威

- 优点：响应快，实现简单
- 缺点：无法防作弊，状态不一致

### 选项 B: 服务端权威

- 优点：防作弊，状态一致，可审计
- 缺点：延迟感知，服务端负载高

### 选项 C: 混合权威

- 优点：部分操作响应快
- 缺点：边界复杂，容易出错

## 决策

采用 **选项 B: 服务端权威**。

MMO 的公平性和一致性是核心需求，延迟可通过客户端预测和表现层优化缓解。

## 后果

### 正面

- 所有玩家看到一致的游戏世界
- 作弊行为可被服务端拦截
- 状态变更可审计和回放

### 负面

- 客户端操作有延迟感，需要预测和插值优化
- 服务端计算负载高，需要优化热路径

### 约束

- 客户端只能提交意图，不能直接修改游戏状态
- 所有影响游戏正确性的计算必须在服务端完成
- 客户端显示的数据是服务端状态的投影

## 验证方式

- [ ] 客户端无法绕过服务端直接修改玩家属性
- [ ] 多客户端看到的同一实体状态一致
- [ ] 所有状态变更有服务端日志
