# ADR-0006: 地图实例化

- **状态**: 已采纳
- **日期**: 2025-02-01

## 决策

采用模板-实例分离架构。地图模板定义静态结构，实例维护运行时状态。

## 架构

```
MapTemplate（静态配置：地形、NPC、刷怪点、传送点）
    │
    ├── Instance A（主线实例）
    ├── Instance B（副本实例）
    └── Instance C（个人空间）
```

## 实例类型与持久化策略

| 类型 | 持久化策略 | 重启恢复 |
|------|-----------|----------|
| 主地图分线 | permanent | 必须恢复 |
| 宗门/洞府 | permanent | 必须恢复 |
| 公共秘境 | long_lived | 按 TTL 判断 |
| 队伍副本 | session | 不恢复，做失败补偿 |
| 教学/临时 | ephemeral | 不恢复 |

## 生命周期

- 创建：按需创建或启动时从 `instance_catalog` 恢复
- 运行：独立 tick 循环，独立 AOI
- 销毁：玩家全部离开 + TTL 到期后销毁，permanent 类型不销毁
- 恢复：启动时只恢复 catalog，实例状态惰性加载（玩家进入时才加载）

## 约束

- 实例间通过消息通信，不共享内存状态
- 跨实例操作（传送、交易）通过服务层协调
- 实例所有权通过 lease 机制保证单写
- 普通怪物不持久化，从模板重建；只有世界 Boss、宗门守卫等高价值对象持久化

## 验证方式（5000 人 / 10000 图口径）

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| 活跃实例数 | < 5000 | > 8000 |
| 单实例玩家数 | < 200 | > 300 |
| 实例创建延迟 | < 100ms | > 500ms |
| 实例 tick 延迟 | < 50ms | > 200ms |
| 惰性恢复延迟 | < 200ms | > 1000ms |
