# ADR-0002: Tick 调度模型

- **状态**: 已采纳
- **日期**: 2025-01-20

## 决策

采用每地图独立 tick，频率 1Hz。每张地图独立运行 tick 循环，地图间通过消息队列通信。

## 约束

- 玩家意图在 tick 内收集，tick 结束时统一执行
- 同类可覆盖意图以最后一次为准（如寻路目标）
- 不可覆盖意图（如交易）需要排队或拒绝机制
- tick 内禁止直接数据库 IO

## 验证方式（5000 人 / 10000 图口径）

- [ ] 单地图 tick 延迟不超过 100ms
- [ ] 地图间消息延迟不超过 2 tick
- [ ] tick 内无数据库阻塞调用
- [ ] 10000 地图实例总 tick 调度开销不超过 2 核
