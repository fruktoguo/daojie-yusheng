## 2. 境界结构与当前有效上限

## 2.1 总体结构（配置层）

- `realm-levels.json` 定义了 `realmLv 1~127`：
  - 凡武段：`1~18`（一境一级）
  - 修仙段：`19~126`（每 12 级为一个修仙大境）
  - 飞升段：`127`
- 品阶带（gradeBands）按区间划分：凡阶、黄阶、玄阶、地阶、天阶、灵阶、圣阶、帝阶。

## 2.2 大境界映射（服务端逻辑）

- `PLAYER_REALM_STAGE_LEVEL_RANGES`：
  - 凡俗 `1~5`
  - 炼体 `6~8`
  - 锻骨 `9~12`
  - 通脉 `13~15`
  - 先天 `16~18`
  - 练气 `19~30`
  - 筑基 `31~42`

## 2.3 当前可突破上限（实装边界）

- 角色最大境界由 `min(realm-levels最后等级, breakthroughs最大toRealmLv)` 决定。
- 当前 `breakthroughs.json` 仅配置到 `29 -> 30`，因此当前有效上限是 `realmLv 30（半步筑基）`。

## 3. 修炼、灵气位与境界推进

## 3.1 修炼基础产出

- 修炼 Buff 每息基础值：
  - 境界修为：`CULTIVATION_REALM_EXP_PER_TICK = 1`
  - 功法经验：`CULTIVATE_EXP_PER_TICK = 5`
- 修炼会被以下行为打断：
  - 移动
  - 主动攻击
  - 受击

## 3.2 灵气等级与修炼倍率

- 灵气等级基准：`DEFAULT_AURA_LEVEL_BASE_VALUE = 1000`
- 灵气阈值递推：
  - `threshold(1)=1000`
  - `threshold(n+1)=ceil(threshold(n) * 1.5)`
- 地块灵气等级：
  - `auraValue < 1000` 时为 `0` 级
  - 否则按阈值递推累加等级
- 修炼灵气乘区：
  - `auraMultiplier = 1 + auraLevel`
  - 境界每息基础增益会乘该系数

## 3.3 境界修为推进主公式

- 修炼时传入基础值：
  - `baseGain = round(numericStats.realmExpPerTick * auraMultiplier)`
- 经过倍率加成与随机小数结算：
  - `applyRateBonus(base, bonusRate)`，其核心是：
  - `exact = max(minimumGain, base * (1 + bonusRate))`
  - `gain = floor(exact) + Bernoulli(exact - floor(exact))`
- 单次战斗境界修为封顶（仅战斗经验链路）：
  - `singleGain <= progressToNext * 5`

## 3.4 圆满溢出与底蕴

- 当当前境界已满或增益超过剩余进度时，可转底蕴（`overflowToFoundation=true`）。
- 底蕴转化存在衰减（防无限滚雪球）：
  - `decayRate = ln(2) / (progressToNext * 10)`
  - `decaySeed = exp(-decayRate * currentFoundation)`
  - `foundationGainExact = ln(1 + decayRate * overflow * decaySeed) / decayRate`
  - 最终再做随机小数取整。

## 4. 突破机制（境界突破）

## 4.1 可突破判定

- `breakthroughReady = (progress >= progressToNext) && (realmLv < maxRealmLv)`

## 4.2 条件类型

- `item`：材料数量
- `technique`：功法数量/品阶/层级/功法境界
- `attribute`：六维属性阈值
- `root`：灵根阈值（单系或任意最高系）

## 4.3 “可选条件抬高属性门槛”机制

- 对 `item/technique` 且带 `increaseAttrRequirementPct` 的条件：
  - 若未满足，不直接阻断，但会抬高所有属性条件门槛
- 叠乘规则：
  - `increaseMultiplier = Π(1 + increasePct_i/100)`
- 最终属性需求：
  - `effectiveAttrReq = ceil(baseAttrReq * max(1, increaseMultiplier))`

## 4.4 配置缺失保护

- 若某境界未配置突破条目，系统返回“仙路断绝”，并直接阻断突破。

## 5. 开天门与灵根

## 5.1 开启阶段与强制前置

- 开天门开放境界：`realmLv >= 18`（叩仙门）
- 达到该阶段后，若未完成开天门（未入天门），后续突破会被拦截。

## 5.2 斩灵根 / 补灵根 / 逆天改命

- 最多斩断 `4` 条灵根（五行最多留一条）
- 单次斩断或补回成本：
  - `severCost = round(progressToNext * 0.1)`
- 单次逆天改命成本：
  - `rerollCost = round(progressToNext * 0.25)`
- 每次逆天改命会累加平均品质加成：
  - `averageBonus += 2`

## 5.3 灵根数值与战斗/修炼影响

- 灵根值范围：`0~100`
- 入天门后锁定（`entered=true`），灵根写入正式属性。
- 灵根直接转化为：
  - 五行增伤 `elementDamageBonus`
  - 五行减伤 `elementDamageReduce`
- 五行吸收效率（灵气投影）：
  - `efficiencyBpMultiplier = rootValue^2`

## 5.4 灵根幼苗与丹药

- 天品幼苗：
  - 五行先置 `99`，每系 `50%` 概率升 `100`，且保底至少 1 系 `100`
  - 等价增加逆天改命累计 `10` 次
- 神品幼苗：
  - 五行全 `100`
  - 等价增加逆天改命累计 `100` 次
- 幼苗消耗的是底蕴，成本按“剩余等价逆天改命次数”折算：
  - `foundationCost = rerollCost * max(0, seedEquivalent - currentRerollCount)`
- 碎灵丹：
  - 成本 `round(currentProgress * 0.25)`，重置天门状态并 +1 次逆天改命累计
- 往生丹：
  - 境界回 `realmLv=1, progress=0`，底蕴清零

## 6. 战斗核心公式

## 6.1 对抗率底层（命中/闪避/暴击/破招/化解）

- 底层函数 `ratioValue(value, divisor)`：
  - 正值：`value / (value + divisor)`
- `getOpposedCombatRate(value, opposing)` 使用：
  - `value / (value + opposing + 100)`（`100` 来自 `DEFAULT_RATIO_DIVISOR`）

## 6.2 破招与化解

- 若 `breakPower > resolvePower`，可触发破招（概率对抗）
- 若 `resolvePower > breakPower`，可触发化解（概率对抗）
- 破招成功会让本次命中与暴击侧增益（命中、暴击按实现翻倍处理）
- 化解成功会使防御翻倍参与减伤计算

## 6.3 防御减伤

- 常量：
  - `DEFENSE_REDUCTION_ATTACK_RATIO = 0.1`
  - `DEFENSE_REDUCTION_BASELINE = 100`
- 防御减伤率：
  - `reduction = defense / (defense + attackBasis*0.1 + 100)`
- 若有元素减伤，再与防御减伤做组合：
  - `finalReduction = 1 - (1 - reduction) * (1 - elementReduce)`

## 6.4 暴击与境界压制

- 暴击倍率：
  - `critMultiplier = (200 + max(0, critDamage)/10) / 100`
- 境界压制伤害倍率：
  - 设 `gap = attackerRealmLv - defenderRealmLv`
  - `gap > 0`：`(1 + 0.2)^gap = 1.2^gap`
  - `gap < 0`：`(1 - 0.2)^|gap| = 0.8^|gap|`

## 6.5 战斗经验对平A伤害的独立乘区

- 仅普通攻击链路启用：
  - `basicAttackMultiplier = clamp(attackerCombatExp / defenderCombatExp, 0.5, 2.0)`

## 7. 经验与成长逻辑

## 7.1 击杀怪物 -> 境界修为

- 公式：
  - `realmCombatExp = expToNext(monsterLv) * expMultiplier * levelAdjustment * levelDecay * contribution / 1000`

## 7.2 击杀怪物 -> 功法经验

- 公式：
  - `techCombatExp = expToNext(monsterLv) * expMultiplier * levelAdjustment * levelDecay * contribution / 200`

## 7.3 等级差修正（levelAdjustment）

- 令 `delta = min(10, |monsterLv - playerLv|)`
- 玩家低于怪物：
  - `mortal_blood`: `(1+0.1)^delta`
  - `variant`: `(1+0.25)^delta`
  - `demon_king`: `(1+0.4)^delta`
- 玩家高于怪物：
  - `0.5^delta`

## 7.4 怪物等级衰减（levelDecay）

- `1~18` 每级乘 `0.98`
- `19~30` 每级乘 `0.95`
- `31+` 每级乘 `0.92`
- 总衰减是三段幂乘积。

## 8. 功法、炼体、底蕴、战斗经验

## 8.1 功法经验的境界差修正

- 常量步长：`TECHNIQUE_EXP_LEVEL_DELTA_MULTIPLIER_STEP = 0.3`
- 玩家境界高于功法境界：
  - `adjust = (1 + 0.3)^delta = 1.3^delta`
- 玩家境界低于功法境界：
  - `adjust = (1 - 0.3)^delta = 0.7^delta`

## 8.2 功法境界（入门/小成/大成/圆满）

- 由当前层数占最大层数比例推导：
  - `>=1`: 圆满
  - `>=0.66`: 大成
  - `>=0.33`: 小成
  - 其他：入门

## 8.3 炼体

- 每层经验：
  - `expToNext(level) = round(10000 * 1.2^level)`
- 炼体每升 1 层固定收益：
  - 体魄、神识、身法、根骨各 `+1`
- 底蕴灌体：
  - 每 `1` 点底蕴 -> `5` 点炼体经验

## 8.4 底蕴与战斗经验在角色中的地位

- 底蕴 `foundation`：
  - 境界溢出转化得到
  - 可在境界推进时被额外消耗，补充境界修为
  - 可用于灵根幼苗催化、炼体灌注
- 战斗经验 `combatExp`：
  - 战斗推进境界时同步累加
  - 影响命中/闪避对抗优势与普通攻击伤害乘区

## 9. 关键常量速查（当前实现）

- 修炼基础：境界 `+1/息`，功法 `+5/息`
- 灵气基准阈值：`1000`，后续每级阈值 `*1.5`（向上取整）
- 开天门开放：`realmLv 18`
- 斩/补灵根成本：`progressToNext * 10%`
- 逆天改命成本：`progressToNext * 25%`
- 单次战斗境界修为上限：`progressToNext * 5`
- 防御减伤基线：`attack*0.1 + 100`
- 境界压制：高打低每级 `*1.2`，低打高每级 `*0.8`
- 平A战斗经验伤害乘区：`0.5 ~ 2.0`
- 当前突破配置上限：`realmLv 30`
