## 2.2 功法字段模型（检索用）

- 主键：`technique.id`
- 关键字段：
  - `name`：功法名
  - `grade`：品阶（mortal/yellow/mystic/earth/heaven/spirit/saint/emperor）
  - `category`：分类（arts/internal/divine/secret）
  - `realmLv`：推荐/归属境界等级
  - `layers[]`：每层 `level`、`expFactor`（或 `expToNext`）、`attrs`
  - `skills[]`：技能数组（可为空）

## 2.3 技能字段模型（检索用）

- 主键：`skill.id`
- 关键字段：
  - `name`、`desc`
  - `cooldown`
  - `costMultiplier`（最终真气消耗会按品阶与 realmLv换算）
  - `range`
  - `unlockLevel` / `unlockRealm` / `unlockPlayerRealm`
  - `effects[]`：`damage/heal/buff/terrain/cleanse`
  - `formula`：递归表达式（`add/sub/mul/div/min/max/clamp/var`）

## 2.4 功法成长与消耗关键公式

- 功法层经验（读取 `expFactor` 时）：
  - `expToNext = round(expFactor * TECHNIQUE_EXP_BASE * realmLv)`
  - 其中 `TECHNIQUE_EXP_BASE = 100`
- 功法境界推导：
  - 进度 `<33%` 入门，`>=33%` 小成，`>=66%` 大成，满层 圆满
- 技能真气消耗（由 `costMultiplier` 转实际 `cost`）：
  - `cost = round(costMultiplier * 品阶耗气倍率 * realmLv)`
  - 品阶耗气倍率：`mortal=1, yellow=2, mystic=3, earth=4, heaven=5...`
- 境界差对功法经验倍率修正（玩家 vs 功法）：
  - 每级差阶梯因子：`0.3`
  - 玩家低于功法：按 `0.7^差值` 衰减
  - 玩家高于功法：按 `1.3^差值` 提升

## 4. 炼丹体系

## 4.1 配方模型

- 配方字段：
  - `recipeId`
  - `outputItemId`
  - `outputCount`
  - `baseBrewTicks`
  - `ingredients[]`（`itemId/count/role(main|aux)`）
- 当前配方总数：`19`

## 4.2 配方分类规则（服务端）

- `outputItem.consumeBuffs` 非空 -> `category = buff`
- 输出物是 `heal/qi` 类字段 -> `category = recovery`
- 非以上两类会被视为非法配方并报错

## 4.3 炼丹核心机制公式

- 材料功率（用于配方强度计算）：
  - `power = level * (gradeValue^2) * count`
- 配方匹配：
  - 精确匹配（数量与种类全等）=> 基础成功率 `1.0`
  - 非精确匹配 => `successRate = (submittedPower/fullPower)^2`
- 等级修正后的成功率：
  - 丹方等级高于炼丹等级时按 `0.9^差值` 衰减
  - 炼丹等级高于丹方时按渐近函数提升
  - 丹炉额外成功率同样走渐近修正
- 耗时：
  - `总耗时 = 起炉准备(10) + 单炉耗时 * 炉数`
  - `speedRate >= 0` 时按 `1/(1+speedRate)` 加速
- 单炉产量：
  - `buff` 配方：每炉 `1`
  - `recovery` 配方：每炉 `6`
- 灵石消耗：
  - 仅 `buff` 类配方消耗灵石
  - 单炉成本近似 `recipe.outputLevel`

## 4.4 炼丹流程关键行为

- 必须装备丹炉标签武器（`alchemy_furnace`）才能开炼丹面板
- 移动/出手会触发炼丹暂歇：`10` 息后续炼
- 取消炼丹时：
  - 未开炼炉次可退材料与灵石
  - 当前已开炼炉次不可退
  - 背包满时退回物落地

## 4.5 典型配方链路（示例）

- 早期恢复：
  - `alchemy.pill.minor_heal -> pill.minor_heal`
  - `alchemy.minor_qi_pill -> minor_qi_pill`
- 中期战斗增益：
  - `alchemy.pill.break_array_elixir -> pill.break_array_elixir`
  - `alchemy.pill.windstride_elixir -> pill.windstride_elixir`
- 练气中后段：
  - `alchemy.pill.guiding_powder -> pill.guiding_powder`
  - `alchemy.pill.chiyang_draught -> pill.chiyang_draught`
  - `alchemy.pill.fivephase_harmony_pellet -> pill.fivephase_harmony_pellet`

## 5. 强化体系

## 5.1 强化基础规则

- 强化入口要求：装备武器带 `enhancement_hammer` 标签
- 当前上限常量：`MAX_ENHANCE_LEVEL = 20`
- 强化增幅：
  - `强化倍率(%) = ceil(100 * (1 + 0.1)^强化等级)`
- 强化耗时：
  - 基础：`5 + (物品等级-1)*1`
  - 速度修正同技艺统一规则 `1/(1+speedRate)`（正数加速）

## 5.2 成功率与降级逻辑

- 目标强化等级基础成功率：
  - `+1: 50%`
  - `+2~+3: 45%`
  - `+4~+6: 40%`
  - `+7~+10: 35%`
  - `+11~+20: 30%`
- 技艺等级修正：
  - 技艺低于目标物等级：按 `0.9^差值` 惩罚
  - 技艺高于目标物等级：按 `0.002*差值` 做渐近加成
- 强化结果：
  - 成功：升至目标等级，结算灵石消耗
  - 失败且保护生效：降 `1` 级
  - 失败且无保护：直接归 `+0`

## 5.3 保护物规则

- 每步可设“从 +N 开始启用保护”
- 有配置 `protectionItemId` 时按指定道具保护
- 无配置时允许“同类装备自保”，但要求保护物为同 `itemId` 且 `+0`
- 保护物在“失败且保护生效”时才扣除

## 5.4 强化资源规则

- 灵石成本：
  - 无材料步：`ceil(itemLevel/10)`
  - 有材料步：`floor(itemLevel/10)`
- 材料步来自配置文件 `enhancements/**.json`
- 当前仓库强化配置仅有两件器具：
  - `equip.copper_enhancement_hammer`（可冲到 +8）
  - `equip.copper_pill_furnace`（可冲到 +8）
- 其余装备当前默认无额外材料步配置

## 5.5 强化流程行为

- 支持队列冲级（例如从当前级一路冲到目标级）
- 每阶都会重算：
  - 成功率
  - 耗时
  - 下一阶材料需求
- 移动/出手会打断并进入 `10` 息暂歇
- 取消强化：
  - 当前阶已投入材料不退
  - 保护物按失败消耗规则处理
  - 若强化完成后背包满，成品会落地

## 6. 技艺经验（炼丹/强化共用）

- 使用统一函数 `computeCraftSkillExpGain`
- 关键特性：
  - 按动作基础耗时折算经验
  - 失败经验默认是成功经验的 `25%`
  - 技艺等级越高，经验衰减（`0.95^(level-1)`）
  - 低等级补偿：`1级约500%`，到 `20级` 回落到 `100%`
- 炼丹与强化在每次结算后都会给对应技艺经验，并额外折算一部分到修为链路
