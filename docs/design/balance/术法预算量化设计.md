# 术法预算量化设计

更新日期：2026-06-01

本文定义 AI 术法草稿到正式 `SkillDef` 的预算口径。目标是把 AI 输出统一为强度权重，由服务端在冷路径展开真实冷却、消耗、施法距离、影响范围和公式倍率。

## 基本原则

- AI 只输出权重草稿，不输出真实伤害、真实灵力消耗、真实冷却、真实施法距离、真实影响半径或技能公式。
- 服务端根据品阶、境界等级、境界阶段、大境界和强度权重生成正式运行时 `SkillDef`。
- `PLAYER_REALM_STAGE_LEVEL_RANGES` 中的 1-33 序号统一称为“境界阶段”，不是大境界。
- 大境界指凡人期、练气期、筑基期、金丹期、元婴期等更高一层修行分组，可作为术法预算的额外增幅乘区。
- 结构权重、目标范围权重和公式权重共同竞争同一份技能预算。
- 权重不是最终计费强度。最终计费强度应来自服务端展开后的真实可生效数值。
- 最终公式展示和运行结构应保持 `基础值组 * 百分比组`，预算缩放吸收到基础值组的 `scale`，不额外暴露第三个整体乘区。

## 单技能预算

当前满层单技能预算公式：

```text
BUDGET_max = 3 + realmLv * 0.5 * 1.4^(gradeIndex - 1) * majorRealmMultiplier
```

- `realmLv`：境界等级，先乘 `0.5` 后参与预算主体。
- `realmStage`：当前境界等级所在境界阶段序号，按 `PLAYER_REALM_STAGE_LEVEL_RANGES` 从 1 起算；用于 AI 命名和境界描述，不参与术法预算公式。
- `gradeIndex`：功法品阶序号，`mortal=1`、`yellow=2`、`mystic=3`、`earth=4`、`heaven=5`、`spirit=6`、`saint=7`、`emperor=8`。
- `majorRealmMultiplier`：真正大境界额外增幅，未配置时为 `1`。
- AI 首版每个术法功法只生成 1 个技能，该技能独占一份预算。

分层预算仍按层数线性折算：

```text
BUDGET(layer) = BUDGET_max * layer / maxLayer
```

## 预算分配与回流

AI 输出的所有数值均视为“分配倾向”，服务端先把同一技能的满层或当前层预算切成每个项目的实际预算，再由每个项目自己的转换公式反推出真实运行时数值。

基础分配：

```text
positiveWeight = Σ max(itemWeight, 0)
sacrificeBudget = Σ (BUDGET(layer) * abs(negativeWeight) / 100)
positiveBudgetPool = BUDGET(layer) + sacrificeBudget

当 itemWeight > 0:
itemBudget = positiveBudgetPool * itemWeight / positiveWeight

当 itemWeight < 0:
itemBudget = -BUDGET(layer) * abs(itemWeight) / 100
```

- `itemWeight` 来自 `structureStrength` 和 `formulaStrength` 中的权重字段。
- `itemWeight > 0` 表示希望该项更强、更省、更短或更远。
- `itemWeight < 0` 只用于支持负向表达的项目，例如更高消耗、更长冷却、更低层数加成。
- 正权重瓜分 `positiveBudgetPool`；负权重会让本项变差，并按绝对权重折算牺牲预算加入正向预算池。
- `positiveWeight <= 0` 或所有权重无效时，拒绝草稿或使用服务端默认模板补齐，不能生成全零术法。
- `damageBudget <= 0` 时按最低 `0.01` 属性基底倍率展开，避免生成完全 0 伤害技能。

钳制与回流：

```text
allocatedBudget -> rawValue -> cappedValue
usedBudget = inverse(cappedValue)
refundBudget = allocatedBudget - usedBudget
```

- 每个分项转换方法都必须返回真实值、已使用预算和未使用预算。
- 上层预算分配器不猜测各项的离散档位或边界，只收集各转换方法传出的 `refundBudget`。
- 有最大值或最小值的项目先按公式算出真实值，再按边界钳制。
- 正预算因上限或离散档位暂时用不完时，`refundBudget > 0` 进入回流池，平均分给仍可继续增长的正向项目。
- 负预算触底后无法继续变差的部分直接丢弃，不转成其它惩罚。
- 回流按固定迭代次数执行，避免因为多个项目互相触顶导致循环不收敛。
- 已触顶的项目不再参与回流；无上限的属性基底、层数加成和移速加成可持续参与。

转换方法返回结构：

```ts
type BudgetConversionResult<T> = {
  value: T;
  usedBudget: number;
  refundBudget: number;
};
```

例如范围转换直接返回真实 targeting 和剩余预算：

```ts
type RangeBudgetConversionResult = BudgetConversionResult<{
  range: number;
  shape: 'single' | 'line' | 'box' | 'area';
  radius?: number;
  width?: number;
  height?: number;
  coveredCells: number;
}>;
```

## 属性基底成本

术法伤害首版只允许以下属性作为基础值组。`attributeBases` 中的数值表示“属性等价百分比”，`1` 表示 `100%` 该属性。

| 属性 | 字段 | 每 100% 基底成本 |
|---|---|---:|
| 生命 | `maxHp` | 12 |
| 灵力 | `maxQi` | 8 |
| 物攻 | `physAtk` | 1 |
| 法攻 | `spellAtk` | 1 |
| 物防 | `physDef` | 1 |
| 法防 | `spellDef` | 1 |
| 命中 | `hit` | 1 |
| 闪避 | `dodge` | 1 |
| 暴击 | `crit` | 1 |
| 免爆 | `antiCrit` | 1 |
| 破招 | `breakPower` | 1 |
| 化解 | `resolvePower` | 1 |

例：

```json
{ "attributeBases": { "spellAtk": 1 } }
```

表示基础值组吃 `100%` 法攻，效果强度成本为 `1`。

```json
{ "attributeBases": { "maxHp": 1 } }
```

表示基础值组吃 `100%` 生命，效果强度成本为 `12`。

转换公式：

```text
属性倍率 = 属性预算 / 每 100% 基底成本
```

- 最小值：`0%`。
- 最大值：无。
- 属性预算不足时可以低于 `100%`，例如 `spellAtk` 预算 `0.25` 表示 `25%` 法攻。
- 多个属性基底共同参与时，先按 `attributeBases` 权重切分基础值组预算，再分别套用各自成本。

## 百分比组

首版百分比组只开放：

- `techLevel`：默认预算 `0` 时，每层增加 `10%` 总伤害。
- `moveSpeed`：预算 `1` 时，额外加入 `caster.stat.moveSpeed * 0.001` 的总百分比加成。

公式结构：

```text
基础值组 * (1 + 功法层数加成 + 移速加成)
```

预算缩放不得作为额外第三乘区暴露。若需要压缩伤害，应缩放基础值组中各属性基底的 `scale`。

转换公式：

| 项目 | 中文公式 | 最小值 | 最大值 |
|---|---|---:|---:|
| 功法层数加成 | `每层加成 = max(0, 10% * (1 + techLevelBudget))` | `0%/层` | 无 |
| 移速加成 | `移速加成 = caster.stat.moveSpeed * max(0, moveSpeedBudget) * 0.001` | `0` | 无 |

`techLevelBudget = -1` 正好表示每层加成降到 `0%`。低于 `-1` 的负预算触底后丢弃，不继续制造负伤害。

## 目标范围权重

AI 草稿中的 `target` 只描述目标形状和目标模式，不承载预算权重。

- `target.type` 只选择形状：`single`、`line`、`box`、`area`。
- `target.targetMode` 选择目标输入模式：`entity` 表示实体目标，`tile` 表示地格目标，`any` 表示两者都可。
- `structureStrength.castRange` 表示施法距离权重，数值越大越偏远程。
- `structureStrength.area` 表示覆盖范围权重；`line` 会换算宽度，`box` 会换算边长，`area` 会换算半径。
- 玩家主题中的“范围 32 格”表示希望覆盖强度接近 32 格，不等于真实半径或真实宽高。

施法距离转换：

```text
castRangeCost(realRange) =
  realRange <= 1 ? 0 : (realRange - 1) * 1.2^(realRange - 1)

realRange = 最大整数 r，使 castRangeCost(r) <= castRangeBudget
usedCastRangeBudget = castRangeCost(realRange)
refundCastRangeBudget = castRangeBudget - usedCastRangeBudget
```

- 最小值：`1`。
- 最大值：普通形状为 `20`，`line` 形状为 `12`。
- 施法范围 `1` 为 `0` 预算，`2` 为 `1 * 1.2` 预算，`3` 为 `2 * 1.2 * 1.2` 预算。
- 预算不足下一档时向下取整，未使用预算回流。

影响范围转换使用覆盖格口径。每 1 点实际影响范围预算购买额外 2 格：

```text
maxCoveredCells = 1 + shapeBudget * 2
usedShapeBudget = (actualCoveredCells - baseCoveredCells) / 2
refundShapeBudget = shapeBudget - usedShapeBudget
```

| 形状 | 转换口径 | 最小值 | 最大值 |
|---|---|---:|---:|
| `single` | 固定单体，覆盖预算无效并回流。 | 1 目标 | 1 目标 |
| `line` | 线长由 `realRange` 决定，覆盖预算购买奇数线宽。 | 长 1、宽 1 | 长 12、宽 9 |
| `box` | 选择最大奇数边长正方形，使 `side * side <= maxCoveredCells`。 | 1x1 | 25x25 |
| `area` | 选择最大圆形半径，使圆形真实覆盖格数不超过 `maxCoveredCells`。 | 半径 0 | 半径 12 |


## 冷却和消耗权重

`structureStrength.cost/cooldown/chant` 都是结构权重，不是真实运行时数值。

各项转换公式：

| 项目 | 中文公式 | 最小值 | 最大值 |
|---|---|---:|---:|
| 灵力消耗倍率 | `消耗倍率 = costBudget >= 0 ? 0.9^costBudget : 1.2^abs(costBudget)` | `0` | 无 |
| 冷却 | `冷却息数 = 3 * realmLv * (cooldownBudget >= 0 ? 0.98^cooldownBudget : 1.02^abs(cooldownBudget))` | `1` | 无 |
| 吟唱 | `吟唱息数 = max(0, round(abs(min(chantBudget, 0))))` | `0` | 无 |

- 冷却正预算越高，真实冷却越短；负预算越低，真实冷却越长。
- `realmLv` 是境界等级；0 冷却预算时基础冷却为 `3 * realmLv` 息。
- 冷却预算按指数衰减，每 1 点正预算让当前冷却乘 `0.98`，约等于在上一档基础上减少 `2%`。
- 消耗正预算越高，真实灵力消耗倍率越低；负预算越低，真实消耗倍率越高。
- 吟唱当前只表达缺点，负预算会生成更长吟唱；正预算不把 0 吟唱继续缩短，触底后回流。
- 解析时不能再按 `1.2^AI权重` 直接扣总预算。必须使用分配后的 `itemBudget`，再按真实可生效值反推 `usedBudget`。
- 这样可以避免 `cooldownWeight: 80` 被钳到 `1` 息后，预算仍继续按 `1.2^80` 扣费。

## 展开流程

建议的 AI 术法展开流程：

```text
1. 校验 AI JSON 结构和枚举。
2. 读取 structure/formula 权重；target 只读取形状和目标模式。
3. 用品阶、境界等级和大境界得到 BUDGET(layer)。
4. 将正权重和负权重按各自口径切分为 itemBudget。
5. 每个项目按自己的转换公式展开真实值。
6. 对有上下限的项目钳制，并反算 usedBudget。
7. 将触顶或离散档位暂时用不完的正预算平均回流到仍可增长的项目。
8. 把最终属性预算写入基础值组属性 scale。
9. 生成正式 SkillDef。
```

目标公式形态：

```text
caster.stat.spellAtk * 0.32 * (1 + techLevel * 0.1)
```

避免生成：

```text
(caster.stat.spellAtk * (1 + techLevel * 0.1)) * 0.32
```

更不能因为保留精度不足生成：

```text
(caster.stat.spellAtk * (1 + techLevel * 0.1)) * 0
```

## 兼容边界

- 本文首先约束 AI 生成术法草稿入口。
- 旧静态内容和已经发布的生成内容若仍使用 `target.range/radius/width/height` 草稿字段，需要通过 GM 一键转换迁移；已经存在的 `target.castRangeWeight/areaWeight` 草稿字段仅作为兼容输入读取，新 AI 生成入口统一写 `structureStrength.castRange/area`。
- 正式运行时 `SkillDef` 仍然保存真实 `cooldown/cost/targeting/effects.formula`，战斗 tick 不读取权重草稿。
- 已发布 AI 功法的 `generated_technique.template.skills` 是展开后的运行时技能快照；预算公式、冷却公式或范围换算公式调整后，已有模板不会自动重新计算。
- 公式调整后的存量同步必须走 GM 快捷操作：先执行“迁移旧版AI术法草稿”，由 `validation_report.artsStrength.rawCandidate` 按当前公式重新展开并写回 `template.skills`；再执行“刷新在线玩家功法模板”，让在线玩家已学技能重新从最新模板水合。
- 离线玩家不需要逐个改存档；下次登录时会从最新模板水合。若只是调整公式常量或转换公式，两个 GM 快捷按钮继续复用当前代码路径，通常不需要为每次公式变更单独改按钮。
