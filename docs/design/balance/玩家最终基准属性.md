# 玩家最终基准属性

统计日期：2026-06-03

## 目标

本文定义“玩家最终基准属性”的计算口径，用来评估技能耗灵、伤害预算、装备强度和怪物强度是否与标准玩家面板匹配。

玩家最终基准属性不是线上玩家实时属性。实时属性会受玩家实际功法、装备、根基、炼体、Buff、丹药、灵根、GM 修正等影响；本文只定义某个境界等级下的期望标准面板。

## 与现有基准的关系

现有基准分三类：

| 基准 | 来源 | 含义 | 是否等于玩家最终基准 |
|---|---|---|---|
| `singleAttr` | `realm-attr-baselines.json` | 标准六维单项期望值 | 否，只是最终基准的六维输入 |
| `singleBaseStatValue` | `realm-attr-baselines.json` | 普通基础属性量化基准 | 否，不计入六维、根基、炼体 |
| 装备基准 | `装备基准值占比改造预览.md` | 装备词条量化值 | 否，只是最终基准的装备输入 |

玩家最终基准属性由三段合成，但不是把三段简单相加。

```text
第一段：当前境界的期望根基、期望炼体、期望六维，合成期望总六维
第二段：境界模板基础值 + 标准装备期望强化值，合成期望基础数值
第三段：境界倍率
```

最终合成口径是：

```text
玩家最终基准属性
= 期望基础数值 × 期望六维乘区 × 境界倍率
```

其中“期望总六维”用于折算六维百分比乘区；它不直接和生命、灵力、攻击、防御等基础数值相加。

## 一、期望总六维输入

期望总六维来自 `docs/design/balance/境界等级基准期望六维公式.md`。

```text
根基上限 = 向下取整(等级 × (等级 + 1) ÷ 2)
期望根基百分比 = 根基上限 × 期望根基比例 × 每点根基百分比
```

当前参数：

```text
期望根基比例 = 0.6
每点根基百分比 = 1%
```

```text
期望炼体百分比 = 等级 × 期望炼体等级倍率 × 每层炼体百分比
```

当前参数：

```text
期望炼体等级倍率 = 1.5
每层炼体百分比 = 1%
```

```text
乘区前单项六维
= 默认基础六维
 + 境界基础单项六维
 + 功法期望单项六维
```

```text
期望单项六维
= 乘区前单项六维
 × (1 + 期望根基百分比)
 × (1 + 期望炼体百分比)
```

`realm-attr-baselines.json` 中的 `singleAttr` 即为：

```text
singleAttr(等级) = 期望单项六维
```

期望总六维为：

```text
期望总六维 = singleAttr × 6
```

如果采用均衡标准玩家：

```text
标准体魄 = singleAttr
标准神识 = singleAttr
标准身法 = singleAttr
标准根骨 = singleAttr
标准力道 = singleAttr
标准经脉 = singleAttr
```

如果后续要定义偏科职业标准，可以在 `singleAttr` 总池基础上改为按比例分配六维，但总六维仍应等于：

```text
singleAttr × 6
```

## 二、期望基础数值输入

玩家最终基准的基础数值不是单独使用 `singleBaseStatValue`。`singleBaseStatValue` 是量化尺，不是玩家最终面板。玩家最终基准的基础数值应由当前境界模板基础值和标准装备期望强化值共同组成。

```text
期望基础数值(属性)
= 境界模板基础值(属性)
 + 标准装备期望强化值(属性)
```

### 境界模板基础值

境界模板基础值来自：

```text
resolvePlayerRealmNumericTemplate(当前境界阶段).stats
```

例如灵力输出速率使用：

```text
境界模板灵力输出 = 当前境界阶段模板中的 maxQiOutputPerTick
```

### 标准装备期望强化值

装备基准来自 `docs/design/balance/装备基准值占比改造预览.md`。

```text
装备等级基准 = 8 + 装备等级 × 0.5
```

```text
期望强化倍率 = 1.1 ^ 期望强化等级
```

```text
单件装备期望属性值
= 四舍五入(
    装备等级基准
    × 装备品阶倍率
    × 期望强化倍率
    × 装备基准占比 ÷ 100
    × 每1量化值对应实际点数
  )
```

```text
标准装备期望强化值(属性)
= 所有标准装备位的单件装备期望属性值之和
```

如果某个基准场景要求“不穿装备”，则：

```text
标准装备期望强化值 = 0
```

但正式用于技能耗灵和战斗强度评估时，应使用“有标准装备”的口径，而不是裸装口径。

## 三、期望六维乘区

期望总六维会折算为各项数值面板的百分比乘区。

当前六维到数值的乘区关系：

| 数值属性 | 六维百分比来源 | 中文公式 |
|---|---|---|
| 生命 | 体魄、根骨 | `1 + (体魄 + 根骨) ÷ 100` |
| 灵力 | 根骨、经脉 | `1 + (根骨 + 经脉) ÷ 100` |
| 物攻 | 体魄、力道 | `1 + (体魄 + 力道) ÷ 100` |
| 法攻 | 神识、经脉 | `1 + (神识 + 经脉) ÷ 100` |
| 物防 | 体魄 | `1 + 体魄 ÷ 100` |
| 法防 | 神识 | `1 + 神识 ÷ 100` |
| 命中 | 神识 | `1 + 神识 ÷ 100` |
| 闪避 | 身法 | `1 + 身法 ÷ 100` |
| 暴击 | 力道 | `1 + 力道 ÷ 100` |
| 抗暴 | 身法 | `1 + 身法 ÷ 100` |
| 破招 | 力道 | `1 + 力道 ÷ 100` |
| 化解 | 根骨 | `1 + 根骨 ÷ 100` |
| 灵力输出 | 经脉 | `1 + 经脉 ÷ 100` |
| 速度 | 身法 | `1 + 身法 × 0.5 ÷ 100` |

均衡标准玩家下，可以简化为：

```text
标准单项六维 = singleAttr
```

例如：

```text
灵力输出期望六维乘区 = 1 + singleAttr ÷ 100
生命期望六维乘区 = 1 + (singleAttr + singleAttr) ÷ 100
```

通用写法：

```text
期望六维百分比(属性)
= 标准体魄 × 该属性体魄权重
 + 标准神识 × 该属性神识权重
 + 标准身法 × 该属性身法权重
 + 标准根骨 × 该属性根骨权重
 + 标准力道 × 该属性力道权重
 + 标准经脉 × 该属性经脉权重
```

```text
期望六维乘区(属性) = 1 + 期望六维百分比(属性) ÷ 100
```

## 四、境界倍率

指数成长属性使用：

```text
境界倍率 = 1.1 ^ (等级 - 1)
```

适用：

```text
生命、灵力、物攻、法攻、物防、法防、命中、闪避、暴击、抗暴、破招、化解、灵力输出
```

线性成长属性使用：

```text
线性境界倍率 = 1 + 成长率 × (等级 - 1)
```

适用：

| 属性 | 成长率 |
|---|---:|
| 暴击伤害 | 0.1 |
| 灵力回复 | 0.02 |
| 生命回复 | 0.02 |
| 每息境界修为 | 0.1 |
| 每息功法经验 | 0.1 |

没有列入指数或线性成长的属性，境界倍率按 `1` 处理。

## 五、最终基准属性公式

指数成长属性：

```text
最终基准属性(属性)
= 四舍五入(
    期望基础数值(属性)
    × 六维百分比乘区(属性)
    × 境界倍率
  )
```

线性成长属性：

```text
最终基准属性(属性)
= 四舍五入(
    期望基础数值(属性)
    × 期望六维乘区(属性)
    × 线性境界倍率
  )
```

没有六维乘区的普通属性：

```text
最终基准属性(属性)
= 四舍五入(
    期望基础数值(属性)
    × 对应境界倍率
  )
```

## 六、灵力输出基准

灵力输出是技能耗灵调平时的核心参考。

```text
最终基准灵力输出
= 四舍五入(
    (境界模板灵力输出 + 标准装备期望灵力输出)
    × (1 + 标准经脉 ÷ 100)
    × 1.1 ^ (等级 - 1)
  )
```

均衡标准玩家下：

```text
标准经脉 = singleAttr
```

所以：

```text
最终基准灵力输出
= 四舍五入(
    (境界模板灵力输出 + 标准装备期望灵力输出)
    × (1 + singleAttr ÷ 100)
    × 1.1 ^ (等级 - 1)
  )
```

裸装调试时：

```text
标准装备期望灵力输出 = 0
```

正式平衡评估时：

```text
标准装备期望灵力输出 = 标准装备在当前等级下提供的灵力输出总和
```

所以半步金丹裸装调试口径只是在上式中令标准装备期望灵力输出为 0；如果真实玩家经脉远高于 `singleAttr`，实时面板会明显高于最终基准，这是正常差异，不应反过来污染基准公式。

## 八、技能耗灵评估口径

技能耗灵不应只和 `singleBaseStatValue` 对比，而应和最终基准灵力输出对比。

当前技能基础耗灵公式：

```text
品阶序号：凡阶=0，黄阶=1，玄阶=2，地阶=3，天阶=4，灵阶=5，圣阶=6，帝阶=7
品阶指数倍率 = 1.4 ^ 品阶序号
技能基础耗灵 = 四舍五入(最终基准灵力输出 × 0.2 × 品阶指数倍率 × 技能耗灵系数)
```

因此：

```text
基础耗灵占用息数 = 0.2 × 1.4 ^ 品阶序号 × 技能耗灵系数
```

```text
技能耗灵占用息数
= 技能基础耗灵 ÷ 最终基准灵力输出
```

建议分层：

| 技能类型 | 建议占用息数 |
|---|---:|
| 低耗/功能技能 | 0.5 - 1.5 息 |
| 常规输出技能 | 2 - 4 息 |
| 爆发技能 | 4 - 8 息 |
| 神通/大招 | 8 - 15 息 |

如果继续保留 `maxQiOutputPerTick` 超限惩罚，还需要再计算实际扣灵：

```text
实际扣灵 = 输出上限惩罚公式(技能基础耗灵, 最终基准灵力输出)
```

技能是否合理，应同时看：

```text
基础耗灵占用息数
实际扣灵占用息数
```

## 九、与实时玩家属性的区别

玩家实时属性是权威运行时结果：

```text
实时玩家属性
= 玩家实际境界
 + 实际功法
 + 实际根基
 + 实际炼体
 + 实际装备
 + Buff
 + 丹药
 + 灵根
 + GM/活动/环境修正
```

玩家最终基准属性是数值设计期望：

```text
最终基准属性
= (境界模板基础值 + 标准装备期望强化值)
 × 期望六维乘区
 × 境界倍率
```

两者用途不同：

| 项目 | 用途 |
|---|---|
| 实时玩家属性 | 战斗结算、UI 面板、真实扣血扣灵 |
| 最终基准属性 | 技能耗灵、怪物强度、装备预算、伤害预算的设计参考 |
