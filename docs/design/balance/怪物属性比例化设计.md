# 怪物属性比例化设计

统计日期：2026-05-07
当前源码核对：2026-05-13

## 目标

怪物配置不再直接写任何固定属性值，包括六维、生命、攻击、防御、命中、闪避等实际数值。怪物配置只写等级、品阶、层级和属性倾向比例；运行配置在冷路径编译成实际战斗数值。

核心目标：

- 怪物 JSON 只表达“这个怪偏肉、偏物攻、偏闪避”等倾向。
- 实际数值统一从基准配置推导，避免每只怪手填后失控。
- 普通、精英、Boss 的强度差异只由层级倍率决定。
- 品阶加成统一使用指数倍率，每升一品阶增加 20%。

当前运行时仍保留旧口径兼容：

1. `valueStats` 存在时优先使用价值数值口径。
2. 没有 `valueStats` 但存在 `attrs` 时使用显式六维口径。
3. 两者都没有时才使用本文主线的 `attrTendency/statTendency` 倾向口径。

新增或批量维护怪物内容时，应使用倾向口径；旧口径只作为兼容路径，不作为长期内容写法。

## 数据来源

基准来源统一使用：

- `packages/server/data/content/realm-attr-baselines.json`
  - `singleAttr`：当前境界等级的单项六维基准值。
  - `singleBaseStatValue`：当前境界等级的单项基础属性量化基准值。

品阶顺序沿用现有功法/物品品阶：

```text
mortal, yellow, mystic, earth, heaven, spirit, saint, emperor
```

品阶倍率：

```text
gradeMultiplier(grade) = 1.2 ^ gradeIndex
```

例如：

| 品阶 | index | 倍率 |
|---|---:|---:|
| mortal | 0 | 1.000 |
| yellow | 1 | 1.200 |
| mystic | 2 | 1.440 |
| earth | 3 | 1.728 |
| heaven | 4 | 2.074 |
| spirit | 5 | 2.488 |
| saint | 6 | 2.986 |
| emperor | 7 | 3.583 |

## 怪物层级

现有 `tier` 可以继续复用，但语义要收敛：

| 配置层级 | 含义 | 六维倍率 | 基础属性倍率 | 生命额外倍率 |
|---|---|---:|---:|---:|
| `mortal_blood` | 普通怪 | 50% | 50% | 1 |
| `variant` | 精英怪 | 80% | 80% | 2 |
| `demon_king` | Boss | 100% | 100% | 10 |

其中：

- 六维倍率作用于 `singleAttr`。
- 基础属性倍率作用于 `singleBaseStatValue`。
- 生命额外倍率只作用于最终 `maxHp`，不反向影响六维。

## 配置字段建议

后续怪物配置只保留倾向比例，不写实际值。

建议字段：

```json
{
  "id": "m_example",
  "name": "示例怪",
  "level": 30,
  "grade": "yellow",
  "tier": "mortal_blood",
  "attrTendency": {
    "constitution": 120,
    "spirit": 80,
    "perception": 100,
    "talent": 70,
    "strength": 130,
    "meridians": 100
  },
  "statTendency": {
    "maxHp": 120,
    "physAtk": 130,
    "physDef": 100,
    "hit": 90,
    "dodge": 80
  }
}
```

字段含义：

- `attrTendency`：六维倾向，`100` 表示吃满一份该怪层级的六维基准。
- `statTendency`：基础属性倾向，`100` 表示吃满一份该怪层级的基础属性基准。
- 未写的单项属性默认 `100`。
- 整个 `attrTendency` 或 `statTendency` 不写时，也按全项 `100` 处理；也就是一只完全没有倾向配置的怪物是均衡怪。
- 不允许写 `attrs`、`stats`、`valueStats` 这类实际值字段作为长期配置。

## 六维公式

每个六维单项：

```text
monsterAttr(attr, level, grade, tier) =
  round(
    singleAttr(level)
    * tierBaseRatio(tier)
    * gradeMultiplier(grade)
    * lowLevelNerf(level)
    * attrTendency[attr] / 100
  )
```

层级基础倍率：

```text
tierBaseRatio(mortal_blood) = 0.5
tierBaseRatio(variant) = 0.8
tierBaseRatio(demon_king) = 1.0
```

低等级削弱：

```text
lowLevelNerf(level <= 30) = 0.7
lowLevelNerf(level >= 31) = 1.0
```

示例：

```text
30 级黄阶普通怪，singleAttr = 2916.20，gradeMultiplier = 1.2
strength 倾向 130%

strength = round(2916.20 * 0.5 * 1.2 * 0.7 * 130 / 100)
         = 1592
```

## 基础属性公式

基础属性先按量化值计算，再按属性换算表转成实际值。

每个基础属性的量化值：

```text
baseStatValue(stat, level, grade, tier) =
  singleBaseStatValue(level)
  * realmExpMultiplier(level)
  * tierBaseRatio(tier)
  * gradeMultiplier(grade)
  * lowLevelNerf(level)
  * statTendency[stat] / 100
```

实际属性：

```text
monsterStat(stat) =
  round(baseStatValue(stat) * sixDimMultiplier(stat, attrs) * statPointsPerValue(stat))
```

`statPointsPerValue` 使用当前数值量化表，例如：

| 属性 | 字段 | 每 1 量化值对应实际点数 |
|---|---|---:|
| 生命 | `maxHp` | 12 |
| 灵力 | `maxQi` | 8 |
| 物攻 | `physAtk` | 1 |
| 法攻 | `spellAtk` | 1 |
| 物防 | `physDef` | 1 |
| 法防 | `spellDef` | 1 |
| 命中 | `hit` | 1 |
| 闪避 | `dodge` | 1 |
| 暴击 | `crit` | 1 |
| 抗暴 | `antiCrit` | 1 |
| 破防 | `breakPower` | 1 |
| 化解 | `resolvePower` | 1 |
| 灵力输出 | `maxQiOutputPerTick` | 1 |
| 速度 | `moveSpeed` | 1 |

生命额外倍率最后应用：

```text
if stat == maxHp:
  monsterStat(maxHp) = round(monsterStat(maxHp) * hpTierMultiplier(tier))
```

生命倍率：

```text
hpTierMultiplier(mortal_blood) = 1
hpTierMultiplier(variant) = 2
hpTierMultiplier(demon_king) = 10
```

特殊属性：

- `viewRange = round(8 * viewRangeTendency / 100)`，不吃等级指数、六维、品阶、层级和低等级削弱。
- `hpRegenRate/qiRegenRate = round(50 * regenTendency / 100)`，不吃等级指数、六维、品阶、层级和低等级削弱。
- 基础数值成型后，主要战斗属性会额外应用等级增幅层；这不是配置倾向，而是所有怪物统一的等级后处理。
- 模板解析完成后，所有怪物都会套全局调节层：`hpRegenRate`、`dodge`、`antiCrit` 乘 `25%`，其它纳入字段默认 `100%`。
- `moveSpeed` 最后经过 `getEffectiveMoveSpeed()` 归一化。

## 主要战斗属性等级增幅

主要战斗属性统一由 `MONSTER_MAIN_COMBAT_STAT_KEYS` 定义：

```text
maxHp, maxQi, maxQiOutputPerTick,
physAtk, spellAtk, physDef, spellDef,
hit, dodge, crit, antiCrit, breakPower, resolvePower
```

其中 `antiCrit` 是免爆，`breakPower` 是破招，`resolvePower` 是化解。暴伤、回血、回灵、速度、视野不在这个集合内。

等级增幅曲线：

```text
mainCombatLevelModifier(level <= 1) = 0
mainCombatLevelModifier(2..18) = (level - 1) * 20 / 17
mainCombatLevelModifier(19..30) = 20 + (level - 18) * 80 / 12
mainCombatLevelModifier(level > 30) = 100 + (level - 30) * 5 + floor((level - 30) / 12) * 40
```

应用公式：

```text
mainCombatStatActual = round(mainCombatStatActual * percentModifierToMultiplier(mainCombatLevelModifier(level)))
```

当前关键点：1 级 `0%`，18 级 `20%`，30 级 `100%`，30 级后每级 `+5%`，每跨 12 级额外 `+40%`，所以 42 级为 `200%`。

## 计算顺序

冷路径编译时按这个顺序：

1. 读取怪物 `level`，从 `realm-attr-baselines.json` 找到对应基准。
2. 读取 `grade`，计算 `1.2 ^ gradeIndex`。
3. 读取 `tier`，取得普通/精英/Boss 基础倍率。
4. 读取 `lowLevelNerf`，1-30 级为 `0.7`，31 级及以上为 `1`。
5. 用 `attrTendency` 编译六维；未写时按全六维 `100`。
6. 用 `statTendency`、境界指数成长和六维乘区编译基础属性；未写时按全部纳入属性 `100`。
7. 对 `maxHp` 追加精英/Boss 生命额外倍率。
8. 对主要战斗属性应用等级增幅层。
9. 应用全局怪物调节层并归一化 `moveSpeed`。
10. 输出运行时实际 `attrs` 和 `numericStats`。

公式汇总：

```text
attr = round(singleAttr * tierBaseRatio * gradeMultiplier * lowLevelNerf * attrTendency / 100)

statValue = singleBaseStatValue * realmExpMultiplier * tierBaseRatio * gradeMultiplier * lowLevelNerf * statTendency / 100
statActual = round(statValue * sixDimMultiplier * statPointsPerValue)

maxHpActual = round(maxHpActual * hpTierMultiplier)
mainCombatStatActual = round(mainCombatStatActual * percentModifierToMultiplier(mainCombatLevelModifier(level)))
```

## 配置倾向约束

倾向比例建议保持整数，便于人工调参。

建议范围：

| 倾向类型 | 推荐范围 | 说明 |
|---|---:|---|
| 弱项 | 30-70 | 明显不擅长 |
| 常规项 | 80-120 | 大多数普通倾向 |
| 强项 | 130-180 | 怪物核心特点 |
| 极端项 | 200+ | 只给少量机制怪或 Boss |

为了避免“所有属性都 150%”造成全面超模，后续可以加预算校验：

```text
sum(补齐默认100后的 attrTendency) / 6 建议接近 100
sum(补齐默认100后的 statTendency) / 纳入属性数量 建议接近 100
```

预算校验先作为编辑器/审计提示，不建议运行时强裁。

## 不进入比例化的字段

以下字段不是战斗属性固定值，可以继续按行为配置保留：

- `id`
- `name`
- `char`
- `color`
- `level`
- `grade`
- `tier`
- `count`
- `maxAlive`
- `radius`
- `aggroRange`
- `respawnSec`
- `respawnTicks`
- `drops`
- `skills`
- `initialBuffs`
- `viewRange`

说明：

- `viewRange` 是感知/交互范围，不纳入本次基础属性池。
- `initialBuffs` 仍可保留百分比 buff 或机制 buff，但不应再用来给怪物补一组固定基础数值。
- 如果某只怪需要“重伤”“虚弱”等剧情状态，优先用百分比 debuff 表达。

## 迁移规则

旧配置迁移时：

1. 删除怪物 JSON 中的 `attrs` 固定六维。
2. 删除 `valueStats`、`stats` 等固定基础属性。
3. 将旧 `statPercents` 迁移为 `statTendency`，但需要重新审视数值，因为当前口径还包含低等级削弱、境界指数成长、六维乘区和全局怪物调节层。
4. 根据怪物定位补 `attrTendency`；不需要特殊倾向的怪物可以不写，默认就是全项 100。
5. 保留 `grade`、`tier`、`level`、技能、掉落和刷新行为。
6. 服务端内容加载或生成脚本把倾向配置编译成运行时实际值。

建议先做一版自动迁移预览文档：

```text
旧固定值 -> 反推倾向比例 -> 新公式编译值 -> 差异百分比
```

确认差异后再批量修改怪物配置。

## 示例

以 30 级黄阶普通怪为例：

```text
singleBaseStatValue(30) = 56.45
tierBaseRatio(普通) = 0.5
gradeMultiplier(黄阶) = 1.2
physAtk 倾向 = 130

physAtk = round(56.45 * 1.1^29 * 0.5 * 1.2 * 0.7 * 130 / 100 * sixDimMultiplier * 1)
```

同级黄阶精英：

```text
tierBaseRatio(精英) = 0.8
physAtk = round(56.45 * 1.1^29 * 0.8 * 1.2 * 0.7 * 130 / 100 * sixDimMultiplier)
```

同级黄阶 Boss：

```text
tierBaseRatio(Boss) = 1.0
physAtk = round(56.45 * 1.1^29 * 1.0 * 1.2 * 0.7 * 130 / 100 * sixDimMultiplier)
```

如果 `maxHp` 倾向也是 130：

```text
普通 maxHp = round(56.45 * 1.1^29 * 0.5 * 1.2 * 0.7 * 130 / 100 * sixDimMultiplier * 12) * 1
精英 maxHp = round(56.45 * 1.1^29 * 0.8 * 1.2 * 0.7 * 130 / 100 * sixDimMultiplier * 12) * 2
Boss maxHp = round(56.45 * 1.1^29 * 1.0 * 1.2 * 0.7 * 130 / 100 * sixDimMultiplier * 12) * 10
```

## 待确认点

- `viewRange` 当前已按独立倾向公式处理，不参与基础属性池。
- 自动恢复当前按独立倾向公式处理；修炼/经验、掉落率是否进入怪物基础属性倾向仍应单独审查。
- 倾向口径下，装备引用不参与基础战斗数值计算；旧 `attrs` 口径仍会合并装备属性用于兼容。
- `initialBuffs` 当前允许固定或百分比数值修饰，并在运行态叠加到基础数值上；长期内容建议优先使用百分比或机制 buff。
