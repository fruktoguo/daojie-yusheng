# 怪物当前属性计算总览

本文档记录当前服务端实际使用的怪物数值规则，并保留一份按当前内容配置生成的怪物数值快照。

源码权威入口：

- 模板解析：`packages/shared/src/monster.ts` 的 `resolveMonsterTemplateRecord()`。
- 服务端加载：`packages/server/src/content/content-template.repository.ts` 的 `normalizeMonsterRuntimeTemplate()`。
- 运行态重算：`packages/server/src/runtime/instance/map-instance.runtime.ts` 的 `recalculateMonsterBaseStatsFromFormula()` 和 `recalculateMonsterDerivedState()`。

当前怪物模板支持三种来源口径，优先级如下：

1. `valueStats`：按价值数值编译成实际 `numericStats`，再吃等级成长、`statPercents` 和全局怪物调节层；该口径不再额外吃 `grade/tier` 倍率。
2. `attrs`：按显式六维、装备、等级和 `statPercents` 计算基础数值，再吃品阶、血脉和全局怪物调节层；这是旧配置兼容口径。
3. `attrTendency/statTendency`：没有 `valueStats` 和显式 `attrs` 时使用，当前内容主线应优先使用此口径。

长期内容配置应使用 `attrTendency/statTendency` 表达怪物偏科，不再直接写固定六维或固定战斗数值。地图刷新点只维护模板引用和可选的等级/层级覆盖，实际基础数值仍由怪物模板公式重算。

## 当前倾向公式

倾向口径的数据源：

- 怪物配置：`packages/server/data/content/monsters/*.json`。
- 等级基准：`packages/server/data/content/realm-attr-baselines.json` 的 `singleAttr` 和 `singleBaseStatValue`。
- 品阶顺序：`mortal, yellow, mystic, earth, heaven, spirit, saint, emperor`。

六维实际值：

```text
attrActual =
  round(singleAttr(level) * tierBaseRatio(tier) * gradeMultiplier(grade) * lowLevelNerf(level) * attrTendency[attr] / 100)
```

战斗数值实际值：

```text
statActual =
  round(
    singleBaseStatValue(level)
    * realmExpMultiplier(level)
    * tierBaseRatio(tier)
    * gradeMultiplier(grade)
    * lowLevelNerf(level)
    * statTendency[stat] / 100
    * sixDimMultiplier(stat, attrs)
    * pointsPerValue(stat)
  )
```

当前倍率：

| 项 | 规则 |
|---|---|
| `gradeMultiplier` | `1.2 ^ gradeIndex` |
| `lowLevelNerf` | 1-30 级为 `0.7`，31 级及以上为 `1` |
| `realmExpMultiplier` | `getRealmAttributeMultiplier(level)`，即 `1.1 ^ (level - 1)` |
| `tierBaseRatio(mortal_blood)` | `0.5` |
| `tierBaseRatio(variant)` | `0.8` |
| `tierBaseRatio(demon_king)` | `1` |
| `hpTierMultiplier(mortal_blood)` | `1` |
| `hpTierMultiplier(variant)` | `2` |
| `hpTierMultiplier(demon_king)` | `10` |

特殊字段：

- 未写的 `attrTendency/statTendency` 单项按 `100%` 处理。
- `maxHp` 先按战斗数值公式计算，再乘血脉生命额外倍率。
- `viewRange = round(8 * viewRangeTendency / 100)`，不吃境界指数、六维、品阶、血脉和低级削弱。
- `hpRegenRate/qiRegenRate = round(50 * regenTendency / 100)`，不吃境界指数、六维、品阶、血脉和低级削弱。
- 解析完成后会应用全局怪物调节层：`hpRegenRate`、`dodge`、`antiCrit` 乘 `25%`，其它纳入字段默认 `100%`。
- `moveSpeed` 会再经过 `getEffectiveMoveSpeed()` 归一化。
- 模板 `initialBuffs` 会在运行态叠加到 `baseAttrs/baseNumericStats` 上；最大生命和最大灵力变化时，当前生命/灵力按比例迁移。

## 统计口径

- 数据源：`packages/server/data/content/monsters/*.json`。
- 倾向来源：从当前 `attrs` 和 `statPercents` 反推，只保留设定偏科，不直接沿用旧配置的绝对强度。
- 基准来源：`packages/server/data/content/realm-attr-baselines.json` 的 `singleAttr` 和 `singleBaseStatValue`。
- 六维为体魄、神识、身法、根骨、力道、经脉；格式为 `新公式实际值(整数倾向百分比)`。
- 六维倾向先把体魄、神识、身法、根骨除以 `10` 拉到和力道、经脉相近的尺度，再按六项权重归一化到合计 `600%`。
- 六维实际值公式：`round(singleAttr(level) * tierBaseRatio * gradeMultiplier * lowLevelNerf * attrTendency / 100)`。
- `tierBaseRatio`：普通 `50%`，精英 `80%`，Boss `100%`。
- `gradeMultiplier`：按品阶顺序使用 `1.2 ^ gradeIndex`。
- `lowLevelNerf`：1-30 级为 `70%`，31 级及以上为 `100%`。
- 例如 30 级 `singleAttr = 2916.2`，普通凡阶 `100%` 单项约 `1021`，普通地阶 `100%` 单项约 `1764`。
- 全属性格式为 `新公式实际值(整数倾向百分比)`。生命旧配置里的 `等级 * 10%` 血量系数只视为历史补血量，单独折回到约 `100%`；其它显式配置过百分比的属性内部归一化到平均 `100%`。
- 视野当前统一按默认倾向 `100%` 处理，不从旧配置反推高视野倾向。
- 全属性实际值公式：`round(singleBaseStatValue(level) * realmExpMultiplier(level) * tierBaseRatio * gradeMultiplier * lowLevelNerf * statTendency / 100 * sixDimMultiplier * pointsPerValue)`。
- `realmExpMultiplier(level)` 使用当前 `getRealmAttributeMultiplier(level)`，即 `1.1 ^ (level - 1)`。
- `sixDimMultiplier` 使用新公式算出的实际六维，按当前六维影响关系转成乘区；没有对应六维影响的属性乘区为 `1`。
- 生命最后追加层级生命倍率：普通 `1`，精英 `2`，Boss `10`。
- 视野不吃境界指数、六维、品阶、层级、1-30 级削弱，只按 `round(8 * viewRangeTendency / 100)` 计算；游戏内 Buff 仍可另行影响视野。
- 回血、回灵不吃境界指数、六维、品阶、层级、1-30 级削弱，只按 `round(50 * regenTendency / 100)` 计算，所以 `100% = 50`，`200% = 100`。
- 明细排序：等级升序，其次品阶升序，再按普通、精英、Boss 排序。
- 生成时间：2026-05-08 00:00:04（Asia/Shanghai）。本明细是当时内容快照；上方“当前倾向公式”以当前源码逻辑为准。
