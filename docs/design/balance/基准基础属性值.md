# 基准基础属性值

统计日期：2026-05-07

## 目标

本文定义“每个属性的基准基础量化值”，用来给装备、怪物、技能和后续数值配置提供同一把尺。这里不计算六维；六维仍按 `境界等级基准期望六维公式.md` 单独处理。

基准基础属性值按“每项属性”计算：

~~~text
基准基础属性值(L, p) = 凡俗基础均分值 + 境界基准基础均分值(L) + 全属性基准池(L) * p / 100
~~~

- `L`：境界等级。
- `p`：某一项属性吃到全属性基准池的占比，`100` 表示该属性吃满一份。
- `凡俗基础均分值`：当前运行时凡俗模板已有基础数值，按纳入属性数量均分后的量化值。
- `境界基准基础均分值(L)`：当前大境界相对凡俗模板增加的基础数值，按纳入属性数量均分后的量化值。
- `全属性基准池(L)`：按等级和期望强化等级得到的单项通用量化池。
- 本文不再使用总量化值作为最终结果；所有最终列都是“每个属性的量化值”。
- 同一结果已写入 `packages/server/data/content/realm-attr-baselines.json` 的 `singleBaseStatValue` 字段。

## 数据源

- 凡俗基础值和境界增量：`packages/shared/src/constants/gameplay/realm.ts` 的 `PLAYER_REALM_NUMERIC_TEMPLATES` 与 `resolvePlayerRealmNumericTemplate()`。
- 属性量化换算：`packages/shared/src/value.ts` 的 `NUMERIC_STAT_POINTS_PER_VALUE`。
- 强化增幅：参考 `packages/shared/src/enhancement.ts` 的每级 10% 指数增幅。本文用于数值期望，强化等级允许小数，不按实际装备强化等级取整。

## 全属性基准池

本稿按你当前口径使用 1 级起算：

~~~text
全属性基础池(L) = 8 + (L - 1) * 0.5
期望强化等级(L) = 按锚点线性插值
期望强化倍率(L) = 1.1 ^ 期望强化等级(L)
全属性基准池(L) = 全属性基础池(L) * 期望强化倍率(L)
~~~

期望强化等级锚点：

| 锚点等级 | 期望强化等级 | 说明 |
|---:|---:|---|
| 1 | +0 | 起点 |
| 18 | +5 | 18 级达到 +5 |
| 30 | +7 | 30 级达到 +7 |
| 42 | +10 | 42 级达到 +10 |
| 67 | +15 | 化神一层达到 +15，后续封顶 |

注意：装备基准值占比文档使用的是 `8 + level * 0.5`，因为那套口径是“0 级给到 8”。本文按“1 级全 8”重新起算，所以 1 级为 8，8 级为 11.5。

## 纳入属性

本稿把下列 20 个属性纳入“基础属性值”主池。实际属性值换算公式为：

~~~text
实际属性(k, L, p) = 凡俗实际均分(k) + 境界实际增量均分(k, L) + round(全属性基准池(L) * p / 100 * 每1量化值对应实际点数(k))
~~~

| 属性 | 字段 | 每 1 量化值对应实际点数 |
|---|---|---:|
| 生命 | `maxHp` | 12 |
| 灵力 | `maxQi` | 8 |
| 物攻 | `physAtk` | 1 |
| 法攻 | `spellAtk` | 1 |
| 物防 | `physDef` | 1 |
| 法防 | `spellDef` | 1 |
| 命中 | `hit` | 1 |
| 闪避 | `dodge` | 1 |
| 暴击 | `crit` | 1 |
| 抗暴 | `antiCrit` | 1 |
| 暴伤 | `critDamage` | 1 |
| 破防 | `breakPower` | 1 |
| 化解 | `resolvePower` | 1 |
| 灵力输出 | `maxQiOutputPerTick` | 1 |
| 冷却 | `cooldownSpeed` | 1 |

| 属性 | 字段 |

| 额外仇恨 | `extraAggroRate` |
| 额外射程 | `extraRange` |

| 每息行动次数 | `actionsPerTurn` |
| 元素增伤 | `elementDamageBonus` |
| 元素减伤 | `elementDamageReduce` |

说明：自动恢复和修炼/经验相关按你的要求排除；视野沿用装备基准改造时的口径，不再进入基础数值池；射程、范围、行动次数和元素增减伤属于特殊机制，先不放进通用基准。
