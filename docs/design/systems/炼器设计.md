# 炼器设计

本文档以当前生产主线配置为基线，定义炼器系统的内容口径、材料红线和后续落地边界。当前只写设计，不修改配置；后续若落地，建议新增 `packages/server/data/content/forging/recipes.json` 作为器方真源，并同步 shared 类型、服务端内容加载、运行时和客户端制造面板。

## 定位

炼器是技艺活动中的装备本体生产线，负责把怪物异材、矿材和长期炼器等级转化为可穿戴装备。它不替代强化系统：

- 炼器：制造装备本体，决定装备的基础品阶、等级、部位和基础词条方向。
- 强化：在既有装备上提升 `enhanceLevel`，消耗灵石、强化材料和保护物，负责同一件装备的纵向成长。
- 炼丹：主要消耗药材和少量特殊药引生产丹药；炼器不得复用草药型药材池，避免草药、花叶、藤蔓类材料同时承担丹药和装备两套经济压力。

服务端仍是唯一权威：开炉、扣材料、扣灵石、创建作业、tick 推进、成败结算、产出装备、写入审计和持久化都必须发生在服务端受控运行时内。客户端只展示器方、候选材料、作业进度和结果，不裁定材料是否合法、成功率、耗时或产出。

## 材料红线

炼器材料只能使用异材。本文把异材定义为：

- 怪物产出的硬质、灵性或异质材料。
- 矿物采集产出的矿材。
- 后续明确标记为炼器异材的特殊容器产物。

以下材料不得进入炼器器方：

- 草木药材、花叶蕊类药引，以及明确标记为 `herb` 采集线的材料，即使它们在特殊剧情怪物掉落里出现，也仍按药材处理。
- 丹药、符书、地图、货币、任务钥物和纯剧情道具。
- 灵石不作为主材，只能作为开炉费用、燃料或系统手续费。
- 已成型装备不作为普通材料；若用于保护、吞噬或拆解，必须走强化或独立拆解规则，不混入基础器方。

当前矿类只认玄铁矿线：

| 来源 | 当前配置 | 产物 | 设计口径 |
|---|---|---|---|
| `tile.black_iron_ore` | `packages/server/data/content/resource-nodes.json` | `black_iron_chunk` 玄铁矿块 | 可作为矿材主材。 |
| `landmark.black_iron_vein` | 玄铁矿脉标记 | `black_iron_chunk` 玄铁矿块 | 可作为矿材主材。 |
| `landmark.ore_basement_black_iron_vein` | 矿窖玄铁矿脉容器 | 按标签池抽取矿材 | 后续需补齐材料标签后才能稳定作为炼器矿脉。 |

注意：当前 `black_iron_chunk` 是 11 级凡阶材料，不能强行用于 2 到 4 级新手装备器方，否则会出现成品等级远低于材料等级的倒挂。低级装备若要炼器化，应先新增低级矿材，例如碎玄铁、粗铁砂；在新增之前，低级装备继续主要由任务、怪物或商店投放。

## 与炼丹材料分流

当前炼丹配置已经使用一部分怪物异材或矿化异材，例如陨火砂、炎髓炭、寒魄露、承脉石、锋纹残晶、净潮水精、炉心赤晶、生灵木心、镇岳石胆、五炁尘、归藏脉核和半基灵胚。炼器初版如果继续使用这些材料，需要满足其中一种口径：

- 共享材料：同一异材同时服务炼丹和炼器，但配方件数必须保守，且投放量要能支撑双系统消耗。
- 分流材料：该异材只保留给其中一个系统，另一个系统改用同地图段其他异材。
- 变体材料：后续新增炼器专用矿材或器核，例如寒铁矿、赤铜砂、星陨矿、妖骨器胚、兽鳞甲片。

初版建议：

- 凡人期炼器优先使用玄铁矿块、灵铁碎片、星陨金、断纹石片、岭兽爪、谷蛇逆鳞等目前不在核心丹方里的硬质异材。
- 练气期炼器优先围绕裂锋原金行异材、厚脉岭土行异材和后续新增矿材设计，少量使用五行核心，不要让每个五行核心同时成为丹药和装备的硬门槛。
- 明确不使用 `landmark.herb.*` 草药采集线作为炼器主材，即使名称带金、铁、火，也必须先判断它在内容分类里是不是药材。

## 器方结构

建议器方使用稳定结构：

```json
{
  "recipeId": "forging.black_iron_blade",
  "outputItemId": "equip.black_iron_blade",
  "outputCount": 1,
  "baseForgeTicks": 180,
  "spiritStoneCost": 4,
  "ingredients": [
    { "itemId": "black_iron_chunk", "count": 2, "role": "core" },
    { "itemId": "crystal_dust", "count": 1, "role": "binder" }
  ]
}
```

字段口径：

- `core`：主材，决定装备品阶、等级段和主要流派。
- `binder`：绑定材，用于把矿材、骨材、晶尘或灵纹稳定成装备。
- `catalyst`：催化材，只用于高阶或特殊器方，不能成为低阶器方常规税。
- `baseForgeTicks`：单件基础炼制息数，装备本体制造比强化单阶更长，但比批量炼丹更少重复。
- `spiritStoneCost`：开炉和稳定器纹费用，不参与材料合法性。

基础约束：

- 成品等级不应低于主材等级太多；初版建议主材等级在 `成品等级 - 3` 到 `成品等级 + 3`。
- 成品品阶必须不高于最高主材品阶；允许辅材低一阶，但不允许高阶成品靠堆低阶异材硬凑。
- 每个器方材料件数以 2 到 4 件为主；高阶装备通过高品阶核心和灵石成本拉开门槛，不靠低级材料堆 10 件以上。
- 同一地图段的器方优先使用同地图段怪物异材和矿材，避免玩家回低级图刷大量低阶材料。
- 器方不产出强化等级，产物默认 `enhanceLevel = 0`。

## 成败和耗时

炼器可以复用制造型技艺的低频作业模型，但成功/失败结果应不同于炼丹：

- 开始炼器时一次性扣除材料和灵石。
- 成功产出 1 件装备本体。
- 失败不产出装备；材料不返还，但可以按成品等级给炼器经验。
- 炼器等级低于成品等级时降低成功率；高于成品等级时做渐近提升，不能溢出成永久 100%。
- 装备品阶越高，基础耗时越长；同等级恢复/增益丹的产量逻辑不能套用到装备。
- 移动、出手或显式取消应中断当前炼器；是否返还材料需要作为后续规则单独确认，默认不返还。

建议初始公式：

```text
基础成功率 = 0.72 + min(0.18, 炼器等级差正向收益)
低等级修正 = 基础成功率 * 0.92^(成品等级 - 炼器等级)
品阶修正 = 按 yellow/mystic/earth 逐级降低少量成功率
最终成功率 = clamp(修正后成功率 + 装备炼器词条, 0.05, 0.98)
```

耗时建议：

```text
基础炼制息 = 120 + 成品等级 * 6 + 品阶附加息
总作业息 = 备炉 10 息 + 基础炼制息 / (1 + 炼器速度修正)
```

## 配置与运行时边界

后续落地建议按以下顺序：

1. 配置层：新增 `packages/server/data/content/forging/recipes.json`，只声明器方，不把运行时状态塞进配置。
2. shared 层：新增 `ForgingRecipeCatalogEntry`、`PlayerForgingJob`、`SyncedForgingPanelState`，复用 `CraftQueueItemView.kind = 'forging'`。
3. 服务端内容加载：启动期解析器方，校验材料存在、材料属于异材、成品是装备、等级和品阶不倒挂。
4. 运行时：在 `CraftPanelRuntimeService` 接入 `startForging / tickForging / cancelForging`，并进入 `player_active_job` 统一作业持久化。
5. 网络：炼器面板走按需同步，作业状态只发给本人，不进入 AOI 高频包。
6. 客户端：替换当前炼器占位面板，展示器方、材料缺口、成功率、耗时、队列和结果。
7. 配置编辑器：新增器方编辑能力，必须带材料合法性校验，禁止选入药材。

当前代码事实：

- `CraftQueueItemView` 已有 `forging` 枚举。
- 客户端制造面板已有炼器标签，但显示“炼器配方尚未接入”。
- 服务端队列遇到炼器会返回“炼器任务暂未接入运行时。”

这说明炼器不是全新入口问题，而是缺少器方配置、shared 结构、服务端运行时和面板真实数据。

## 验证计划

落地配置和运行时后，至少补以下验证：

- 内容引用审计：器方材料、成品装备、灵石费用引用都存在。
- 材料红线审计：任何 `forging` 器方不得引用药材、丹药、任务物或装备。
- 等级品阶审计：成品等级和主材等级差不超过设计阈值，成品品阶不高于主材上限。
- 服务端 smoke：材料足够可开始炼器，材料不足列出每项缺口，成功/失败都能正确扣材料和写作业状态。
- 持久化 smoke：炼器作业开始、取消、完成后 `player_active_job`、背包、装备产出和审计记录可恢复。
- 协议审计：炼器面板只同步本人低频状态，不把静态器方塞进高频 AOI 包。
- 客户端 smoke：炼器面板能展示材料缺口、成功率、耗时和队列，且不打断炼丹/强化现有面板状态。
