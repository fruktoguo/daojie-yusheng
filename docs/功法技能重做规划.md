# 功法技能重做规划

状态：草案 v1

用途：

- 这份文档只用于本轮功法技能重做规划
- 先确定哪些功法保留技能、哪些改为纯内功
- 后续由你逐个指定技能效果，我再按文档实现到正式配置

相关约束：

- 每个功法最多只有 `1` 个技能
- 凡人期总技能数为 `9`
- 练气期总技能数为 `18`
- `神通` 不计入练气期这 `18` 个技能预算
- `太初抱元章` 必须有技能
- `归藏合脉经` 必须有技能

## 1. 设计口径

本轮重做默认采用下面的收束方式：

- 少数功法保留主动技能，作为玩家战斗手段来源
- 大多数功法回归内功定位，负责属性成长、续航、护体、修炼效率或流派底盘
- 不再让单个功法自带多个技能，避免技能池膨胀
- 等你逐条定效果后，再正式回写到真源配置

### 1.1 百分比系数折算机制（全局）

本轮新增统一机制：

- 技能公式里的“百分比攻击系数”先做折算，再进入最终伤害公式
- 前 `200%` 按完整收益计算
- 超过 `200%` 的部分按递减收益处理，并且要做平滑过渡

锚点口径按当前方案固定为：

- 标注 `200%` => 有效 `200%`
- 标注 `300%` => 有效 `250%`
- 标注 `400%` => 有效 `300%`
- 标注 `500%` => 有效 `325%`

执行规则：

- 文档里继续保留“原始标注系数”（例如 `500%`）便于设计沟通
- 正式落地配置与结算实现时，统一按上述折算机制换算为“有效系数”
- 所有已定和后续新增技能都遵守同一套折算口径

### 5.1 凡人期已确定技能细则

#### 青木剑诀

- 技能名：`青木剑诀`
- 类型：单体直接伤害
- 射程：`2`
- 范围：单体
- 基础公式：`30 + 400% 物理攻击力`
- 层数成长：每层功法层数提供 `+10%` 总伤害加成
- 备注：先不附带额外印记或持续效果，作为凡人期最朴素的木系输出技能

#### 赤焰诀

- 技能名：`赤焰诀`
- 类型：单体火系伤害 + 施加灼烧
- 射程：`3`
- 范围：`3 x 3`
- 基础公式：`50 + 400% 法术攻击力`
- 层数成长：每层功法层数提供 `+10%` 总伤害加成
- 灼烧联动：目标身上每有 `1` 层灼烧，本次技能额外 `+1%` 总伤害
- 附带效果：命中后施加 `5` 层灼烧

#### 流云刀谱

- 技能名：`流云刀谱`
- 类型：单体直接伤害
- 射程：`3`
- 范围：横向 `3 x 1`
- 基础公式：`50 + 500% 物理攻击力`
- 层数成长：每层功法层数提供 `+10%` 总伤害加成
- 移速联动：额外吃移速百分比加成，系数 `0.1%`
- 备注：定位为凡人期机动型物理输出技能

#### 惊雷掌

- 技能名：`惊雷掌`
- 类型：单体法术伤害 + 施加雷系 Debuff
- 射程：`2`
- 范围：单体
- 基础公式：`50 + 600% 法术攻击力`
- 层数成长：每层功法层数提供 `+10%` 总伤害加成
- Debuff 联动：目标身上每有 `1` 层雷系 Debuff，本次技能额外 `+10%` 总伤害
- 附带效果：命中后施加雷系 Debuff，最多 `5` 层
- Debuff 效果：每层降低目标 `20%` 护甲与魔抗

### 5.5 镇灵系统一 Debuff 口径

后续镇压、封灵、压制类技能如果走同一体系，尽量统一复用 `镇灵` Debuff，不再每个技能单独造一套近似效果。

当前统一原则：

- Debuff 名统一为 `镇灵`
- 这一体系主要体现封镇、迟滞、压法，不走现代感电式命名
- 后续正式落地时，尽量统一复用同一个 `镇灵` Debuff 真源
- 当前凡人期定义里，`镇灵` 使目标 `-50%` 移速、`-100%` 法抗

### 5.4 冰系统一 Debuff 口径

后续冰系技能不再各自定义一套完全不同的冰霜类 Debuff，统一按共享冰系 Debuff 口径处理。

当前统一原则：

- 冰系 Debuff 统一命名为 `冰霜`
- 后续正式落地时，尽量统一复用同一个冰系 Debuff 真源
- 不同冰系技能的区别主要体现在：
  - 施加层数不同
  - 是否读取层数增伤
  - 是否消耗层数
  - 是否额外附带定身、迟滞或控场
- 当前凡人期定义里，`冰霜` 每层降低目标 `10%` 移速与 `20%` 闪避，最多 `5` 层

### 5.3 雷系统一 Debuff 口径

后续雷系技能不再各自定义一套完全不同的“感电”类 Debuff，统一按共享雷系 Debuff 口径处理。

当前统一原则：

- 雷系 Debuff 统一命名为 `雷殛`
- 命中描述走修仙风格，不直接使用纯现代词“感电”
- 后续正式落地时，尽量统一复用同一个雷系 Debuff 真源
- 不同雷系技能的区别主要体现在：
  - 施加层数不同
  - 是否读取层数增伤
  - 是否消耗层数
  - 是否额外附带控制或破防
- 当前凡人期定义里，`雷殛` 每层降低目标 `20%` 护甲与魔抗，最多 `5` 层

### 5.2 火系统一 Buff 口径

后续火系技能不再各自定义一套完全不同的灼烧 Buff，统一按共享火系灼烧口径处理。

当前统一原则：

- 火系灼烧统一复用共享 Buff
- 后续正式落地时，优先统一到现有共享火系灼烧 Buff：`buff.fire_burn_mark`
- Buff 展示名也尽量统一，不再在同一体系里混用多套“灼印 / 灼脉 / 火烙”作为不同 Buff 真源
- 不同火系技能的区别主要体现在：
  - 施加层数不同
  - 持续时间不同
  - 是否读取灼烧层数
  - 是否引爆或消耗灼烧
- 不再通过“每个技能都定义一个新灼烧 Buff”来制造差异
