# UI 三态自检（浅色 / 深色 / 手机）

本次涉及 UI（DOM UI、`react-ui`、HUD、面板、弹层、tooltip、Canvas overlay 等）。请按下面清单逐项确认。

## 三态适配

- **浅色模式**：色板、对比度、阴影、边框是否仍清晰；与现有主题 token 一致。
- **深色模式**：是否复用 `packages/client/src/styles/**` 里的 token，没有硬编码 `#fff / #000`；图标、徽章、胶囊（`chat-target-pill` / `chat-skill-pill` / `chat-damage-pill` / `chat-combat-badge`）有合适反色。
- **手机模式**：触控命中尺寸、安全区、滚动路径、弹层高度、固定按钮遮挡、虚拟键盘弹起后的可视区域、横竖屏切换。

## 更新策略

- 优先局部 patch；除首次初始化、跨场景重建、空态变更、结构整体变化外，**没有整页 / 整面板 / 整弹层全量刷新**。
- 高频更新不打断当前焦点、滚动位置、选区、展开态、输入、交易、阅读、选择、拖拽。
- "点击展开详情"使用单实例详情弹层，并支持点击外部关闭。

## 接入与边界

- 复用现有 store、弹层宿主、样式 token、响应式工具、模块边界；不新建一套并行真源。
- React UI 不绕过现有网络 / 运行时 / UI 状态边界；DOM UI 与 React UI 不维护两套互相冲突的真源。
- 通知消息符合 `AGENTS.md` 第 22 节：服务端只发结构化 `{ key, vars, pills, badges }`，客户端按语言包模板渲染胶囊；**禁止在客户端用正则解析纯文本**还原结构化信息。
- Canvas 渲染：静态层、动态实体层、overlay 层尽量分离更新；高频变化只更新受影响区域。

## 必跑门禁

- `pnpm verify:client`
- 涉及 React UI / DOM UI 同屏：本地起 `pnpm dev:client` 做目检
- 涉及通知 / 战斗文案：同步检查 `i18n.generated.ts` 模板与 `pills` 标记
