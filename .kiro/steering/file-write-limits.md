# Kiro 专属规则：单次文件写入行数限制

> 适用范围：`kiro-cli chat` / Kiro 默认 agent。本文件仅由 Kiro 通过 `.kiro/steering/**/*.md` 加载，与通用的 `AGENTS.md` 并行生效。

## 规则

- **单次文件写入操作的内容不得超过 200 行**。
- "单次文件写入"指一次工具调用所提交的内容，包括但不限于：
  - `write` 工具的 `create` 命令（`content` 参数）
  - `write` 工具的 `insert` 命令（`content` 参数）
  - `write` 工具的 `strReplace` 命令（`newStr` 参数）
  - 通过 `shell` 等工具执行的 heredoc / `cat > file` / `tee` / `printf` 等等价写入
- 行数以提交内容中的换行符（`\n`）数量 + 1 计算，不区分代码、注释或空行。

## 超出处理

如果计划写入的内容超过 200 行，必须拆分为多次写入：

1. 先用 `write` 的 `create` 命令写入第一段（≤ 200 行），可以是文件骨架或前 N 行实际内容。
2. 后续内容通过 `write` 的 `insert` 命令按段追加，每段 ≤ 200 行，直到文件完整。
3. 如需在已有文件中插入或替换大段内容，同样按 ≤ 200 行的段拆成多次 `insert` / `strReplace` 调用。
4. 拆分时尽量在自然边界（函数、类、章节、顶层语句）处切分，避免破坏语法或语义连续性。

## 不允许的规避方式

- 不得通过 `shell` 工具的 heredoc、`echo`、`printf`、`sed -i` 等渠道一次性写入超过 200 行内容来绕过本规则。
- 不得为了压缩行数而把多行代码人为合并到同一行（影响可读性、diff 与回归定位）。

## 例外

- 仅限"完整删除文件"或"将文件清空"这类不携带新内容的操作不受行数限制约束。
- 自动格式化、`format` 工具等不直接由模型构造内容的操作不计入本规则。
